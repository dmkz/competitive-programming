## Задача A. Тип линии

- Имя входного файла: стандартный ввод
- Имя выходного файла: стандартный вывод
- Ограничение по времени: 1 секунда
- Ограничение по памяти: 1024 мегабайта

**Это интерактивная задача.**

Во время туристической поездки вы прибыли в крупный город в далёкой, редко посещаемой обычными туристами стране и сейчас находитесь на станции метро.

С каждой из сторон платформы размещён список станций, до которых можно доехать по этой линии в соответствующем направлении. К сожалению, вы не знаете алфавита, принятого в этой стране, а перевод списка на более известные вам языки отсутствует. Так что вы можете только подсчитать количество станций.

Линии в метро этого города, как и обычно, бывают кольцевыми и радиальными. В случае кольцевой линии название станции размещают с той стороны, с которой до этой станции можно доехать за наименьшее число перегонов (один перегон соединяет две конечные станции). Если до станции можно доехать за одинаковое число перегонов с той или другой стороны, название станции размещают только с одной из сторон (регулярного правила, с какой именно, вам обнаружить не удалось).

Вы можете сесть в поезд с любой из двух сторон и проехать ровно один перегон. Ваша задача — определить, является ли линия, на которой вы находитесь, кольцевой или радиальной.

### Протокол взаимодействия

Взаимодействие начинает программа жюри, выводя два целых числа `p` и `q` — количество станций в каждом направлении. Числа отсортированы по неубыванию:

`0 ≤ p ≤ q ≤ 24`, `10 ≤ p + q ≤ 24`.

После этого вы выводите строку с числом `1`, если хотите проехать в направлении, в котором указаны `p` станций, и с числом `2`, если хотите проехать в направлении, в котором указаны `q` станций.

Если `p = 0` и вы выбрали это направление, то вы уехали в депо и ваше решение будет признано неверным.

Иначе программа жюри выведет два целых числа `p1` и `q1` — количество станций, указанных в каждом направлении на новой станции, также отсортированные по неубыванию, то есть `p1 ≤ q1`.

После чего ваша программа должна вывести `c`, если это кольцевая линия, и `r`, если линия радиальная.

Информация о конфигурации линии при каждом взаимодействии определяется перед его началом, то есть интерактор не является адаптивным. Гарантируется, что сообщаемые `p`, `q`, `p1`, `q1` корректны при соответствующем типе линии.

### Пример

Один из возможных сеансов взаимодействия:

**Стандартный ввод**
```text
3 8
4 7
```

**Стандартный вывод**
```text
2
r
```

### Замечание

Не забывайте после вывода направления поездки или типа линии выводить перевод строки, а также сбрасывать буфер ввода-вывода вызовом функции `flush` используемого вами языка программирования.

## Задача B. Различные номера

- Имя входного файла: стандартный ввод
- Имя выходного файла: стандартный вывод
- Ограничение по времени: 1 секунда
- Ограничение по памяти: 1024 мегабайта

Недавно компания МТС начала акцию вместе с известной сетью автосалонов. При регистрации нового автомобиля владельцу предоставляется на выгодных условиях десятизначный номер, в котором первая цифра равна `9`, а девять последних цифр могут быть получены из номерного знака следующим образом:

- Номерной знак личного автомобиля имеет вид `lDDDll-DDD`, где `l` — место для буквы латинского алфавита из подмножества `A, B, C, E, H, K, M, O, P, T, X, Y`, а `D` — место для цифры от `0` до `9`.
- Все цифры в телефонном номере, кроме первой девятки, соответствуют цифрам в номерном знаке, буквам же соответствуют цифры, которых нет в номерном знаке, при этом одинаковым буквам соответствуют одинаковые цифры, а разным — разные.
- Первая девятка не оказывает влияния на выбор цифр и просто приписывается к уже выбранным 9 цифрам номера.

Например, если у автомобиля номер `A451HA-077`, то телефонный номер может выглядеть как `9245192077` (A заменили двойкой, H — девяткой), или как `9345123077` (тройкой и двойкой соответственно):

```text
A451HA077
9245192077
9345123077
```

Но соответствующий этому номерному знаку телефонный номер не может выглядеть, например, как `9245172077` (так как семёрка и заменяет `H`, и встречается в номере как цифра), или как `9245122077` (так как двойка заменяет и `H`, и `A`), или как `9245136077` (так как буква `A` заменена в одном случае двойкой, а в другой — тройкой).

При этом, как можно заметить, одному автомобильному номеру может соответствовать несколько телефонных номеров. Вам дано целое число `x`, определите, существует ли автомобильный номер, который имеет ровно `x` различных телефонных номеров, соответствующих ему (то есть `x + 1` различный номер уже поставить в соответствие нельзя), и, если существует, выведите лексикографически наименьший такой номер.

Номер `p` считается лексикографически меньше номера `q`, если самый левый из несовпадающих символов меньше: если это цифра, то меньше цифра, если это буква, то меньше буква, которая идёт раньше в латинском алфавите.

### Формат ввода

Первая строка входных данных содержит одно целое число `T` — количество тестовых примеров (`1 ≤ T ≤ 1000`).

Каждая из последующих `T` строк содержит по одному целому числу `x` (`1 ≤ x ≤ 1000`).

### Формат вывода

Для каждого `x`, если не существует автомобильного номера, которому соответствует ровно `x` телефонных номеров в рамках акции, выведите `-1`.

Иначе выведите лексикографически наименьший номер, который обладает таким свойством.

### Пример

**Стандартный ввод**
```text
2
19
5
```

**Стандартный вывод**
```text
-1
A001AA234
```

## Задача C. Уровни доверия

- Имя входного файла: стандартный ввод
- Имя выходного файла: стандартный вывод
- Ограничение по времени: 1 секунда
- Ограничение по памяти: 1024 мегабайта

При современном распространении спама и мошеннических звонков оценка уровня доверия к входящему номеру является важной функцией, предоставляемой мобильным оператором. Разумеется, такая оценка формируется на базе информации от абонентов. Компания МТС тестирует новую систему оценки, которая позволяет автоматически поддерживать несколько уровней доверия — «неизвестный», «обычный», «доверенный», «привилегированный» и так далее. Уровни занумерованы целыми числами, начиная с `0`.

Также существует внутренний параметр — «оценка номера».

До первого звонка с этого номера уровень номера в списке равен `0`, текущая оценка номера также равна `0`.

Далее происходит следующий процесс:

- Идёт входящий звонок.
- Абонент ставит оценку звонка от `-10^9` до `10^9` включительно.
- Оценка прибавляется к текущей оценке номера. Если в результате получилось отрицательное число, текущая оценка становится равной `0`.
- Как только оценка достигла или превысила заданное в алгоритме значение `k`, текущая оценка устанавливается в `0`, а номер переходит на следующий уровень (то есть уровень увеличивается на `1`).

Екатерина — давний абонент МТС. Она участвует в тестировании новой системы. У неё есть список звонков с некоторого номера и список оценок, которые она ставила этим звонкам. Также она узнала уровень номера, который он получил после всех этих звонков. Екатерину заинтересовал возможный интервал, в котором может изменяться значение `k`. Помогите ей и вычислите наибольшее и наименьшее возможные значения `k` или определите, что в алгоритме есть баг и ситуация невозможна ни при каком значении `k`.

### Формат ввода

Первая строка входных данных содержит два целых числа `n` и `r` (`1 ≤ n ≤ 10^5`, `1 ≤ r ≤ 10^4`) — количество звонков с некоторого номера и уровень этого номера, выставленный новой системой оценки МТС, соответственно.

`i`-я из последующих `n` строк содержит одно целое число `c_i` — оценку, которую Екатерина выставила после `i`-го звонка (`-10^9 ≤ c_i ≤ 10^9`).

### Формат вывода

Выведите два целых числа — минимальное и максимальное значения `k`, при которых номер получит уровень `r` при указанных оценках Екатерины.

Если это невозможно ни при каком значении `k`, выведите одно число `-1`.

### Примеры

#### Пример 1

**Стандартный ввод**
```text
4 2
3
6
-4
10
```

**Стандартный вывод**
```text
4 9
```

#### Пример 2

**Стандартный ввод**
```text
1 1
0
```

**Стандартный вывод**
```text
-1
```

## Задача D. Деньги на стол!

- Имя входного файла: стандартный ввод
- Имя выходного файла: стандартный вывод
- Ограничение по времени: 1 секунда
- Ограничение по памяти: 1024 мегабайта

Коллекционер редких купюр Скрудж рассматривает свою коллекцию, одновременно раскладывая пасьянс. `i`-я купюра номинирована в валюте страны `c_i` и оценивается на аукционах в `p_i` долларов.

Скрудж выкладывает купюры на стол в ряд в некотором порядке слева направо. Изначально результат Скруджа равен `0`.

После того, как купюры выложены, Скрудж может неограниченное число раз проделать следующее действие: выбрать несколько (две или более) подряд идущих купюр, из которых самая левая и самая правая номинированы в валюте одной и той же страны, убрать все эти купюры и прибавить к результату суммарную стоимость этих купюр на аукционах.

Две купюры считаются идущими подряд, если между ними нет других купюр (в том числе и в результате убирания блока купюр на предыдущем ходу).

По заданному расположению купюр (последовательностям `c_i` и `p_i`) определите, какого максимального результата сможет добиться Скрудж при оптимальном выборе порядка действий.

### Формат ввода

Первая строка входных данных содержит два целых числа `n` и `m` — количество купюр на столе и максимальный возможный номер страны, соответственно (`1 ≤ n, m ≤ 10^6`).

Вторая строка содержит `n` целых чисел `c_i`; `i`-е из этих чисел задаёт номер страны, в валюте которой номинирована `i`-я купюра (`1 ≤ c_i ≤ m`).

Третья строка содержит `n` целых чисел `p_i`; `i`-е из этих чисел задаёт стоимость купюры на аукционах (`1 ≤ p_i ≤ 10^9`).

### Формат вывода

Выведите одно целое число — максимальный результат, которого может добиться Скрудж при оптимальном выборе порядка действий.

### Пример

**Стандартный ввод**
```text
7 3
2 3 2 3 1 3 1
4 5 6 7 8 10 9
```

**Стандартный вывод**
```text
45
```

## Задача E. Отличный сигнал

- Имя входного файла: стандартный ввод
- Имя выходного файла: стандартный вывод
- Ограничение по времени: 1 секунда
- Ограничение по памяти: 1024 мегабайта

Если в первые годы развития мобильной связи про мобильный интернет в метро можно было забыть, то сейчас всё наоборот — мобильный интернет в метро работает стабильно. Для этого в метро расставлены базовые станции.

Новая очередь метро города `S` состоит из `n` станций, соединённых `m` перегонами так, что между любыми двумя станциями можно проехать, не выходя из метро. Каждый перегон соединяет напрямую некоторые две станции; разумеется, поезда по перегонам ходят в обе стороны. Никакие две станции не соединены напрямую более чем одним перегоном.

Прокладка тоннелей уже завершена, идёт установка оборудования, так что МТС устанавливает на станциях метро свои базовые станции.

На схеме метро станции и перегоны занумерованы числами от `1` до `n` и от `1` до `m` соответственно. Пусть `j`-й перегон соединяет станции `a_j` и `b_j`. По расчётам, для уверенного приёма на перегоне нужен уровень сигнала, в точности равный `c_j` (уровень сигнала — целое число в диапазоне от `-10^6` до `10^6`).

Уровень сигнала на перегоне равен сумме уровней сигнала базовых станций на двух станциях, которые он соединяет. В силу конструкции используемых базовых станций их уровень также может быть только целым числом.

Стоимость установки базовой станции с уровнем `l_i` равна `|l_i|`.

Требуется по заданной схеме новой очереди метро и значениям `c_i` определить, существует ли способ расстановки базовых станций МТС по станциям метро таким образом, чтобы достигались требуемые значения `c_i`. Если такой способ есть, требуется вывести одну из возможных расстановок, которая минимизирует суммарную стоимость установки всех базовых станций.

### Формат ввода

Первая строка входных данных содержит два целых числа `n` и `m` (`2 ≤ n ≤ 10^5`, `1 ≤ m ≤ 2 · 10^5`) — количество станций в новой очереди и количество перегонов, соответственно.

Каждая из последующих `m` строк содержит по три целых числа `a_i`, `b_i` и `c_i` (`1 ≤ a_i, b_i ≤ n`, `a_i ≠ b_i`, `-10^6 ≤ c_i ≤ 10^6`) — номера станций, соединённых очередным перегоном, и требуемый уровень сигнала на перегоне.

Гарантируется, что любая пара станций соединена напрямую не более чем одним перегоном и что между любыми двумя станциями можно проехать, используя один или несколько перегонов.

### Формат вывода

Если способа расстановки базовых станций не существует, выведите в первой строке `No`.

Иначе выведите в первой строке `Yes`, а во второй — `n` целых чисел `x_i`, `i`-е из которых соответствует уровню сигнала базовой станции, устанавливаемой на `i`-й станции метро, таких, что сумма абсолютных значений всех `x_i` минимальна.

Если вариантов расстановки, минимизирующих суммарную стоимость, несколько, можно вывести любой.

### Примеры

#### Пример 1

**Стандартный ввод**
```text
3 3
2 1 4
1 3 2
3 2 3
```

**Стандартный вывод**
```text
No
```

#### Пример 2

**Стандартный ввод**
```text
5 4
2 3 6
1 3 -3
4 1 -11
5 3 4
```

**Стандартный вывод**
```text
Yes
-7 2 4 -4 0
```

## Задача F. Ёлочные строки

- Имя входного файла: стандартный ввод
- Имя выходного файла: стандартный вывод
- Ограничение по времени: 1 секунда
- Ограничение по памяти: 1024 мегабайта

Рассмотрим строки длины `n`, составленные из заглавных латинских букв.

Назовём ёлочными строки, обладающие следующим свойством: ни одна подстрока этой строки (то есть `k` подряд идущих символов, где `1 ≤ k ≤ n`) не образует чётное римское число.

Например, строка `START` является ёлочной (в ней нет ни одной подстроки, образующей какое-то римское число, а значит, и нет ни одной подстроки, образующей чётное римское число), строка `RUCODE` не является ёлочной (подстрока с 3 по 3 символ `C` задаёт чётное римское число `100`), строка `WINNER` является ёлочной (единственная подстрока, задающая римское число — подстрока с 2 по 2 символ, которая задаёт число `1`).

Напоминаем способ записи римских чисел:

| Значение разряда | Тысячи | Сотни | Десятки | Единицы |
|---|---|---|---|---|
| 1 | M | C | X | I |
| 2 | MM | CC | XX | II |
| 3 | MMM | CCC | XXX | III |
| 4 | CD | XL | IV |
| 5 | D | L | V |
| 6 | DC | LX | VI |
| 7 | DCC | LXX | VII |
| 8 | DCCC | LXXX | VIII |
| 9 | CM | XC | IX |

Заметим, что:

- Числа `4`, `9`, `40`, `90`, `400` и `900` записываются в реверсивной нотации, где первый символ вычитается из второго (например, для `40` (`XL`) `X` (`10`) вычитается из `L` (`50`)). Это единственные места, где реверсивная нотация используется.
- Число, содержащее несколько десятичных цифр, строится дописыванием римского эквивалента каждой цифры от старшего разряда к младшему.
- Если в десятичном разряде стоит `0`, никаких цифр в этом разряде в римском представлении не пишется.
- Наибольшее число, которое может быть представлено в римской системе счисления — это число `3999` (`MMMCMXCIX`).

Так как ёлочных строк может быть очень много, выведите остаток от деления их количества на `998244353`.

### Формат ввода

Первая строка входных данных содержит одно целое число `n` — длину строки (`1 ≤ n ≤ 10^12`).

### Формат вывода

Выведите одно целое число — остаток от деления количества различных ёлочных строк длины `n` на `998244353`.

### Примеры

#### Пример 1

**Стандартный ввод**
```text
1
```

**Стандартный вывод**
```text
21
```

#### Пример 2

**Стандартный ввод**
```text
2
```

**Стандартный вывод**
```text
438
```

## Задача G. Массив и запросы

- Имя входного файла: стандартный ввод
- Имя выходного файла: стандартный вывод
- Ограничение по времени: 1 секунда
- Ограничение по памяти: 1024 мегабайта

Вам дан массив `a_i` из целых чисел от `0` до `n - 1`, где `n` — степень двойки. Все `a_i` попарно различны и принимают значения от `0` до `n - 1`.

Ваша задача — обрабатывать следующие запросы к массиву:

- `1 x` — применить операцию `a_i := a_i xor x` для всех `i` от `0` до `n - 1`, где `xor` — побитовое исключающее ИЛИ.
- `2 l r` — найти длину наименьшего подмассива массива `a`, в котором встретятся все числа `l, l + 1, l + 2, ..., r`.

### Формат ввода

Первая строка входных данных содержит два целых числа `n` и `q` — длина массива и количество запросов (`1 ≤ n, q ≤ 2^18`, `n = 2^k` для некоторого целого `k`).

Вторая строка содержит `n` чисел `a_i` — начальные значения элементов массива (`0 ≤ a_i ≤ n - 1`, `a_i ≠ a_j` при `i ≠ j`). Массив индексирован с нуля.

Каждая из последующих `q` строк содержит по одному запросу. Запросы имеют следующий формат:

- `1 x` — применить `a_i := a_i xor x` для всех элементов массива (`0 ≤ x ≤ n - 1`).
- `2 l r` — запрос на длину наименьшего подмассива, содержащего числа от `l` до `r` включительно (`0 ≤ l ≤ r ≤ n - 1`).

### Формат вывода

Для каждого запроса второго типа выведите одно целое число — ответ на запрос.

### Пример

**Стандартный ввод**
```text
8 10
4 5 6 1 3 7 0 2
1 2
2 4 6
1 2
2 5 7
1 4
1 5
2 5 6
1 0
2 7 7
2 2 3
```

**Стандартный вывод**
```text
6
5
6
1
4
```

## Задача H. Кикшеринг

- Имя входного файла: стандартный ввод
- Имя выходного файла: стандартный вывод
- Ограничение по времени: 1 секунда
- Ограничение по памяти: 1024 мегабайта

В последнее время сервисы кикшеринга (прокат электросамокатов) стали очень популярными. Среди самых известных сервисов — МТС-Юрент.

Недавно в одном городе построили новый бизнес-центр. Всего в бизнес-центре `n` офисов различных компаний. Одна из этих компаний — МТС Финтех. Для каждой компании известны координаты офиса, максимальное расстояние `r_i` от станции метро, при котором посетитель этой компании поедет на самокате (поскольку рекомендации указаны на сайтах компаний в разделах «как добраться», для всех посетителей одной компании это расстояние совпадает), а также количество самокатов `e_i`, которые уже были оставлены у данного офиса сотрудниками к моменту начала рабочего дня.

После начала рабочего дня в офисы прибывают посетители. Все они прибывают на территорию бизнес-центра на метро. Всего есть `m` станций метро, для каждой станции известны её координаты. Также известно количество самокатов `s_j` около каждой станции в момент начала рабочего дня.

Когда посетитель прибывает на станцию метро, он смотрит, достаточно ли близко находится офис нужной компании, и, если офис находится достаточно близко для того, чтобы использовать самокат, и если на этой станции ещё остались самокаты, то посетитель арендует самокат и едет к своему офису (самокат после этого остаётся у офиса), иначе он едет на такси.

Анализ ситуации с распределением самокатов в течение дня — сложная и важная задача: если её не решать, то может оказаться, что в требуемом месте нет самокатов, а около какого-то офиса количество самокатов настолько велико, что мешает проходу посетителей или проезду автомобилей...

Аналитики МТС-Юрент хотят промоделировать следующую пиковую ситуацию:

- Все самокаты от станций метро, которые находятся хотя бы от одной компании `i` на расстоянии, не большем, чем `r_i`, разобраны (то есть остались все самокаты у станций, которые находятся достаточно далеко от каждой из компаний, и только они, остальные самокаты находятся у офисов).
- Все самокаты, которые приехали от какой-либо станции метро к какому-то офису (как и те, которые стояли у офиса перед началом рабочего дня), так и остаются около этого офиса.
- Итоговое количество самокатов у офиса МТС Финтех не меньше, чем количество самокатов у офиса любой другой компании в бизнес-центре.

Ваша задача — определить, возможна ли такая ситуация в каждом конкретном сценарии.

### Формат ввода

Первая строка входных данных содержит одно целое число `T` (`1 ≤ T ≤ 100`) — количество сценариев. Далее следуют описания самих сценариев.

Первая строка описания сценария содержит два целых числа `n` и `m` — количество офисов и количество станций метро, соответственно (`1 ≤ n ≤ 100`, `1 ≤ m ≤ 400`).

`i`-я из последующих `n` строк задаёт офис `i`-й компании и содержит четыре целых числа `x_i`, `y_i`, `e_i` и `r_i` — координаты офиса, количество самокатов, которые уже стоят у офиса к моменту начала рабочего дня, и верхнюю границу расстояния от станции метро до этого офиса, для которого посетитель соответствующей компании едет в офис на самокате, соответственно (`0 ≤ x_i, y_i, r_i ≤ 10^4`, `1 ≤ e_i ≤ 10^4`).

Офис МТС Финтех соответствует первой из этих строк.

`j`-я из последующих `m` строк задаёт `j`-ю станцию метро и содержит три целых числа `x_j`, `y_j` и `s_j` — координаты станции и количество самокатов около этой станции (`0 ≤ x_j, y_j ≤ 10^4`, `1 ≤ s_j ≤ 10^4`).

Гарантируется, что сумма `n` по всем сценариям не превосходит `5000`, сумма `m` по всем сценариям не превосходит `25000`.

### Формат вывода

Если ситуация, при которой все самокаты у станций метро (кроме самокатов у станций, которые не находятся ни от одной компании на рекомендуемом этой компанией для поездки на самокате расстоянии) разобраны, и при этом количество самокатов у офиса МТС Финтех не меньше, чем количество самокатов у офиса любой другой компании, возможна, выведите для соответствующего сценария `1`.

Иначе выведите `0`.

### Пример

**Стандартный ввод**
```text
2
3 2
1 1 1 10
11 1 1 10
21 1 1 10
6 1 2
16 1 4
3 2
0 0 1 2
2 0 1 2
4 0 1 2
1 0 2
3 0 5
```

**Стандартный вывод**
```text
1
0
```

## Задача I. Ориентиры

- Имя входного файла: стандартный ввод
- Имя выходного файла: стандартный вывод
- Ограничение по времени: 5 секунд
- Ограничение по памяти: 1024 мегабайта

Желтокаменский заповедник занимает очень большую территорию. Так что туристы перемещаются над заповедником на электровертолётах.

Для упрощения полётов на территории заповедника установлены точечные радиопередатчики, из которых составляются так называемые ориентиры. Каждый ориентир представляет собой фиксированное множество из `p` точечных передатчиков.

Система распознавания устроена следующим образом: бортовой компьютер фиксирует очередной ориентир. Если два ориентира можно перевести друг в друга конечным числом действий следующего вида:

- параллельный перенос;
- поворот вокруг начала координат;
- масштабирование с центром в начале координат (то есть все координаты увеличиваются в одно и то же число раз);

то ориентиры считаются совпадающими.

Вам дан лог системы распознавания с одного из вертолётов, содержащий список зафиксированных ориентиров. Ваша задача — определить, сколько для каждого ориентира в этом списке есть ориентиров, с ними совпадающих.

### Формат ввода

Первая строка входных данных содержит два целых числа `n` и `p` — количество ориентиров в логе и количество передатчиков в каждом ориентире, соответственно (`1 ≤ n, p ≤ 1000`).

`j`-я из последующих `n` строк содержит по `2p` целых чисел. `i`-я из `p` пар задаёт координаты `x_{ij}`, `y_{ij}` очередного передатчика в ориентире (`-1000 ≤ x_{ij}, y_{ij} ≤ 1000`).

Гарантируется, что для каждого ориентира все `p` передатчиков находятся в попарно различных точках.

### Формат вывода

Выведите `n` строк.

`i`-я из этих строк задаёт ответ для `i`-го ориентира и начинается с числа `s_i` — количества ориентиров, совпадающих с `i`-м (порядок ориентиров совпадает с порядком ориентиров во входном файле), а далее, если `s_i ≠ 0`, в той же строке выведите `s_i` целых чисел от `1` до `n`, не равных `i` — номера ориентиров, совпадающих с `i`-м, отсортированные по возрастанию.

### Пример

**Стандартный ввод**
```text
4 3
1 3 0 0 4 4
0 0 2 2 0 2
0 0 0 -4 4 0
-1 -1 1 -1 1 1
```

**Стандартный вывод**
```text
0
2 3 4
2 2 4
2 2 3
```
