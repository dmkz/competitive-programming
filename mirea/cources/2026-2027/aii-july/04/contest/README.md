# Разбор задач

## A. Сумма чётных и нечётных на отрезке

<details>
<summary>Текстовый разбор</summary>

Будем хранить две префиксные суммы: $s_i^{(0)}$ — сумму чётных элементов среди $a_1, \ldots, a_i$, а $s_i^{(1)}$ — сумму нечётных элементов на том же префиксе. Тогда ответ на запрос $[l, r]$ считается так же, как и обычная сумма на отрезке: для чётных это $s_r^{(0)} - s_{l-1}^{(0)}$, а для нечётных — $s_r^{(1)} - s_{l-1}^{(1)}$. Значит, после одного линейного предподсчёта каждый запрос обрабатывается за $O(1)$. Асимптотика: $O(n + q)$ по времени и $O(n)$ по памяти.

</details>

<details>
<summary>Решение на C++</summary>

```cpp
#include <bits/stdc++.h>
using namespace std;
void solve() {
    // читаем массив
    int n, q;
    cin >> n >> q;
    vector<int> a(n);
    for (int i = 0; i < n; i++)
        cin >> a[i];
    // строим префиксные суммы: отдельно для чётных и отдельно для нечётных чисел
    vector<int64_t> s0(n+1, 0), s1(n+1, 0);
    for (int i = 0; i < n; i++) {
        s0[i+1] = s0[i], s1[i+1] = s1[i];
        if (a[i] % 2 == 0) s0[i+1] += a[i];
        else s1[i+1] += a[i];
    }
    // отвечаем на запросы:
    while (q --> 0) {
        int l, r;
        cin >> l >> r;
        cout << (s0[r] - s0[l-1]) << " " << (s1[r] - s1[l-1]) << '\n';
    }
}
main() {
    int tt; cin >> tt;
    while (tt --> 0) solve();
}
```

</details>

<details>
<summary>Решение на Python3</summary>

```python
        q -= 1

    solve()
    tt -= 1
```

## B. Префиксы и суффиксы

<details>
<summary>Текстовый разбор</summary>

Для каждого $i$ заранее посчитаем два значения: $p_i$ — количество различных чисел на префиксе $a_1, \ldots, a_i$, и $s_i$ — количество различных чисел на суффиксе $a_i, \ldots, a_n$. Массив $p$ строится слева направо: вставляем очередной элемент в множество и сохраняем его размер. Массив $s$ строится справа налево точно так же. После этого запрос с буквой $p$ отвечает числом $p_i$, а запрос с буквой $s$ — числом $s_i$.

В текущем решении на C++ используется упорядоченное множество, поэтому предподсчёт стоит $O(n \log(n))$. В Python3 множество хеш-табличное, поэтому средняя оценка предподсчёта там $O(n)$. Ответы на все запросы после этого занимают $O(q)$.

</details>

<details>
<summary>Решение на C++</summary>

```cpp
#include <bits/stdc++.h>
using namespace std;
void solve() {
    // читаем массив
    int n, q;
    cin >> n >> q;
    vector<int> a(n);
    for (int i = 0; i < n; i++)
        cin >> a[i];
    // считаем и запоминаем количество уникальных элементов на каждом префиксе:
    set<int> numbers;
    vector<int> p(n);
    for (int i = 0; i < n; i++) {
        numbers.insert(a[i]);
        p[i] = numbers.size();
    }
    // аналогично для каждого суффикса:
    numbers.clear();
    vector<int> s(n);
    for (int i = n-1; i >= 0; i--) {
        numbers.insert(a[i]);
        s[i] = numbers.size();
    }
    // отвечаем на запросы:
    while (q --> 0) {
        char c; int i;
        cin >> c >> i;
        i--;
        cout << (c == 'p' ? p[i] : s[i]) << '\n';
    }
}
main() {
    int tt; cin >> tt;
    while (tt --> 0) solve();
}
```

</details>

<summary>Решение на Python3</summary>

## C. Минимальная правая граница

<details>
<summary>Текстовый разбор</summary>

Построим префиксные суммы $s_i$ поверх массива $a_i$. Тогда сумма на отрезке $[l, r]$ равна $s_r - s_{l-1}$. В запросе нужно найти минимальный индекс $r \ge l$, для которого $s_r - s_{l-1} \ge x$. Переносим известное направо и получаем условие $s_r \ge x + s_{l-1}$.

Так как элементы массива неотрицательны, префиксные суммы не убывают. Значит, по массиву $s$ можно запустить бинарный поиск и найти первый индекс, где значение стало не меньше $x + s_{l-1}$. Именно это и делают вызовы `std::lower_bound` в C++ и `bisect_left` в Python3. Если подходящего индекса нет, поиск вернёт конец массива префиксных сумм, то есть ответом станет $n + 1$. Асимптотика: $O(n + q \log(n))$.

</details>

<details>
<summary>Решение на C++</summary>

```cpp
#include <bits/stdc++.h>
using namespace std;
void solve() {
    // читаем массив
    int n, q;
    cin >> n >> q;
    vector<int> a(n);
    for (int i = 0; i < n; i++)
        cin >> a[i];
    // считаем префикс-суммы:
    vector<int64_t> s(n+1);
    for (int i = 0; i < n; i++)
        s[i+1] = s[i] + a[i];
    // отвечаем на запросы:
    while (q --> 0) {
        int l; int64_t x;
        cin >> l >> x;
        // s[r] - s[l-1] >= x ==> s[r] >= x + s[l-1]
        auto iter = lower_bound(s.begin()+l, s.end(), x + s[l-1]);
        cout << int(iter - s.begin()) << '\n';
    }
}
main() {
    int tt; cin >> tt;
    while (tt --> 0) solve();
}
```

</details>

<details>
<summary>Решение на Python3</summary>

```python
from bisect import bisect_left


def solve():
    # читаем массив
    n, q = map(int, input().split())
    a = list(map(int, input().split()))
    # считаем префикс-суммы:
    s = [0] * (n + 1)
    for i in range(n):
        s[i + 1] = s[i] + a[i]
    # отвечаем на запросы:
    while q > 0:
        l, x = map(int, input().split())
        # s[r] - s[l-1] >= x ==> s[r] >= x + s[l-1]
        print(bisect_left(s, x + s[l - 1], l))
        q -= 1

while tt > 0:
    solve()
    tt -= 1
```

</details>

<details>
<summary>Разбор решения на C++</summary>

Самый важный фрагмент здесь такой:

```cpp
auto iter = lower_bound(s.begin()+l, s.end(), x + s[l-1]);
cout << int(iter - s.begin()) << '\n';
```

Вызов `std::lower_bound` ищет первый элемент не меньше заданного значения. Мы начинаем поиск с позиции $l$, потому что правая граница не может быть меньше левой. Ищем число $x + s_{l-1}$, так как именно к такому виду мы привели условие $s_r - s_{l-1} \ge x$. После этого остаётся перевести найденный итератор обратно в индекс.

</details>

<details>
<summary>Разбор решения на Python3</summary>

Ключевой фрагмент здесь полностью повторяет идею C++-решения:

```python
print(bisect_left(s, x + s[l - 1], l))
```

Функция `bisect_left` возвращает первую позицию, где можно поставить значение так, чтобы порядок не нарушился. Так как массив префиксных сумм неубывающий, это и есть минимальный индекс $r$, для которого выполнено $s_r \ge x + s_{l-1}$. Третий аргумент равен $l$, поэтому поиск сразу ограничен индексами $r \ge l$.

</details>

## D. Количество отрезков с заданной суммой

<details>
<summary>Текстовый разбор</summary>

Построим знакочередующиеся суффиксные суммы $s_i$ справа налево по правилу $s_i = a_i - s_{i+1}$ и положим $s_{n+1} = 0$. Тогда для полуинтервала $[i, j)$ его знакочередующаяся сумма выражается очень коротко: если $i$ и $j$ одной чётности, то она равна $s_i - s_j$, а если разной чётности, то $s_i + s_j$.

Теперь будем фиксировать левую границу $i$ и перебирать её справа налево. Все подходящие правые границы $j$ уже обработаны, поэтому достаточно знать, сколько раз справа встречалось каждое значение $s_j$ отдельно для двух чётностей. Если чётности совпадают, нужно количество значений $s_j = s_i - x$. Если чётности различаются, нужно количество значений $s_j = x - s_i$. Значит, достаточно вести два словаря: для чётных и для нечётных индексов. В текущем C++-коде используется упорядоченный словарь, поэтому оценка равна $O(n \log(n))$. В Python3 используется хеш-словарь, поэтому средняя оценка там $O(n)$.

</details>

<details>
<summary>Решение на C++</summary>

```cpp
#include <bits/stdc++.h>
using namespace std;
void solve() {
    // читаем массив
    int n; int64_t x;
    cin >> n >> x;
    vector<int> a(n);
    for (int i = 0; i < n; i++)
        cin >> a[i];
    // считаем знакочередующиеся суффиксные суммы:
    vector<int64_t> s(n+1);
    for (int i = n-1; i >= 0; i--)
        s[i] = a[i] - s[i+1];
    // проходимся справа налево и считаем ответ:
    map<int64_t, int> count[2] = {};
    int64_t cnt{};
    for (int i = n; i >= 0; i--) {
        // Если чётности совпадают, то: s[i] - s[j] = x => s[i] - x = s[j]
        // Иначе: s[i] + s[j] = x => x - s[i] = s[j]
        cnt += count[i % 2][s[i] - x] + count[1 - i % 2][x - s[i]];
        count[i % 2][s[i]]++;
    }
    cout << cnt << '\n';
}
main() {
    int tt; cin >> tt;
    while (tt --> 0) solve();
}
```

</details>


## E. Двойное переупорядочивание

<details>
<summary>Текстовый разбор</summary>

Пусть $c_i$ — итоговый коэффициент, с которым число на позиции $i$ входит в общую сумму после всех запросов. Тогда вся задача сводится к двум шагам. Сначала нужно посчитать коэффициенты $c_i$. Это удобно сделать разностным массивом: запрос $+\ [l, r]$ добавляет $+1$ на отрезок $[l, r]$, а запрос $-\ [l, r]$ добавляет $-1$ на этом же отрезке. После одного префиксного прохода получаем все коэффициенты.

Дальше нужно не считать сумму напрямую, а правильно расставить числа массива. Если хотим максимум, то наибольшим коэффициентам надо выдать наибольшие элементы. Если хотим минимум, то наибольшим коэффициентам надо выдать наименьшие элементы. Поэтому сортируем массив $a$, сортируем пары $(c_i, i)$ по коэффициенту, а затем для максимума сопоставляем оба отсортированных списка в одном порядке, а для минимума — в противоположном. Асимптотика: $O(q + n \log(n))$ по времени и $O(n)$ по памяти.

</details>

<details>
<summary>Решение на C++</summary>

```cpp
#include <bits/stdc++.h>
#include <cassert>
using namespace std;
void solve() {
    // читаем массив
    int n, q;
    cin >> n >> q;
    vector<int> a(n);
    for (int i = 0; i < n; i++)
        cin >> a[i];
    // считываем и откладываем запросы в разностный массив:
    vector<int> coeff(n+1);
    while (q --> 0) {
        char c; int l, r;
        cin >> c >> l >> r;
        l--, r--;
        if (c == '+') {
            coeff[l]++;
            coeff[r+1]--;
        } else {
            assert(c == '-');
            coeff[l]--;
            coeff[r+1]++;
        }
    }
    // считаем коэффициенты:
    for (int i = 1; i <= n; i++)
        coeff[i] += coeff[i-1];
    // создаём пары и сортируем их по возрастанию коэффициентов:
    vector<pair<int,int>> pairs;
    for (int i = 0; i < n; i++)
        pairs.emplace_back(coeff[i], i);
    sort(pairs.begin(), pairs.end());
    // сортируем значения массива по возрастанию и сопоставляем индексы:
    sort(a.begin(), a.end());
    vector<int> Max(n), Min(n);
    for (int i = 0; i < n; i++) {
        auto [_, maxI] = pairs[i];
        auto [_, minI] = pairs[n-i-1];
        Max[maxI] = a[i];
        Min[minI] = a[i];
    }
    // вывод ответа:
    for (auto &it : Max)
        cout << it << ' ';
    cout << '\n';
    for (auto &it : Min)
        cout << it << ' ';
    cout << '\n';
}
main() {
    int tt; cin >> tt;
    while (tt --> 0) solve();
}
```

</details>

```python
def solve():
    # читаем массив
    n, q = map(int, input().split())
    a = list(map(int, input().split()))
    # считываем и откладываем запросы в разностный массив:
    coeff = [0] * (n + 1)
    while q > 0:
        c, l, r = input().split()
        l = int(l) - 1
        r = int(r) - 1
        if c == '+':
            coeff[l] += 1
            coeff[r + 1] -= 1
        else:
            coeff[l] -= 1
            coeff[r + 1] += 1
        q -= 1
    # считаем коэффициенты:
    for i in range(1, n + 1):
        coeff[i] += coeff[i - 1]
    # создаём пары и сортируем их по возрастанию коэффициентов:
    pairs = []
    for i in range(n):
        pairs.append((coeff[i], i))
    pairs.sort()
    # сортируем значения массива по возрастанию и сопоставляем индексы:
    a.sort()
    max_answer = [0] * n
    min_answer = [0] * n
    for i in range(n):
        max_i = pairs[i][1]
        min_i = pairs[n - i - 1][1]
        max_answer[max_i] = a[i]
        min_answer[min_i] = a[i]
    # вывод ответа:
    print(*max_answer)
    print(*min_answer)


tt = int(input())
while tt > 0:
    solve()
    tt -= 1
```

<summary>Разбор решения на C++</summary>

Здесь полезно отдельно посмотреть на два фрагмента. Первый отвечает за разностный массив:

```cpp

} else {
    coeff[l]--;
```

После префиксного суммирования этот массив превращается в набор коэффициентов $c_i$. Именно столько раз каждая позиция участвует в итоговой сумме с учётом знака.

Второй ключевой фрагмент — это сопоставление отсортированных коэффициентов и значений:


Для максимума мы идём по коэффициентам слева направо и по значениям слева направо, то есть маленькие значения получают маленькие коэффициенты, а большие — большие. Для минимума коэффициенты берутся в обратном порядке, поэтому те же маленькие значения уходят уже к большим коэффициентам.

Первая пара строк строит максимум, потому что коэффициенты и значения идут в одном порядке. Вторая пара строк строит минимум, потому что коэффициенты берутся в обратном порядке.
