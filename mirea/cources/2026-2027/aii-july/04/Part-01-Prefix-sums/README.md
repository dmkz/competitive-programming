## 1. Префиксные суммы

### Что это такое?

Пусть дан массив $a_0, a_1, \ldots, a_{n-1}$. Будем хранить массив $s$, где $s_0 = 0$ и для каждого $i \ge 1$ значение $s_i$ равно сумме первых $i$ элементов массива. То есть:

$$
s_i = \sum_{j=0}^{i-1} a_j \quad \text{при } 1 \le i \le n.
$$

Эквивалентная запись:

$$
s_{i+1} = \sum_{j=0}^{i} a_j \quad \text{при } 0 \le i < n.
$$

В рекуррентном виде (через уже вычисленные суммы):

$$
s_i = s_{i-1} + a_{i-1} \quad \text{при } 1 \le i \le n.
$$

Первые значения выглядят так:

- $s_1 = a_0$;
- $s_2 = a_0 + a_1$;
- $s_3 = a_0 + a_1 + a_2$;
- и так далее.

Тогда сумма на отрезке $[l, r]$ в нулевой индексации равна

$$
a_l + a_{l+1} + \ldots + a_r = s_{r+1} - s_l.
$$

Если вход использует единичную индексацию и элементы обозначены как $a_1, a_2, \ldots, a_n$, то та же идея записывается со смещением индексов:

$$
a_l + a_{l+1} + \ldots + a_r = s_r - s_{l-1}.
$$

> [!NOTE]
> В коде такой массив префиксных сумм часто называют `pref`.

### Построение префиксных сумм в C++

```cpp
int n;
cin >> n;
vector<long long> a(n);
for (int i = 0; i < n; i++) {
    cin >> a[i];
}

vector<long long> pref(n + 1, 0);
for (int i = 0; i < n; i++) {
    pref[i + 1] = pref[i] + a[i];
}
```

### Построение префиксных сумм в Python3

```python
n = int(input())
a = list(map(int, input().split()))

pref = [0] * (n + 1)
for i in range(n):
    pref[i + 1] = pref[i] + a[i]
```

### Ответ на запросы суммы на отрезке

```cpp
int q;
cin >> q;
while (q--) {
    int l, r;
    cin >> l >> r;
    --l;
    --r;
    cout << pref[r + 1] - pref[l] << '\n';
}
```

Полная программа для этого базового примера: [C++](examples/example-02.cpp), [Python3](examples/example-02.py).

Таким образом, вместо того чтобы каждый раз считать сумму в цикле, мы заранее считаем префиксные суммы, а затем отвечаем на запросы. Сложность:

- построение массива префиксных сумм занимает $O(n)$;
- каждый запрос обрабатывается за $O(1)$;
- суммарно получается $O(n + q)$ вместо $O(nq)$.

> [!IMPORTANT]
> Для префиксных сумм почти всегда нужен тип `long long` в C++: даже если сами элементы массива помещаются в `int`, их суммарное значение может не поместиться.

## 2. Префиксные суммы есть в C++, но их никто не использует, и вот почему

> [!NOTE]
> Это бонусный раздел. `std::partial_sum` есть давно, а `std::inclusive_scan` и `std::exclusive_scan` появились в C++17.

В стандартной библиотеке C++ действительно есть готовые алгоритмы для префиксных сумм. Но в олимпиадном коде их используют редко: обычный цикл короче, прозрачнее и даёт меньше сюрпризов с типами и соглашениями об индексации.

<details>
<summary><strong>Бонус: std::partial_sum</strong></summary>

Самый старый вариант — `std::partial_sum` из `<numeric>`.

```cpp
vector<long long> pref(n + 1, 0);
vector<long long> a64(a.begin(), a.end());
partial_sum(a64.begin(), a64.end(), pref.begin() + 1);
```

Такой код корректно строит массив

$pref_i = a_0 + a_1 + \ldots + a_{i-1}$.

**Внимание.** У `std::partial_sum` есть неприятное ограничение интерфейса: тип накопителя берётся не из третьего параметра, а из типа элементов входного диапазона. Поэтому такой код опасен:

```cpp
vector<int> a = {1000000000, 1000000000, 1000000000};
vector<long long> pref(a.size());
partial_sum(a.begin(), a.end(), pref.begin());
```

**Внимание.** Здесь всё равно возможно переполнение, потому что накопление идёт в `int`, а в `long long` результат попадает уже потом.

Полная программа: [C++](examples/example-03.cpp), [Python3](examples/example-03.py).

</details>

<details>
<summary><strong>Бонус: std::inclusive_scan</strong></summary>

У `std::inclusive_scan` есть перегрузка, где тип накопителя можно задать через последний параметр.

```cpp
vector<long long> pref(n + 1, 0);
inclusive_scan(a.begin(), a.end(), pref.begin() + 1, plus<long long>(), 0LL);
```

После этого:

- `pref[0] = 0`;
- `pref[1] = a[0]`;
- `pref[2] = a[0] + a[1]`;
- и так далее.

**Практика.** Здесь входной массив вполне может иметь тип `vector<int>`. Главное, чтобы последний параметр был `0LL`, тогда накопление пойдёт в `long long`.

Полная программа: [C++](examples/example-04.cpp), [Python3](examples/example-04.py).

</details>

<details>
<summary><strong>Бонус: std::exclusive_scan</strong></summary>

У `std::exclusive_scan` такой же плюс: тип накопителя тоже можно задать через последний параметр.

```cpp
vector<int> extended = a;
extended.push_back(0);

vector<long long> pref(n + 1);
exclusive_scan(extended.begin(), extended.end(), pref.begin(), 0LL);
```

Этот приём даёт ровно нашу привычную нумерацию:

- `pref[0] = 0`;
- `pref[1] = a[0]`;
- `pref[2] = a[0] + a[1]`;
- `...`;
- `pref[n] = a[0] + a[1] + \ldots + a[n-1]`.

**Практика.** И здесь входные данные могут быть `int`. Достаточно передать `0LL` последним параметром, чтобы накопление выполнялось в `long long`.

Полная программа: [C++](examples/example-05.cpp), [Python3](examples/example-05.py).

</details>

> [!IMPORTANT]
> Вот почему в олимпиадной практике эти алгоритмы пишут редко: у `std::partial_sum` есть ловушка с типом накопителя, `std::inclusive_scan` и `std::exclusive_scan` требуют помнить перегрузки и стандарт C++17, а обычный цикл короче, понятнее и почти всегда читается лучше.

## 3. Префиксы не только по значениям, но и по признакам

Префиксные суммы можно строить не только по самому массиву $a$. Иногда нас интересуют:

- количество чётных элементов на отрезке;
- количество элементов больше нуля;
- сумма только нечётных элементов;
- количество букв $a$ в подстроке;
- количество пар равных соседних элементов на отрезке.

Тогда мы строим суммы не по самим значениям $a_i$, а по некоторому вкладу, который каждый элемент даёт в ответ.

<details>
<summary><strong>Пример: сумма положительных чисел на отрезке</strong></summary>

Пусть $b_i = a_i$, если $a_i > 0$, и $b_i = 0$ иначе.

Пусть $s_i$ — префиксные суммы массива $b$. Тогда сумма положительных чисел на отрезке $[l, r]$ равна $s_{r+1} - s_l$.

</details>

<details>
<summary><strong>C++: сумма положительных чисел на отрезке</strong></summary>

```cpp
int n, q;
cin >> n >> q;
vector<long long> a(n);
for (int i = 0; i < n; i++) {
    cin >> a[i];
}

vector<long long> pref_pos(n + 1, 0);

for (int i = 0; i < n; i++) {
    pref_pos[i + 1] = pref_pos[i];
    if (a[i] > 0) {
        pref_pos[i + 1] += a[i];
    }
}

while (q--) {
    int l, r;
    cin >> l >> r;
    --l;
    --r;
    cout << pref_pos[r + 1] - pref_pos[l] << '\n';
}
```

</details>

<details>
<summary><strong>Python3: сумма положительных чисел на отрезке</strong></summary>

```python
n, q = map(int, input().split())
a = list(map(int, input().split()))

pref_pos = [0] * (n + 1)
for i in range(n):
    pref_pos[i + 1] = pref_pos[i]
    if a[i] > 0:
        pref_pos[i + 1] += a[i]

for _ in range(q):
    l, r = map(int, input().split())
    l -= 1
    r -= 1
    print(pref_pos[r + 1] - pref_pos[l])
```

</details>

<details>
<summary><strong>Пример ввода и вывода</strong></summary>

Для ввода

```text
5 4
2 -7 4 0 5
1 5
2 4
4 5
3 3
```

получим вывод

```text
11
4
5
4
```

</details>

Файлы примера: [C++](examples/example-01.cpp), [Python3](examples/example-01.py).

<details>
<summary><strong>Пример: количество каждой буквы от 'a' до 'z' на отрезке</strong></summary>

Пусть дана строка длины $n$. Самая прямолинейная идея: завести $26$ префиксных массивов $p^{(0)}, p^{(1)}, \ldots, p^{(25)}$, где каждый массив имеет длину $n+1$, а массив $p^{(c)}$ отвечает только за свою букву.

Для выбранной буквы $c$ рекуррентная формула выглядит так: $p^{(c)}_{i+1} = p^{(c)}_i + [s_i = c]$.

Тогда количество буквы $c$ на отрезке $[l, r]$ равно $p^{(c)}_{r+1} - p^{(c)}_l$.

Если запрос просит количества всех букв сразу, такая реализация делает $52$ обращения к памяти: по два чтения на каждую из $26$ букв.

На практике обычно быстрее хранить один префиксный массив длины $n+1$, где в каждой позиции лежит массив из $26$ счётчиков:

```cpp
string s;
cin >> s;
int n = (int)s.size();

vector<array<int, 26>> pref(n + 1);
pref[0].fill(0);
for (int i = 0; i < n; i++) {
    pref[i + 1] = pref[i];
    pref[i + 1][s[i] - 'a']++;
}

int l, r;
cin >> l >> r;
--l;
--r;
for (int c = 0; c < 26; c++) {
    cout << pref[r + 1][c] - pref[l][c] << " \n"[c + 1 == 26];
}
```

**Практика.** Идея та же самая, но доступ к памяти получается гораздо лучше. На практике такой вариант нередко работает примерно в $5$ раз быстрее: вместо $52$ разрозненных чтений мы работаем с двумя компактными объектами `pref[l]` и `pref[r + 1]`, а типичная кеш-линия имеет размер $64$ байта, то есть $512$ бит. Если вместо `vector<int>` использовать `array<int, 26>`, лишней косвенности тоже не будет, и это обычно ещё немного ускоряет код.

Полная программа: [C++](examples/example-09.cpp), [Python3](examples/example-09.py).

</details>

<details>
<summary><strong>Бонусный пример: функция Эйлера на произведении чисел отрезка</strong></summary>

**Бонус.** Это именно бонусный пример.

Если все значения массива не превосходят $300$, то простых чисел до $300$ всего $62$. Пронумеруем их как $p_0, p_1, \ldots, p_{61}$.

Теперь можно хранить не одно число на позицию, а вектор показателей простых. Пусть $b_{i, j}$ — степень вхождения $j$-го простого числа в разложение числа $a_i$.

Тогда для каждого фиксированного $j$ мы можем построить префиксные суммы по столбцу $b_{0, j}, b_{1, j}, \ldots, b_{n-1, j}$. На отрезке $[l, r]$ получим суммарную степень $K_j = b_{l, j} + b_{l+1, j} + \ldots + b_{r, j}$.

Значит, произведение всех чисел на отрезке раскладывается как $a_l a_{l+1} \cdots a_r = \prod_{j=0}^{61} p_j^{K_j}$.

После этого можно, например, посчитать функцию Эйлера от этого произведения: $\varphi\left(\prod_{j=0}^{61} p_j^{K_j}\right) = \prod_{j : K_j > 0} \left(p_j^{K_j} - p_j^{K_j - 1}\right)$.

Если хочется записывать множитель в виде $p^{k+1} - p^k$, то здесь просто $k = K_j - 1$.

**Идея.** Это хороший пример того, что иногда вклад элемента — не одно число, а целый небольшой вектор признаков.

Полная программа с ответом по модулю $10^9+7$: [C++](examples/example-10.cpp), [Python3](examples/example-10.py).

</details>

> [!NOTE]
> Это очень важный паттерн: если запрос спрашивает не про сами элементы, а про их вклад по какому-то признаку, часто достаточно заменить исходный массив на массив вкладов и дальше снова использовать префиксные суммы.

## 4. Какие функции можно считать на отрезке?

Идея префиксов работает не только для обычной суммы. Мы хотим заранее считать значения функции на префиксах, а потом получать ответ на отрезке через два префикса.

Для сумм и количеств это обычно записывают так:

$$
f(l, r) = f(0, r) - f(0, l - 1).
$$

Общая часть $[0, l-1]$ входит в оба префикса и взаимоуничтожается.

Если использовать нулевую индексацию и обозначать через $f_i$ значение на первых $i$ элементах, то формула выглядит так:

$$
f(l, r) = f_{r+1} - f_l.
$$

В более общем случае вместо вычитания нужна операция, которая умеет убрать общий префикс. Значит, техника подходит не для любой функции, а только для тех, где общая часть действительно сокращается.

Подходят, например:

- обычная сумма;
- количество элементов с некоторым свойством;
- сумма вкладов элементов по некоторому признаку;
- `XOR` (взаимоисключающее ИЛИ);
- суммы и произведения по модулю $m$.

Обычно не подходят:

- минимум;
- максимум;
- побитовое `OR` (побитовое ИЛИ);
- побитовое `AND` (побитовое И);
- `gcd` (наибольший общий делитель).

> [!WARNING]
> По двум префиксам нельзя просто убрать общую часть для таких функций.

### Пример: XOR на отрезке

Пусть

$$
x_i = a_0 \oplus a_1 \oplus \ldots \oplus a_{i-1}.
$$

Тогда XOR на отрезке $[l, r]$ равен

$$
a_l \oplus a_{l+1} \oplus \ldots \oplus a_r = x_{r+1} \oplus x_l.
$$

Это работает, потому что общая часть встречается два раза, а для XOR верно $u \oplus u = 0$.

<details>
<summary><strong>C++: XOR на отрезке</strong></summary>

```cpp
int n;
cin >> n;
vector<int> a(n);
for (int i = 0; i < n; i++) {
    cin >> a[i];
}

vector<int> pref_xor(n + 1, 0);
for (int i = 0; i < n; i++) {
    pref_xor[i + 1] = pref_xor[i] ^ a[i];
}

int q;
cin >> q;
while (q--) {
    int l, r;
    cin >> l >> r;
    --l;
    --r;
    cout << (pref_xor[r + 1] ^ pref_xor[l]) << '\n';
}
```

</details>

<details>
<summary><strong>Python3: XOR на отрезке</strong></summary>

```python
n = int(input())
a = list(map(int, input().split()))

pref_xor = [0] * (n + 1)
for i in range(n):
    pref_xor[i + 1] = pref_xor[i] ^ a[i]

q = int(input())
for _ in range(q):
    l, r = map(int, input().split())
    l -= 1
    r -= 1
    print(pref_xor[r + 1] ^ pref_xor[l])
```

</details>

> [!NOTE]
> Если общую часть можно убрать обратной операцией или самоуничтожением, как у XOR, то префиксный подход сработает.

## 5. Иногда удобнее использовать суффикс-суммы

Префиксы не обязательны. Иногда формулу проще писать справа налево.

У суффиксных сумм есть два удобных способа индексации:

- индексация позицией начала суффикса: ${suff}_i$;
- индексация длиной суффикса: $s_k$.

Оба способа могут пригодиться в задачах.

### Индексация позицией начала суффикса

Для обычных суффиксных сумм задаём: ${suff}_n = 0$.

Для $i = n - 1, n - 2, \ldots, 0$ выполняется рекурсия ${suff}_i = {suff}_{i+1} + a_i$.

Тогда сумма на отрезке $[l, r]$ равна $a_l + a_{l+1} + \ldots + a_r = {suff}_l - {suff}_{r+1}$.

```cpp
vector<long long> suff(n + 1, 0);
for (int i = n - 1; i >= 0; i--) {
    suff[i] = suff[i + 1] + a[i];
}
```

Полная программа: [C++](examples/example-06.cpp), [Python3](examples/example-06.py).

> [!TIP]
> Этот способ удобен, когда состояние естественно выражается через позицию начала хвоста.

### Индексация длиной суффикса

Иногда удобнее считать не суффикс, начинающийся в позиции $i$, а суффикс длины $k$.

Пусть $s_k$ — сумма последних $k$ элементов массива. Универсальный олимпиадный способ построения:

```cpp
vector<long long> s = {0};
for (int i = n - 1; i >= 0; i--) {
    s.push_back(s.back() + a[i]);
}
```

Полная программа: [C++](examples/example-07.cpp), [Python3](examples/example-07.py).

> [!TIP]
> У этого способа есть важный плюс: внутри цикла не нужно думать про индексы результирующего массива. Меняется только порядок обхода.

Например, для префиксных сумм цикл такой:

```cpp
vector<long long> s = {0};
for (int i = 0; i < n; i++) {
    s.push_back(s.back() + a[i]);
}
```

Полная программа: [C++](examples/example-07.cpp), [Python3](examples/example-07.py).

> [!IMPORTANT]
> Содержимое цикла и интерфейс построения остаются общими: мы всегда делаем `push_back(s.back() + a[i])`, а меняется только направление обхода.

### Нестандартные суффиксные рекурсии

Иногда через суффиксную рекурсию удобнее записывать и не совсем обычные величины. Например, если нужен знакочередующийся хвост

$$
a_i - a_{i+1} + a_{i+2} - a_{i+3} + \ldots,
$$

то естественная формула выглядит так: ${alt}_i = a_i - {alt}_{i+1}$.

> [!NOTE]
> Здесь уже естественнее именно индексация позицией начала, а не длиной.

> [!WARNING]
> Но у суффиксного мышления есть минус: в таких формулах обычно приходится думать справа налево, а это для многих заметно тяжелее, чем обычные префиксы.

## 6. Восстановление оригинального массива, зная префиксные суммы

Если известны все префиксные суммы, исходный массив восстанавливается единственным образом.

Причина простая: два соседних префикса отличаются ровно на один элемент. Значит,

$$
a_i = s_{i+1} - s_i.
$$

В единичной индексации это та же самая формула в виде $a_i = s_i - s_{i-1}$.

> [!IMPORTANT]
> Другого варианта восстановления нет, потому что каждое значение $a_i$ уже однозначно задаётся разностью двух соседних префиксов.

```cpp
int n;
cin >> n;
vector<long long> pref(n + 1);
for (int i = 0; i <= n; i++) {
    cin >> pref[i];
}

vector<long long> a(n);
```
