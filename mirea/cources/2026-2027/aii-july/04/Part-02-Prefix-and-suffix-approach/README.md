## 7. Что такое префиксный подход

Иногда ответ на задачу нельзя получить одной формулой для суммы на отрезке. Но всё равно можно перейти к массиву префиксных состояний и выразить условие на подотрезок через связь между двумя префиксами.

Это и есть префиксный подход.

### Общая схема

1. Для каждой позиции строим некоторое префиксное состояние $p_i$.
2. Рассматриваем отрезок $[l, r]$ не сам по себе, а через связь между состояниями $p_l$ и $p_{r+1}$.
3. Получаем задачу не про элементы отрезка напрямую, а про пары префиксов.
4. После этого часто подключаются map, set, массивы частот или бинарный поиск.

> [!IMPORTANT]
> Если в задаче встречается условие вида «сумма на подотрезке равна ...», «баланс равен ...», «количество двух типов объектов совпадает ...», очень часто нужно не пересчитывать отрезки напрямую, а смотреть на отношения между префиксами.

### Пример 1. Существует ли непустой подотрезок с суммой $0$?

Пусть $s_i$ — сумма первых $i$ элементов. Тогда сумма на отрезке $[l, r]$ равна $s_{r+1} - s_l$. Она равна нулю тогда и только тогда, когда $s_{r+1} = s_l$.

Значит, задача превращается в вопрос: есть ли два равных префиксных значения?

Решение строится так:

- идём слева направо;
- храним все уже встреченные префиксные суммы;
- если очередная сумма уже была, найден подотрезок с суммой $0$.

```cpp
set<long long> seen;
seen.insert(0);

long long sum = 0;
for (int x : a) {
    sum += x;
    if (seen.count(sum)) {
        cout << "YES\n";
        return;
    }
    seen.insert(sum);
}
cout << "NO\n";
```

Полная программа: [C++](examples/example-01.cpp), [Python3](examples/example-01.py).

### Пример 2. Школа танцев

В задаче [«Школа танцев»](https://acmp.ru/index.asp?main=task&id_task=433) нужно посчитать количество подряд идущих групп учеников, в которых мальчиков и девочек поровну.

Для простоты будем считать, что символ `a` обозначает, например, мальчика, а любой другой символ обозначает девочку. Тогда заменим `a` на $+1$, а любой другой символ на $-1$.

Тогда на отрезке мальчиков и девочек поровну тогда и только тогда, когда сумма на этом отрезке равна $0$.

Значит, задача снова сводится к поиску двух равных префиксных значений. Но теперь нас интересует не существование такого отрезка, а количество всех таких пар префиксов.

> [!NOTE]
> В `map<int, int>` удобно хранить количество появлений каждого префиксного баланса: `cnt[balance]` — сколько раз такой баланс уже встречался раньше.

Если очередная префиксная сумма уже встречалась $k$ раз, то она сразу добавляет $k$ новых подходящих отрезков.

```cpp
map<int, int> cnt;
cnt[0] = 1;

int balance = 0;
long long answer = 0;
for (char ch : s) {
    balance += (ch == 'a' ? 1 : -1);
    answer += cnt[balance];
    cnt[balance]++;
}
```

Полная программа: [C++](examples/example-02.cpp), [Python3](examples/example-02.py).

> [!TIP]
> Это хороший пример того, что префиксный подход часто начинается не с формулы, а с удачной замены модели.

## 8. Суффиксный подход

Не все задачи удобно решать слева направо. Иногда запрос спрашивает про суффикс, или нужное состояние естественно выражается через элементы справа.

Тогда мы строим суффиксные массивы и идём справа налево.

> [!NOTE]
> Базовый пример суффиксных сумм уже был в первой части, поэтому здесь сразу перейдём к примеру, где суффиксный подход используется не только для сумм.

### Пример: минимум на каждом суффиксе

Иногда для каждой позиции $i$ нужно быстро знать минимальный элемент на отрезке от $i$ до конца массива.

Тогда можно построить суффиксный массив минимумов $m$, где

- $m_{n-1} = a_{n-1}$;
- $m_i = \min(a_i, m_{i+1})$.

```cpp
vector<int> suf_min(n);
suf_min[n - 1] = a[n - 1];
for (int i = n - 2; i >= 0; i--) {
    suf_min[i] = min(a[i], suf_min[i + 1]);
}
```

Полная программа: [C++](examples/example-04.cpp), [Python3](examples/example-04.py).

> [!TIP]
> Теперь значение $m_i$ хранит минимум на суффиксе, начинающемся в позиции $i$.
>
> Если одновременно нужна информация и слева, и справа, очень часто рядом строят и префиксный, и суффиксный массивы одного типа: например, префиксные максимумы и суффиксные минимумы.

> [!NOTE]
> Суффиксный подход — это не отдельная магия, а зеркальное отражение префиксного.
> Если слева направо получается громоздко, всегда полезно спросить себя: «А если развернуть направление обработки?»

### Суффиксный подход бывает не только про суммы

Можно строить:

- суффиксный минимум;
- суффиксный максимум;
- количество различных значений на суффиксе;
- число подходящих объектов справа;
- более сложные суффиксные состояния, если рекуррентная формула идёт от правого края.

В олимпиадных задачах такое встречается регулярно.

### Пример: удалить один элемент и максимизировать НОД

Краткая постановка задачи: дан массив. Разрешается удалить ровно один его элемент. Нужно удалить его так, чтобы НОД всех оставшихся элементов был максимально возможным.

Здесь удобно одновременно использовать и префиксный, и суффиксный подход.

Будем работать в $1$-индексации.

Пусть $p_i$ — НОД первых $i$ элементов, а $s_i$ — НОД суффикса, начинающегося в позиции $i$.

Тогда строим два массива:

- $p_0 = 0$;
- $p_i = \gcd(p_{i-1}, a_i)$ при $1 \le i \le n$;
- $s_{n+1} = 0$;
- $s_i = \gcd(a_i, s_{i+1})$ при $1 \le i \le n$.

Если мы удаляем элемент $a_i$, то слева от него остаётся префикс, а справа — суффикс. Значит, НОД всех оставшихся элементов равен НОД этих двух частей.

В коде кандидат на ответ для удаления элемента с индексом `i` считается как `gcd(pref[i - 1], suff[i + 1])`.

> [!IMPORTANT]
> Именно поэтому удобно добавить нули по краям: `pref[0] = 0` и `suff[n + 1] = 0`. Тогда одна и та же формула работает всегда, и не нужно отдельно обрабатывать удаление первого или последнего элемента.

Остаётся просто перебрать все позиции $i$ и взять максимум из всех таких значений.

```cpp
vector<long long> pref(n + 2, 0), suff(n + 2, 0);
for (int i = 1; i <= n; i++) {
    pref[i] = gcd(pref[i - 1], a[i]);
}
for (int i = n; i >= 1; i--) {
    suff[i] = gcd(a[i], suff[i + 1]);
}

long long answer = 0;
for (int i = 1; i <= n; i++) {
    answer = max(answer, gcd(pref[i - 1], suff[i + 1]));
}
```

Полная программа: [C++](examples/example-05.cpp), [Python3](examples/example-05.py).
