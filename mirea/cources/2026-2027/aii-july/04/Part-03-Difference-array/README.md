## 9. Разностный массив

Пусть массив изначально заполнен нулями. Есть много операций вида: прибавить число $x$ ко всем элементам на отрезке $[l, r]$. После выполнения всех операций нужно вывести итоговый массив.

Если делать это честно по всем элементам отрезка, одна операция занимает $O(r - l + 1)$, а все вместе могут дать слишком медленное (квадратичное) решение.

Разностный массив позволяет запланировать за $O(1)$ выполнение каждой такой операции, а затем выполнить их все за линейное время.

### Идея

Давайте представим, что наш итоговый массив будет некоторым массивом префиксных сумм $s$, построенным по массиву $a$.

Тогда элементы массива $s$ устроены так:

$$
s_i = \sum_{j=1}^{i} a_j
$$

Отсюда видно, что элемент $a_j$ входит во все суммы $s_i$, где $i \ge j$. Значит, если мы увеличим $a_j$ на число $x$, то все префиксные суммы $s_j, s_{j+1}, \ldots, s_n$ тоже увеличатся на $x$.

Поэтому, чтобы прибавить $x$ на отрезке $[l, r]$, достаточно увеличить $a_l$ на $x$ и уменьшить $a_{r+1}$ на $x$. Тогда элементы $s_l, s_{l+1}, \ldots, s_r$ увеличатся на $x$, а $s_{r+1}, \ldots, s_n$ останутся без изменений.

Но всё это произойдёт не сразу! Ведь прямо сейчас нам не нужно знать значение какого-то элемента $s_i$, верно? Тогда и префиксные суммы $s$ пока считать не надо, а лучше перейти к обработке следующего запроса.

Именно это и делает разностный массив: мы **накапливаем** все операции в массиве $a$, а когда операции заканчиваются, один раз считаем префиксные суммы поверх массива $a$ и получаем итоговый массив $s$.

### Реализация в C++

<details>
<summary><strong>Показать код на C++</strong></summary>

```cpp
vector<long long> a(n + 1, 0);

for (int i = 0; i < q; i++) {
    int l, r;
    long long x;
    cin >> l >> r >> x;
    --l;
    --r;
    a[l] += x;
    if (r + 1 < n) {
        a[r + 1] -= x;
    }
}

vector<long long> s(n);
long long current = 0;
for (int i = 0; i < n; i++) {
    current += a[i];
    s[i] = current;
}
```

</details>

### Реализация в Python3

<details>
<summary><strong>Показать код на Python3</strong></summary>

```python
a = [0] * (n + 1)

for _ in range(q):
    l, r, x = map(int, input().split())
    l -= 1
    r -= 1
    a[l] += x
    if r + 1 < n:
        a[r + 1] -= x

s = [0] * n
current = 0
for i in range(n):
    current += a[i]
    s[i] = current
```

</details>

Полная программа: [C++](examples/example-01.cpp), [Python3](examples/example-01.py).

> [!IMPORTANT]
> Разностный массив почти всегда означает офлайн-обработку.
> Сначала мы собираем все изменения, потом один раз восстанавливаем итоговые значения.
> Если после каждого запроса нужен немедленный ответ, одной этой техники уже недостаточно.

## 10. Разностный массив как массив покрытий

Очень полезно смотреть на разностный массив не только как на способ прибавлять числа, но и как на способ считать покрытия отрезками (например, множества точек).

Если каждый отрезок просто добавляет $1$ на своём диапазоне, то после восстановления массива мы узнаем, сколько отрезков покрывает каждую позицию (чаще: каждую точку на прямой).

Это даёт целый класс задач:

- сколько раз каждая позиция была затронута;
- какие позиции покрыты хотя бы $k$ отрезками;
- где достигается максимальное покрытие;
- сколько точек, покрытых хотя бы $k$ раз, попадает в каждый диапазон запроса.

### Схема

1. По всем отрезкам накапливаем изменения в разностном массиве $a$.
2. Один раз считаем по нему префиксные суммы и получаем массив покрытий $s$.
3. Если дальше ещё есть запросы по массиву $s$, можно поверх него построить ещё один префиксный массив.

> [!TIP]
> Если задача сначала просит восстановить покрытия, а потом задаёт ещё и запросы по диапазонам, почти всегда нужен второй шаг: из массива покрытий построить ещё один префиксный массив по нужному признаку.

## 11. Двойной шаг предподсчёта

Иногда одного префиксного или одного разностного массива недостаточно. Тогда задача решается в два этапа.

### Типовой сюжет

Есть много отрезков на массиве длины $n$. Позиция называется хорошей, если она покрыта хотя бы $k$ отрезками. После этого приходят запросы $[l, r]$, и для каждого нужно узнать, сколько хороших позиций лежит внутри.

### Как мыслить

Сразу отвечать на такие запросы неудобно. Но можно сделать две последовательные предобработки.

1. Разностным массивом посчитать для каждой позиции число покрывающих её отрезков.
2. Превратить это в бинарный массив $g$, где $g_i = 1$, если позиция хорошая, и $g_i = 0$ иначе.
3. Построить обычные префиксные суммы по массиву $g$.

После этого любой запрос по диапазону снова считается за $O(1)$.

<details>
<summary><strong>C++: два слоя предобработки</strong></summary>

```cpp
vector<int> a(n + 1, 0);
for (auto [l, r] : segments) {
    a[l]++;
    if (r + 1 < n) {
        a[r + 1]--;
    }
}

vector<int> g(n, 0);
int current = 0;
for (int i = 0; i < n; i++) {
    current += a[i];
    if (current >= k) {
        g[i] = 1;
    }
}

vector<int> h(n + 1, 0);
for (int i = 0; i < n; i++) {
    h[i + 1] = h[i] + g[i];
}
```

</details>

<details>
<summary><strong>Python3: два слоя предобработки</strong></summary>

```python
a = [0] * (n + 1)
for l, r in segments:
    a[l] += 1
    if r + 1 < n:
        a[r + 1] -= 1

g = [0] * n
current = 0
for i in range(n):
    current += a[i]
    if current >= k:
        g[i] = 1

h = [0] * (n + 1)
for i in range(n):
    h[i + 1] = h[i] + g[i]
```

</details>

Полная программа: [C++](examples/example-02.cpp), [Python3](examples/example-02.py).

Ответ на запрос $[l, r]$ после этого равен $h_{r+1} - h_l$, где $h$ — префиксные суммы массива $g$.

> [!NOTE]
> Такие многошаговые схемы очень характерны: сначала одна предобработка строит полезный промежуточный массив, а затем вторая предобработка делает по нему быстрые запросы.

> [!IMPORTANT]
> В следующем примере `diff` уже не совпадает с обычным разностным массивом итогового ответа. Здесь это ещё один слой разностей, поэтому правая граница закрывается не одной отметкой, а двумя.

### Финальный пример: прибавление арифметической прогрессии на отрезке

Поступают запросы вида $l$, $r$, $d$. Каждый запрос означает, что нужно выполнить прибавление **арифметической прогрессии** на отрезке:

```cpp
for (int i = l; i <= r; i++) {
    a[i] += (i - l + 1) * d;
}
```

Если делать это честно, один запрос работает за $O(r - l + 1)$, а все запросы вместе могут дать слишком медленное (квадратичное) решение.

Но мы применим два слоя разностных массивов.

Первый слой: какую величину нужно прибавить к префиксным суммам, начиная с каждой позиции.

Если бы мы сделали только

- `diff[l] += d`;
- `diff[r + 1] -= d`;

и потом посчитали префиксные суммы массива `diff`, то получили бы массив $\ldots, d, d, d, \ldots, d, \ldots$ на отрезке $[l, r]$. А если теперь ещё раз посчитать префиксные суммы, то на отрезке как раз получится:

$$
\ldots, d, 2d, 3d, \ldots, (r - l + 1)d, \ldots.
$$

Но тут есть тонкость: без дополнительной коррекции это значение «потечёт» дальше и после позиции $r$.

Если на этом месте всё-таки оставить только

- `diff[l] += d`;
- `diff[r + 1] -= d`;

то получится неправильный ответ.

<details>
<summary><strong>Контрпример к формуле <code>diff[l] += d</code>, <code>diff[r + 1] -= d</code></strong></summary>

Возьмём массив длины $6$ и один запрос: $l = 2$, $r = 4$, $d = 3$.

Тогда после обновлений массив `diff` будет таким:

```text
0 0 3 0 0 -3 0
```

После первого префиксного прохода получаем:

```text
0 3 3 3 0 0
```

После второго префиксного прохода получаем:

```text
0 3 6 9 9 9
```

На отрезке $[2, 4]$ арифметическая прогрессия действительно появилась, но после позиции $4$ массив не вернулся к нулю. Вместо этого справа осталась константа $9$.

Значит, одной отметки `-d` в точке `r + 1` недостаточно: она выключает постоянное прибавление $d$, но не убирает уже накопленную сумму.

</details>

Чтобы обрубить этот хвост, нужно в точке $r + 1$ вычесть всю уже накопившуюся величину $(r - l + 1)\cdot d$, а в точке $r + 2$ вернуть баланс обратно.

Может показаться, что после этого достаточно упростить формулу до

- `diff[l] += d`;
- `diff[r + 1] -= (r - l + 1) * d`.

Но это тоже неверно.

<details>
<summary><strong>Контрпример к формуле <code>diff[l] += d</code>, <code>diff[r + 1] -= len * d</code></strong></summary>

Снова возьмём массив длины $6$ и запрос $l = 2$, $r = 4$, $d = 3$. Тогда `len = 3`.

После обновлений массив `diff` будет таким:

```text
0 0 3 0 0 -9 0
```

После первого префиксного прохода получаем:

```text
0 3 3 3 -6 -6
```

После второго префиксного прохода получаем:

```text
0 3 6 9 3 -3
```

Теперь справа уже нет плато, но появился отрицательный хвост. Причина в том, что мы сделали слишком сильный скачок вниз в точке `r + 1`, а затем не вернули значение к нулю на следующей позиции.

Именно для этого и нужна третья отметка `diff[r + 2] += len * d`: она завершает правую границу корректно.

</details>

Итоговые обновления такие:

- `diff[l] += d`;
- `diff[r + 1] -= (r - l + 2) * d`;
- `diff[r + 2] += (r - l + 1) * d`.

После этого мы два раза считаем префиксные суммы:

1. первый раз получаем массив приращений;
2. второй раз получаем итоговый ответ.

Простыми словами это работает так: первый префикс превращает точечные отметки в постоянное прибавление $d$ на всём отрезке, а второй префикс превращает это постоянное прибавление в арифметическую прогрессию. Дополнительные два обновления справа нужны только для того, чтобы это прибавление вовремя закончилось.

**Физический смысл.** Мы задаём ускорение (вторую производную, `a(t)`). Затем, считая префиксные суммы, мы получаем скорость (первую производную, `v(t)`). И наконец, считая префиксные суммы префиксных сумм, мы получаем пройденное расстояние `s(t)`.

> [!WARNING]
> В этой реализации обязательно нужен запас по размеру массива: мы пишем не только в `l` и `r + 1`, но и в `r + 2`. Поэтому удобно сразу выделять `n + 3` элементов и не смешивать разные схемы индексации.

<details>
<summary><strong>C++: два слоя разностей для арифметической прогрессии</strong></summary>

```cpp
vector<long long> diff(n + 3, 0);
for (int i = 0; i < q; i++) {
    int l, r;
    long long d;
    cin >> l >> r >> d;
    long long len = r - l + 1;
    diff[l] += d;
    diff[r + 1] -= (len + 1) * d;
    diff[r + 2] += len * d;
}

vector<long long> a(n + 1, 0);
long long current = 0;
for (int i = 1; i <= n; i++) {
    current += diff[i];
    a[i] = current;
}

vector<long long> s(n + 1, 0);
for (int i = 1; i <= n; i++) {
    s[i] = s[i - 1] + a[i];
}
```

</details>

Полная программа: [C++](examples/example-03.cpp), [Python3](examples/example-03.py).
