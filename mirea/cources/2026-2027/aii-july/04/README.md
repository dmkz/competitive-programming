# Лекция 4. Префиксные суммы, запросы на отрезках, префиксный и суффиксный подход. Разностный массив

## 1. Префиксные суммы

### Что это такое?

Пусть дан массив $a_0, a_1, \ldots, a_{n-1}$. Будем хранить массив $s$, где $s_0 = 0$ и для каждого $i \ge 1$ значение $s_i$ равно сумме первых $i$ элементов массива. То есть:

$$
s_i = \sum_{j=0}^{i-1} a_j \quad \text{при } 1 \le i \le n.
$$

Эквивалентная запись:

$$
s_{i+1} = \sum_{j=0}^{i} a_j \quad \text{при } 0 \le i < n.
$$

В рекуррентном виде (через уже вычисленные суммы):

$$
s_i = s_{i-1} + a_{i-1} \quad \text{при } 1 \le i \le n.
$$

Первые значения выглядят так:

- $s_1 = a_0$;
- $s_2 = a_0 + a_1$;
- $s_3 = a_0 + a_1 + a_2$;
- и так далее.

Тогда сумма на отрезке $[l, r]$ в нулевой индексации равна

$$
a_l + a_{l+1} + \ldots + a_r = s_{r+1} - s_l.
$$

Если вход использует единичную индексацию и элементы обозначены как $a_1, a_2, \ldots, a_n$, то та же идея записывается со смещением индексов:

$$
a_l + a_{l+1} + \ldots + a_r = s_r - s_{l-1}.
$$

> [!NOTE]
> В коде такой массив префиксных сумм часто называют `pref`.

### Построение префиксных сумм в C++

<details>
<summary><strong>Показать код на C++</strong></summary>

```cpp
int n;
cin >> n;
vector<long long> a(n);
for (int i = 0; i < n; i++) {
    cin >> a[i];
}

vector<long long> pref(n + 1, 0);
for (int i = 0; i < n; i++) {
    pref[i + 1] = pref[i] + a[i];
}
```

</details>

### Построение префиксных сумм в Python3

<details>
<summary><strong>Показать код на Python3</strong></summary>

```python
n = int(input())
a = list(map(int, input().split()))

pref = [0] * (n + 1)
for i in range(n):
    pref[i + 1] = pref[i] + a[i]
```

</details>

### Ответ на запросы суммы на отрезке

```cpp
int q;
cin >> q;
while (q--) {
    int l, r;
    cin >> l >> r;
    --l;
    --r;
    cout << pref[r + 1] - pref[l] << '\n';
}
```

Полная программа для этого базового примера: [C++](Part-01-Prefix-sums/examples/example-02.cpp), [Python3](Part-01-Prefix-sums/examples/example-02.py).

Таким образом, вместо того чтобы каждый раз считать сумму в цикле, мы заранее считаем префиксные суммы, а затем отвечаем на запросы. Сложность:

- построение массива префиксных сумм занимает $O(n)$;
- каждый запрос обрабатывается за $O(1)$;
- суммарно получается $O(n + q)$ вместо $O(nq)$.

> [!IMPORTANT]
> Для префиксных сумм почти всегда нужен тип `long long` в C++: даже если сами элементы массива помещаются в `int`, их суммарное значение может не поместиться.

## 2. Префиксные суммы есть в C++, но их никто не использует, и вот почему

> [!NOTE]
> Это бонусный раздел. `std::partial_sum` есть давно, а `std::inclusive_scan` и `std::exclusive_scan` появились в C++17.

В стандартной библиотеке C++ действительно есть готовые алгоритмы для префиксных сумм. Но в олимпиадном коде их используют редко: обычный цикл короче, прозрачнее и даёт меньше сюрпризов с типами и соглашениями об индексации.

<details>
<summary><strong>Бонус: std::partial_sum</strong></summary>

Самый старый вариант — `std::partial_sum` из `<numeric>`.

```cpp
vector<long long> pref(n + 1, 0);
vector<long long> a64(a.begin(), a.end());
partial_sum(a64.begin(), a64.end(), pref.begin() + 1);
```

Такой код корректно строит массив

$pref_i = a_0 + a_1 + \ldots + a_{i-1}$.

**Внимание.** У `std::partial_sum` есть неприятное ограничение интерфейса: тип накопителя берётся не из третьего параметра, а из типа элементов входного диапазона. Поэтому такой код опасен:

```cpp
vector<int> a = {1000000000, 1000000000, 1000000000};
vector<long long> pref(a.size());
partial_sum(a.begin(), a.end(), pref.begin());
```

**Внимание.** Здесь всё равно возможно переполнение, потому что накопление идёт в `int`, а в `long long` результат попадает уже потом.

Полная программа: [C++](Part-01-Prefix-sums/examples/example-03.cpp), [Python3](Part-01-Prefix-sums/examples/example-03.py).

</details>

<details>
<summary><strong>Бонус: std::inclusive_scan</strong></summary>

У `std::inclusive_scan` есть перегрузка, где тип накопителя можно задать через последний параметр.

```cpp
vector<long long> pref(n + 1, 0);
inclusive_scan(a.begin(), a.end(), pref.begin() + 1, plus<long long>(), 0LL);
```

После этого:

- `pref[0] = 0`;
- `pref[1] = a[0]`;
- `pref[2] = a[0] + a[1]`;
- и так далее.

**Практика.** Здесь входной массив вполне может иметь тип `vector<int>`. Главное, чтобы последний параметр был `0LL`, тогда накопление пойдёт в `long long`.

Полная программа: [C++](Part-01-Prefix-sums/examples/example-04.cpp), [Python3](Part-01-Prefix-sums/examples/example-04.py).

</details>

<details>
<summary><strong>Бонус: std::exclusive_scan</strong></summary>

У `std::exclusive_scan` такой же плюс: тип накопителя тоже можно задать через последний параметр.

```cpp
vector<int> extended = a;
extended.push_back(0);

vector<long long> pref(n + 1);
exclusive_scan(extended.begin(), extended.end(), pref.begin(), 0LL);
```

Этот приём даёт ровно нашу привычную нумерацию:

- `pref[0] = 0`;
- `pref[1] = a[0]`;
- `pref[2] = a[0] + a[1]`;
- `...`;
- `pref[n] = a[0] + a[1] + ... + a[n-1]`.

**Практика.** И здесь входные данные могут быть `int`. Достаточно передать `0LL` последним параметром, чтобы накопление выполнялось в `long long`.

Полная программа: [C++](Part-01-Prefix-sums/examples/example-05.cpp), [Python3](Part-01-Prefix-sums/examples/example-05.py).

</details>

> [!IMPORTANT]
> Вот почему в олимпиадной практике эти алгоритмы пишут редко: у `std::partial_sum` есть ловушка с типом накопителя, `std::inclusive_scan` и `std::exclusive_scan` требуют помнить перегрузки и стандарт C++17, а обычный цикл короче, понятнее и почти всегда читается лучше.

## 3. Префиксы не только по значениям, но и по признакам

Префиксные суммы можно строить не только по самому массиву $a$. Иногда нас интересуют:

- количество чётных элементов на отрезке;
- количество элементов больше нуля;
- сумма только нечётных элементов;
- количество букв $a$ в подстроке;
- количество пар равных соседних элементов на отрезке.

Тогда мы строим суммы не по самим значениям $a_i$, а по некоторому вкладу, который каждый элемент даёт в ответ.

<details>
<summary><strong>Пример: сумма положительных чисел на отрезке</strong></summary>

Пусть $b_i = a_i$, если $a_i > 0$, и $b_i = 0$ иначе.

Пусть $s_i$ — префиксные суммы массива $b$. Тогда сумма положительных чисел на отрезке $[l, r]$ равна $s_{r+1} - s_l$.

</details>

<details>
<summary><strong>C++: сумма положительных чисел на отрезке</strong></summary>

```cpp
int n, q;
cin >> n >> q;
vector<long long> a(n);
for (int i = 0; i < n; i++) {
    cin >> a[i];
}

vector<long long> pref_pos(n + 1, 0);

for (int i = 0; i < n; i++) {
    pref_pos[i + 1] = pref_pos[i];
    if (a[i] > 0) {
        pref_pos[i + 1] += a[i];
    }
}

while (q--) {
    int l, r;
    cin >> l >> r;
    --l;
    --r;
    cout << pref_pos[r + 1] - pref_pos[l] << '\n';
}
```

</details>

<details>
<summary><strong>Python3: сумма положительных чисел на отрезке</strong></summary>

```python
n, q = map(int, input().split())
a = list(map(int, input().split()))

pref_pos = [0] * (n + 1)
for i in range(n):
    pref_pos[i + 1] = pref_pos[i]
    if a[i] > 0:
        pref_pos[i + 1] += a[i]

for _ in range(q):
    l, r = map(int, input().split())
    l -= 1
    r -= 1
    print(pref_pos[r + 1] - pref_pos[l])
```

</details>

<details>
<summary><strong>Пример ввода и вывода</strong></summary>

Для ввода

```text
5 4
2 -7 4 0 5
1 5
2 4
4 5
3 3
```

получим вывод

```text
11
4
5
4
```

</details>

Файлы примера: [C++](Part-01-Prefix-sums/examples/example-01.cpp), [Python3](Part-01-Prefix-sums/examples/example-01.py).

<details>
<summary><strong>Пример: количество каждой буквы от 'a' до 'z' на отрезке</strong></summary>

Пусть дана строка длины $n$. Самая прямолинейная идея: завести $26$ префиксных массивов $p^{(0)}, p^{(1)}, \ldots, p^{(25)}$, где каждый массив имеет длину $n+1$, а массив $p^{(c)}$ отвечает только за свою букву.

Для выбранной буквы $c$ рекуррентная формула выглядит так: $p^{(c)}_{i+1} = p^{(c)}_i + [s_i = c]$.

Тогда количество буквы $c$ на отрезке $[l, r]$ равно $p^{(c)}_{r+1} - p^{(c)}_l$.

Если запрос просит количества всех букв сразу, такая реализация делает $52$ обращения к памяти: по два чтения на каждую из $26$ букв.

На практике обычно быстрее хранить один префиксный массив длины $n+1$, где в каждой позиции лежит массив из $26$ счётчиков:

```cpp
string s;
cin >> s;
int n = (int)s.size();

vector<array<int, 26>> pref(n + 1);
pref[0].fill(0);
for (int i = 0; i < n; i++) {
    pref[i + 1] = pref[i];
    pref[i + 1][s[i] - 'a']++;
}

int l, r;
cin >> l >> r;
--l;
--r;
for (int c = 0; c < 26; c++) {
    cout << pref[r + 1][c] - pref[l][c] << " \n"[c + 1 == 26];
}
```

**Практика.** Идея та же самая, но доступ к памяти получается гораздо лучше. На практике такой вариант нередко работает примерно в $5$ раз быстрее: вместо $52$ разрозненных чтений мы работаем с двумя компактными объектами `pref[l]` и `pref[r + 1]`, а типичная кеш-линия имеет размер $64$ байта, то есть $512$ бит. Если вместо `vector<int>` использовать `array<int, 26>`, лишней косвенности тоже не будет, и это обычно ещё немного ускоряет код.

Полная программа: [C++](Part-01-Prefix-sums/examples/example-09.cpp), [Python3](Part-01-Prefix-sums/examples/example-09.py).

</details>

<details>
<summary><strong>Бонусный пример: функция Эйлера на произведении чисел отрезка</strong></summary>

**Бонус.** Это именно бонусный пример.

Если все значения массива не превосходят $300$, то простых чисел до $300$ всего $62$. Пронумеруем их как $p_0, p_1, \ldots, p_{61}$.

Теперь можно хранить не одно число на позицию, а вектор показателей простых. Пусть $b_{i, j}$ — степень вхождения $j$-го простого числа в разложение числа $a_i$.

Тогда для каждого фиксированного $j$ мы можем построить префиксные суммы по столбцу $b_{0, j}, b_{1, j}, \ldots, b_{n-1, j}$. На отрезке $[l, r]$ получим суммарную степень $K_j = b_{l, j} + b_{l+1, j} + \ldots + b_{r, j}$.

Значит, произведение всех чисел на отрезке раскладывается как $a_l a_{l+1} \cdots a_r = \prod_{j=0}^{61} p_j^{K_j}$.

После этого можно, например, посчитать функцию Эйлера от этого произведения: $\varphi\left(\prod_{j=0}^{61} p_j^{K_j}\right) = \prod_{j : K_j > 0} \left(p_j^{K_j} - p_j^{K_j - 1}\right)$.

Если хочется записывать множитель в виде $p^{k+1} - p^k$, то здесь просто $k = K_j - 1$.

**Идея.** Это хороший пример того, что иногда вклад элемента — не одно число, а целый небольшой вектор признаков.

Полная программа с ответом по модулю $10^9+7$: [C++](Part-01-Prefix-sums/examples/example-10.cpp), [Python3](Part-01-Prefix-sums/examples/example-10.py).

</details>

> [!NOTE]
> Это очень важный паттерн: если запрос спрашивает не про сами элементы, а про их вклад по какому-то признаку, часто достаточно заменить исходный массив на массив вкладов и дальше снова использовать префиксные суммы.

## 4. Какие функции можно считать на отрезке?

Идея префиксов работает не только для обычной суммы. Мы хотим заранее считать значения функции на префиксах, а потом получать ответ на отрезке через два префикса.

Для сумм и количеств это обычно записывают так:

$$
f(l, r) = f(0, r) - f(0, l - 1).
$$

Общая часть $[0, l-1]$ входит в оба префикса и взаимоуничтожается.

Если использовать нулевую индексацию и обозначать через $f_i$ значение на первых $i$ элементах, то формула выглядит так:

$$
f(l, r) = f_{r+1} - f_l.
$$

В более общем случае вместо вычитания нужна операция, которая умеет убрать общий префикс. Значит, техника подходит не для любой функции, а только для тех, где общая часть действительно сокращается.

Подходят, например:

- обычная сумма;
- количество элементов с некоторым свойством;
- сумма вкладов элементов по некоторому признаку;
- `XOR` (взаимоисключающее ИЛИ);
- суммы и произведения по модулю $m$.

Обычно не подходят:

- минимум;
- максимум;
- побитовое `OR` (побитовое ИЛИ);
- побитовое `AND` (побитовое И);
- `gcd` (наибольший общий делитель).

> [!WARNING]
> По двум префиксам нельзя просто убрать общую часть для таких функций.

### Пример: XOR на отрезке

Пусть

$$
x_i = a_0 \oplus a_1 \oplus \ldots \oplus a_{i-1}.
$$

Тогда XOR на отрезке $[l, r]$ равен

$$
a_l \oplus a_{l+1} \oplus \ldots \oplus a_r = x_{r+1} \oplus x_l.
$$

Это работает, потому что общая часть встречается два раза, а для XOR верно $u \oplus u = 0$.

<details>
<summary><strong>C++: XOR на отрезке</strong></summary>

```cpp
int n;
cin >> n;
vector<int> a(n);
for (int i = 0; i < n; i++) {
    cin >> a[i];
}

vector<int> pref_xor(n + 1, 0);
for (int i = 0; i < n; i++) {
    pref_xor[i + 1] = pref_xor[i] ^ a[i];
}

int q;
cin >> q;
while (q--) {
    int l, r;
    cin >> l >> r;
    --l;
    --r;
    cout << (pref_xor[r + 1] ^ pref_xor[l]) << '\n';
}
```

</details>

<details>
<summary><strong>Python3: XOR на отрезке</strong></summary>

```python
n = int(input())
a = list(map(int, input().split()))

pref_xor = [0] * (n + 1)
for i in range(n):
    pref_xor[i + 1] = pref_xor[i] ^ a[i]

q = int(input())
for _ in range(q):
    l, r = map(int, input().split())
    l -= 1
    r -= 1
    print(pref_xor[r + 1] ^ pref_xor[l])
```

</details>

> [!NOTE]
> Если общую часть можно убрать обратной операцией или самоуничтожением, как у XOR, то префиксный подход сработает.

## 5. Иногда удобнее использовать суффикс-суммы

Префиксы не обязательны. Иногда формулу проще писать справа налево.

У суффиксных сумм есть два удобных способа индексации:

- индексация позицией начала суффикса: $s_i$;
- индексация длиной суффикса: $s_k$.

Оба способа могут пригодиться в задачах. Буква одна и та же, меняется только смысл индекса: в первом случае это позиция начала, во втором случае это длина.

### Индексация позицией начала суффикса

Для обычных суффиксных сумм задаём: $s_n = 0$.

Для $i = n - 1, n - 2, \ldots, 0$ выполняется рекурсия $s_i = s_{i+1} + a_i$.

Тогда сумма на отрезке $[l, r]$ равна $s_l - s_{r+1}$.

```cpp
vector<long long> suff(n + 1, 0);
for (int i = n - 1; i >= 0; i--) {
    suff[i] = suff[i + 1] + a[i];
}
```

Полная программа: [C++](Part-01-Prefix-sums/examples/example-06.cpp), [Python3](Part-01-Prefix-sums/examples/example-06.py).

> [!TIP]
> Этот способ удобен, когда состояние естественно выражается через позицию начала хвоста.

### Индексация длиной суффикса

Иногда удобнее считать не суффикс, начинающийся в позиции $i$, а суффикс длины $k$.

Пусть $s_k$ — сумма последних $k$ элементов массива. Универсальный олимпиадный способ построения:

```cpp
vector<long long> s = {0};
for (int i = n - 1; i >= 0; i--) {
    s.push_back(s.back() + a[i]);
}
```

Полная программа: [C++](Part-01-Prefix-sums/examples/example-07.cpp), [Python3](Part-01-Prefix-sums/examples/example-07.py).

> [!TIP]
> У этого способа есть важный плюс: внутри цикла не нужно думать про индексы результирующего массива. Меняется только порядок обхода.

Например, для префиксных сумм цикл такой:

```cpp
vector<long long> s = {0};
for (int i = 0; i < n; i++) {
    s.push_back(s.back() + a[i]);
}
```

Полная программа: [C++](Part-01-Prefix-sums/examples/example-07.cpp), [Python3](Part-01-Prefix-sums/examples/example-07.py).

> [!IMPORTANT]
> Содержимое цикла и интерфейс построения остаются общими: мы всегда делаем `push_back(s.back() + a[i])`, а меняется только направление обхода.

### Нестандартные суффиксные рекурсии

Иногда через суффиксную рекурсию удобнее записывать и не совсем обычные величины. Например, если нужен знакочередующийся хвост

$$
a_i - a_{i+1} + a_{i+2} - a_{i+3} + \ldots,
$$

то естественная формула выглядит так: $t_i = a_i - t_{i+1}$.

> [!NOTE]
> Здесь уже естественнее именно индексация позицией начала, а не длиной.

> [!WARNING]
> Но у суффиксного мышления есть минус: в таких формулах обычно приходится думать справа налево, а это для многих заметно тяжелее, чем обычные префиксы.

## 6. Восстановление оригинального массива, зная префиксные суммы

Если известны все префиксные суммы, исходный массив восстанавливается единственным образом.

Причина простая: два соседних префикса отличаются ровно на один элемент. Значит,

$$
a_i = s_{i+1} - s_i.
$$

В единичной индексации это та же самая формула в виде $a_i = s_i - s_{i-1}$.

> [!IMPORTANT]
> Другого варианта восстановления нет, потому что каждое значение $a_i$ уже однозначно задаётся разностью двух соседних префиксов.

<details>
<summary><strong>Показать код восстановления на C++</strong></summary>

```cpp
int n;
cin >> n;
vector<long long> pref(n + 1);
for (int i = 0; i <= n; i++) {
    cin >> pref[i];
}

vector<long long> a(n);
for (int i = 0; i < n; i++) {
    a[i] = pref[i + 1] - pref[i];
}
```

</details>

<details>
<summary><strong>Показать код восстановления на Python3</strong></summary>

```python
n = int(input())
pref = list(map(int, input().split()))

a = [0] * n
for i in range(n):
    a[i] = pref[i + 1] - pref[i]
```

</details>

Полная программа: [C++](Part-01-Prefix-sums/examples/example-08.cpp), [Python3](Part-01-Prefix-sums/examples/example-08.py).

## 7. Что такое префиксный подход

Иногда ответ на задачу нельзя получить одной формулой для суммы на отрезке. Но всё равно можно перейти к массиву префиксных состояний и выразить условие на подотрезок через связь между двумя префиксами.

Это и есть префиксный подход.

### Общая схема

1. Для каждой позиции строим некоторое префиксное состояние $p_i$.
2. Рассматриваем отрезок $[l, r]$ не сам по себе, а через связь между состояниями $p_l$ и $p_{r+1}$.
3. Получаем задачу не про элементы отрезка напрямую, а про пары префиксов.
4. После этого часто подключаются map, set, массивы частот или бинарный поиск.

> [!IMPORTANT]
> Если в задаче встречается условие вида «сумма на подотрезке равна ...», «баланс равен ...», «количество двух типов объектов совпадает ...», очень часто нужно не пересчитывать отрезки напрямую, а смотреть на отношения между префиксами.

### Пример 1. Существует ли непустой подотрезок с суммой $0$?

Пусть $s_i$ — сумма первых $i$ элементов. Тогда сумма на отрезке $[l, r]$ равна $s_{r+1} - s_l$. Она равна нулю тогда и только тогда, когда $s_{r+1} = s_l$.

Значит, задача превращается в вопрос: есть ли два равных префиксных значения?

Решение строится так:

- идём слева направо;
- храним все уже встреченные префиксные суммы;
- если очередная сумма уже была, найден подотрезок с суммой $0$.

<details>
<summary><strong>C++: поиск двух равных префиксов</strong></summary>

```cpp
set<long long> seen;
seen.insert(0);

long long sum = 0;
for (int x : a) {
    sum += x;
    if (seen.count(sum)) {
        cout << "YES\n";
        return;
    }
    seen.insert(sum);
}
cout << "NO\n";
```

</details>

Полная программа: [C++](Part-02-Prefix-and-suffix-approach/examples/example-01.cpp), [Python3](Part-02-Prefix-and-suffix-approach/examples/example-01.py).

### Пример 2. Школа танцев

В задаче [«Школа танцев»](https://acmp.ru/index.asp?main=task&id_task=433) нужно посчитать количество подряд идущих групп учеников, в которых мальчиков и девочек поровну.

Для простоты будем считать, что символ `a` обозначает, например, мальчика, а любой другой символ обозначает девочку. Тогда заменим `a` на $+1$, а любой другой символ на $-1$.

Тогда на отрезке мальчиков и девочек поровну тогда и только тогда, когда сумма на этом отрезке равна $0$.

Значит, задача снова сводится к поиску двух равных префиксных значений. Но теперь нас интересует не существование такого отрезка, а количество всех таких пар префиксов.

> [!NOTE]
> В `map<int, int>` удобно хранить количество появлений каждого префиксного баланса: `cnt[balance]` — сколько раз такой баланс уже встречался раньше.

Если очередная префиксная сумма уже встречалась $k$ раз, то она сразу добавляет $k$ новых подходящих отрезков.

```cpp
map<int, int> cnt;
cnt[0] = 1;

int balance = 0;
long long answer = 0;
for (char ch : s) {
    balance += (ch == 'a' ? 1 : -1);
    answer += cnt[balance];
    cnt[balance]++;
}
```

Полная программа: [C++](Part-02-Prefix-and-suffix-approach/examples/example-02.cpp), [Python3](Part-02-Prefix-and-suffix-approach/examples/example-02.py).

> [!TIP]
> Это хороший пример того, что префиксный подход часто начинается не с формулы, а с удачной замены модели.

## 8. Суффиксный подход

Не все задачи удобно решать слева направо. Иногда запрос спрашивает про суффикс, или нужное состояние естественно выражается через элементы справа.

Тогда мы строим суффиксные массивы и идём справа налево.

> [!NOTE]
> Базовый пример суффиксных сумм уже был в первой части, поэтому здесь сразу перейдём к примеру, где суффиксный подход используется не только для сумм.

### Пример: минимум на каждом суффиксе

Иногда для каждой позиции $i$ нужно быстро знать минимальный элемент на отрезке от $i$ до конца массива.

Тогда можно построить суффиксный массив минимумов $m$, где

- $m_{n-1} = a_{n-1}$;
- $m_i = \min(a_i, m_{i+1})$.

```cpp
vector<int> suf_min(n);
suf_min[n - 1] = a[n - 1];
for (int i = n - 2; i >= 0; i--) {
    suf_min[i] = min(a[i], suf_min[i + 1]);
}
```

Полная программа: [C++](Part-02-Prefix-and-suffix-approach/examples/example-04.cpp), [Python3](Part-02-Prefix-and-suffix-approach/examples/example-04.py).

> [!TIP]
> Теперь значение $m_i$ хранит минимум на суффиксе, начинающемся в позиции $i$.
>
> Если одновременно нужна информация и слева, и справа, очень часто рядом строят и префиксный, и суффиксный массивы одного типа: например, префиксные максимумы и суффиксные минимумы.

> [!NOTE]
> Суффиксный подход — это не отдельная магия, а зеркальное отражение префиксного.
> Если слева направо получается громоздко, всегда полезно спросить себя: «А если развернуть направление обработки?»

### Суффиксный подход бывает не только про суммы

Можно строить:

- суффиксный минимум;
- суффиксный максимум;
- количество различных значений на суффиксе;
- число подходящих объектов справа;
- более сложные суффиксные состояния, если рекуррентная формула идёт от правого края.

В олимпиадных задачах такое встречается регулярно.

### Пример: удалить один элемент и максимизировать НОД

Краткая постановка задачи: дан массив. Разрешается удалить ровно один его элемент. Нужно удалить его так, чтобы НОД всех оставшихся элементов был максимально возможным.

Здесь удобно одновременно использовать и префиксный, и суффиксный подход.

Будем работать в $1$-индексации.

Пусть $p_i$ — НОД первых $i$ элементов, а $s_i$ — НОД суффикса, начинающегося в позиции $i$.

Тогда строим два массива:

- $p_0 = 0$;
- $p_i = \gcd(p_{i-1}, a_i)$ при $1 \le i \le n$;
- $s_{n+1} = 0$;
- $s_i = \gcd(a_i, s_{i+1})$ при $1 \le i \le n$.

Если мы удаляем элемент $a_i$, то слева от него остаётся префикс, а справа — суффикс. Значит, НОД всех оставшихся элементов равен НОД этих двух частей.

В коде кандидат на ответ для удаления элемента с индексом `i` считается как `gcd(pref[i - 1], suff[i + 1])`.

> [!IMPORTANT]
> Именно поэтому удобно добавить нули по краям: `pref[0] = 0` и `suff[n + 1] = 0`. Тогда одна и та же формула работает всегда, и не нужно отдельно обрабатывать удаление первого или последнего элемента.

Остаётся просто перебрать все позиции $i$ и взять максимум из всех таких значений.

<details>
<summary><strong>C++: максимум НОД после удаления одного элемента</strong></summary>

```cpp
vector<long long> pref(n + 2, 0), suff(n + 2, 0);
for (int i = 1; i <= n; i++) {
    pref[i] = gcd(pref[i - 1], a[i]);
}
for (int i = n; i >= 1; i--) {
    suff[i] = gcd(a[i], suff[i + 1]);
}

long long answer = 0;
for (int i = 1; i <= n; i++) {
    answer = max(answer, gcd(pref[i - 1], suff[i + 1]));
}
```

</details>

Полная программа: [C++](Part-02-Prefix-and-suffix-approach/examples/example-05.cpp), [Python3](Part-02-Prefix-and-suffix-approach/examples/example-05.py).

## 9. Разностный массив

Пусть массив изначально заполнен нулями. Есть много операций вида: прибавить число $x$ ко всем элементам на отрезке $[l, r]$. После выполнения всех операций нужно вывести итоговый массив.

Если делать это честно по всем элементам отрезка, одна операция занимает $O(r - l + 1)$, а все вместе могут дать слишком медленное (квадратичное) решение.

Разностный массив позволяет запланировать за $O(1)$ выполнение каждой такой операции, а затем выполнить их все за линейное время.

### Идея

Давайте представим, что наш итоговый массив будет некоторым массивом префиксных сумм $s$, построенным по массиву $a$.

Тогда элементы массива $s$ устроены так:

$$
s_i = \sum_{j=1}^{i} a_j
$$

Отсюда видно, что элемент $a_j$ входит во все суммы $s_i$, где $i \ge j$. Значит, если мы увеличим $a_j$ на число $x$, то все префиксные суммы $s_j, s_{j+1}, \ldots, s_n$ тоже увеличатся на $x$.

Поэтому, чтобы прибавить $x$ на отрезке $[l, r]$, достаточно увеличить $a_l$ на $x$ и уменьшить $a_{r+1}$ на $x$. Тогда элементы $s_l, s_{l+1}, \ldots, s_r$ увеличатся на $x$, а $s_{r+1}, \ldots, s_n$ останутся без изменений.

Но всё это произойдёт не сразу! Ведь прямо сейчас нам не нужно знать значение какого-то элемента $s_i$, верно? Тогда и префиксные суммы $s$ пока считать не надо, а лучше перейти к обработке следующего запроса.

Именно это и делает разностный массив: мы **накапливаем** все операции в массиве $a$, а когда операции заканчиваются, один раз считаем префиксные суммы поверх массива $a$ и получаем итоговый массив $s$.

### Реализация в C++

<details>
<summary><strong>Показать код на C++</strong></summary>

```cpp
vector<long long> a(n + 1, 0);

for (int i = 0; i < q; i++) {
    int l, r;
    long long x;
    cin >> l >> r >> x;
    --l;
    --r;
    a[l] += x;
    if (r + 1 < n) {
        a[r + 1] -= x;
    }
}

vector<long long> s(n);
long long current = 0;
for (int i = 0; i < n; i++) {
    current += a[i];
    s[i] = current;
}
```

</details>

### Реализация в Python3

<details>
<summary><strong>Показать код на Python3</strong></summary>

```python
a = [0] * (n + 1)

for _ in range(q):
    l, r, x = map(int, input().split())
    l -= 1
    r -= 1
    a[l] += x
    if r + 1 < n:
        a[r + 1] -= x

s = [0] * n
current = 0
for i in range(n):
    current += a[i]
    s[i] = current
```

</details>

Полная программа: [C++](Part-03-Difference-array/examples/example-01.cpp), [Python3](Part-03-Difference-array/examples/example-01.py).

> [!IMPORTANT]
> Разностный массив почти всегда означает офлайн-обработку.
> Сначала мы собираем все изменения, потом один раз восстанавливаем итоговые значения.
> Если после каждого запроса нужен немедленный ответ, одной этой техники уже недостаточно.

## 10. Разностный массив как массив покрытий

Очень полезно смотреть на разностный массив не только как на способ прибавлять числа, но и как на способ считать покрытия отрезками (например, множества точек).

Если каждый отрезок просто добавляет $1$ на своём диапазоне, то после восстановления массива мы узнаем, сколько отрезков покрывает каждую позицию (чаще: каждую точку на прямой).

Это даёт целый класс задач:

- сколько раз каждая позиция была затронута;
- какие позиции покрыты хотя бы $k$ отрезками;
- где достигается максимальное покрытие;
- сколько точек, покрытых хотя бы $k$ раз, попадает в каждый диапазон запроса.

### Схема

1. По всем отрезкам накапливаем изменения в разностном массиве $a$.
2. Один раз считаем по нему префиксные суммы и получаем массив покрытий $s$.
3. Если дальше ещё есть запросы по массиву $s$, можно поверх него построить ещё один префиксный массив.

> [!TIP]
> Если задача сначала просит восстановить покрытия, а потом задаёт ещё и запросы по диапазонам, почти всегда нужен второй шаг: из массива покрытий построить ещё один префиксный массив по нужному признаку.

---

## 11. Двойной шаг предподсчёта

Иногда одного префиксного или одного разностного массива недостаточно. Тогда задача решается в два этапа.

### Типовой сюжет

Есть много отрезков на массиве длины $n$. Позиция называется хорошей, если она покрыта хотя бы $k$ отрезками. После этого приходят запросы $[l, r]$, и для каждого нужно узнать, сколько хороших позиций лежит внутри.

### Как мыслить

Сразу отвечать на такие запросы неудобно. Но можно сделать две последовательные предобработки.

1. Разностным массивом посчитать для каждой позиции число покрывающих её отрезков.
2. Превратить это в бинарный массив $g$, где $g_i = 1$, если позиция хорошая, и $g_i = 0$ иначе.
3. Построить обычные префиксные суммы по массиву $g$.

После этого любой запрос по диапазону снова считается за $O(1)$.

<details>
<summary><strong>C++: два слоя предобработки</strong></summary>

```cpp
vector<int> a(n + 1, 0);
for (auto [l, r] : segments) {
    a[l]++;
    if (r + 1 < n) {
        a[r + 1]--;
    }
}

vector<int> g(n, 0);
int current = 0;
for (int i = 0; i < n; i++) {
    current += a[i];
    if (current >= k) {
        g[i] = 1;
    }
}

vector<int> h(n + 1, 0);
for (int i = 0; i < n; i++) {
    h[i + 1] = h[i] + g[i];
}
```

</details>

<details>
<summary><strong>Python3: два слоя предобработки</strong></summary>

```python
a = [0] * (n + 1)
for l, r in segments:
    a[l] += 1
    if r + 1 < n:
        a[r + 1] -= 1

g = [0] * n
current = 0
for i in range(n):
    current += a[i]
    if current >= k:
        g[i] = 1

h = [0] * (n + 1)
for i in range(n):
    h[i + 1] = h[i] + g[i]
```

</details>

Полная программа: [C++](Part-03-Difference-array/examples/example-02.cpp), [Python3](Part-03-Difference-array/examples/example-02.py).

Ответ на запрос $[l, r]$ после этого равен $h_{r+1} - h_l$, где $h$ — префиксные суммы массива $g$.

> [!NOTE]
> Такие многошаговые схемы очень характерны: сначала одна предобработка строит полезный промежуточный массив, а затем вторая предобработка делает по нему быстрые запросы.

> [!IMPORTANT]
> В следующем примере `diff` уже не совпадает с обычным разностным массивом итогового ответа. Здесь это ещё один слой разностей, поэтому правая граница закрывается не одной отметкой, а двумя.

### Финальный пример: прибавление арифметической прогрессии на отрезке

Поступают запросы вида $l$, $r$, $d$. Каждый запрос означает, что нужно выполнить прибавление **арифметической прогрессии** на отрезке:

```cpp
for (int i = l; i <= r; i++) {
    a[i] += (i - l + 1) * d;
}
```

Если делать это честно, один запрос работает за $O(r - l + 1)$, а все запросы вместе могут дать слишком медленное (квадратичное) решение.

Но мы применим два слоя разностных массивов.

Первый слой: какую величину нужно прибавить к префиксным суммам, начиная с каждой позиции.

Если бы мы сделали только

- `diff[l] += d`;
- `diff[r + 1] -= d`;

и потом посчитали префиксные суммы массива `diff`, то получили бы массив $\ldots, d, d, d, \ldots, d, \ldots$ на отрезке $[l, r]$. А если теперь ещё раз посчитать префиксные суммы, то на отрезке как раз получится:

$$
\ldots, d, 2d, 3d, \ldots, (r - l + 1)d, \ldots.
$$

Но тут есть тонкость: без дополнительной коррекции это значение «потечёт» дальше и после позиции $r$.

Если на этом месте всё-таки оставить только

- `diff[l] += d`;
- `diff[r + 1] -= d`;

то получится неправильный ответ.

<details>
<summary><strong>Контрпример к формуле <code>diff[l] += d</code>, <code>diff[r + 1] -= d</code></strong></summary>

Возьмём массив длины $6$ и один запрос: $l = 2$, $r = 4$, $d = 3$.

Тогда после обновлений массив `diff` будет таким:

```text
0 0 3 0 0 -3 0
```

После первого префиксного прохода получаем:

```text
0 3 3 3 0 0
```

После второго префиксного прохода получаем:

```text
0 3 6 9 9 9
```

На отрезке $[2, 4]$ арифметическая прогрессия действительно появилась, но после позиции $4$ массив не вернулся к нулю. Вместо этого справа осталась константа $9$.

Значит, одной отметки `-d` в точке `r + 1` недостаточно: она выключает постоянное прибавление $d$, но не убирает уже накопленную сумму.

</details>

Чтобы обрубить этот хвост, нужно в точке $r + 1$ вычесть всю уже накопившуюся величину $(r - l + 1)\cdot d$, а в точке $r + 2$ вернуть баланс обратно.

Может показаться, что после этого достаточно упростить формулу до

- `diff[l] += d`;
- `diff[r + 1] -= (r - l + 1) * d`.

Но это тоже неверно.

<details>
<summary><strong>Контрпример к формуле <code>diff[l] += d</code>, <code>diff[r + 1] -= len * d</code></strong></summary>

Снова возьмём массив длины $6$ и запрос $l = 2$, $r = 4$, $d = 3$. Тогда `len = 3`.

После обновлений массив `diff` будет таким:

```text
0 0 3 0 0 -9 0
```

После первого префиксного прохода получаем:

```text
0 3 3 3 -6 -6
```

После второго префиксного прохода получаем:

```text
0 3 6 9 3 -3
```

Теперь справа уже нет плато, но появился отрицательный хвост. Причина в том, что мы сделали слишком сильный скачок вниз в точке `r + 1`, а затем не вернули значение к нулю на следующей позиции.

Именно для этого и нужна третья отметка `diff[r + 2] += len * d`: она завершает правую границу корректно.

</details>

Итоговые обновления такие:

- `diff[l] += d`;
- `diff[r + 1] -= (r - l + 2) * d`;
- `diff[r + 2] += (r - l + 1) * d`.

После этого мы два раза считаем префиксные суммы:

1. первый раз получаем массив приращений;
2. второй раз получаем итоговый ответ.

Простыми словами это работает так: первый префикс превращает точечные отметки в постоянное прибавление $d$ на всём отрезке, а второй префикс превращает это постоянное прибавление в арифметическую прогрессию. Дополнительные два обновления справа нужны только для того, чтобы это прибавление вовремя закончилось.

**Физический смысл.** Мы задаём ускорение (вторую производную, `a(t)`). Затем, считая префиксные суммы, мы получаем скорость (первую производную, `v(t)`). И наконец, считая префиксные суммы префиксных сумм, мы получаем пройденное расстояние `s(t)`.

> [!WARNING]
> В этой реализации обязательно нужен запас по размеру массива: мы пишем не только в `l` и `r + 1`, но и в `r + 2`. Поэтому удобно сразу выделять `n + 3` элементов и не смешивать разные схемы индексации.

<details>
<summary><strong>C++: два слоя разностей для арифметической прогрессии</strong></summary>

```cpp
vector<long long> diff(n + 3, 0);
for (int i = 0; i < q; i++) {
    int l, r;
    long long d;
    cin >> l >> r >> d;
    long long len = r - l + 1;
    diff[l] += d;
    diff[r + 1] -= (len + 1) * d;
    diff[r + 2] += len * d;
}

vector<long long> a(n + 1, 0);
long long current = 0;
for (int i = 1; i <= n; i++) {
    current += diff[i];
    a[i] = current;
}

vector<long long> s(n + 1, 0);
for (int i = 1; i <= n; i++) {
    s[i] = s[i - 1] + a[i];
}
```

</details>
