# Разбор задач

## A. Очередь

<details>
<summary>Текстовый разбор</summary>

В этой задаче нужно смоделировать работу очереди. Запрос $\text{+}\ x$ добавляет число в конец, а запрос $-$ должен вывести и удалить первый элемент. Возьмём готовую структуру данных и выполним операции в том порядке, в котором они приходят. В C++ для этого подходит `queue`, а в Python – `deque`.

Такое решение работает за $O(q)$ по времени, потому что каждый запрос обрабатывается за $O(1)$, и за $O(q)$ по памяти в худшем случае.

</details>

<details>
<summary>Решение на C++</summary>

```cpp
#include <bits/stdc++.h>
#include <cassert>
using namespace std;
void solve() {
    int nQ; cin >> nQ;
    queue<int> q;
    while (nQ --> 0) {
        char c; cin >> c;
        if (c == '+') {
            // добавление элемента в конец очереди:
            int x;
            cin >> x;
            q.push(x);
        } else {
            assert(c == '-');
            // вывод первого элемента в очереди и его удаление:
            cout << q.front() << '\n';
            q.pop();
        }
    }
}
main() {
    int tt; cin >> tt;
    while (tt --> 0) solve();
}
```

</details>

<details>
<summary>Решение на Python3</summary>

```python
from collections import deque

def solve():
    nQ = int(input())
    q = deque()
    for _ in range(nQ):
        query = input().split()
        c = query[0]
        if c == '+':
            # добавление элемента в конец очереди:
            x = int(query[1])
            q.append(x)
        else:
            # вывод первого элемента в очереди и его удаление:
            print(q.popleft())

tt = int(input())
for _ in range(tt):
    solve()
```

</details>

## B. Стек на минимум

<details>
<summary>Текстовый разбор</summary>

Если хранить в стеке сами значения, то на запрос $\text{?}$ пришлось бы каждый раз заново искать минимум. Чтобы этого избежать, держим на вершине не последний элемент, а минимум во всём стеке. Тогда при добавлении числа $x$ наверх кладётся значение $\min(x, \text{текущий минимум})$, а запрос $\text{?}$ просто смотрит в вершину стека.

Удаление работает автоматически: когда мы снимаем верхушку, вместе с ней исчезает и минимум для текущего состояния, а на новой вершине уже лежит минимум для предыдущего состояния. По времени решение работает за $O(q)$, потому что каждый запрос обрабатывается за $O(1)$. По памяти используется $O(q)$.

</details>

<details>
<summary>Решение на C++</summary>

```cpp
#include <bits/stdc++.h>
#include <cassert>
using namespace std;
const int inf = 1000000007;
void solve() {
    int q; cin >> q;
    stack<int> st;
    st.push(inf); // добавляем бесконечность
    while (q --> 0) {
        char c; cin >> c;
        if (c == '+') {
            // добавление элемента + обновление минимума:
            int x;
            cin >> x;
            st.push(min(x, st.top()));
        } else if (c == '?') {
            // вывод минимума на экран:
            cout << st.top() << '\n';
        } else {
            // удаление последнего элемента:
            assert(c == '-');
            st.pop();
        }
    }
}
main() {
    int tt; cin >> tt;
    while (tt --> 0) solve();
}
```

</details>

<details>
<summary>Решение на Python3</summary>

```python
def solve():
    q = int(input())
    st = [10**10]
    for _ in range(q):
        query = input().split()
        c = query[0]
        if c == '+':
            # добавление элемента + обновление минимума:
            x = int(query[1])
            st.append(min(x, st[-1]))
        elif c == '?':
            # вывод минимума на экран:
            print(st[-1])
        else:
            # удаление последнего элемента:
            st.pop()

tt = int(input())
for _ in range(tt):
    solve()
```

</details>

<details>
<summary>Разбор решения на C++</summary>

Ключевые строки:

```cpp
st.push(inf);
...
st.push(min(x, st.top()));
```

Первая строка кладёт в стек фиктивную бесконечность. Она нужна затем, чтобы стек никогда не оказался пустым при первом добавлении и чтобы минимум можно было считать одной и той же формулой.

Во второй строке в стек кладётся не сам элемент $x$, а минимум между $x$ и предыдущей вершиной. Поэтому после каждого запроса $\text{+}$ на вершине лежит минимум среди всех текущих элементов стека. Именно поэтому запрос $\text{?}$ работает за $O(1)$, так как минимум всех элементов уже лежит на верхушке стека.

</details>

<details>
<summary>Разбор решения на Python3</summary>

Ключевые строки:

```python
st = [10**10]
...
st.append(min(x, st[-1]))
```

В списке `st` хранится не обычный стек значений, а стек минимумов. Первый элемент `10**10` играет роль фиктивной бесконечности, чтобы при первом добавлении можно было использовать ту же самую формулу, что и дальше.

Выражение `st[-1]` в Python обращается к последнему элементу списка, то есть к текущему минимуму. Поэтому строка `st.append(min(x, st[-1]))` кладёт наверх минимум среди нового числа и всех предыдущих элементов. После этого ответ на запрос `?` уже лежит в `st[-1]`.

</details>

## C. Парные скобки

<details>
<summary>Текстовый разбор</summary>

Будем идти по строке слева направо и хранить в стеке позиции открывающих скобок. Когда встречаем открывающую скобку, просто кладём её индекс в стек. Когда встречаем закрывающую, снимаем верхнюю позицию из стека: это и есть индекс парной ей открывающей скобки.

Гарантия правильности скобочной последовательности как раз означает, что в нужный момент наверху стека лежит именно та открывающая скобка, которая должна соединиться с текущей закрывающей. После этого можно записать ответ сразу для обеих позиций. Заранее подготовим массив для записи ответов, а потом выведем его содержимое. По времени решение работает за $O(n)$, по памяти – за $O(n)$.

</details>

<details>
<summary>Решение на C++</summary>

```cpp
#include <bits/stdc++.h>
using namespace std;
void solve() {
    int n; cin >> n;
    string s; cin >> s;
    stack<int> st;
    vector<int> answ(n);
    for (int i = 0; i < n; i++)
        if (s[i] == '(' || s[i] == '{' || s[i] == '[')
            st.push(i);
        else {
            // достаём последнюю открывающую скобку из стека:
            int j = st.top();
            st.pop();
            // элементы i <--> j образуют пару:
            answ[i] = j;
            answ[j] = i;
        }
    // вывод ответа в 1-индексации:
    for (int i = 0; i < n; i++)
        cout << answ[i]+1 << ' ';
    cout << '\n';
}
main() {
    int tt; cin >> tt;
    while (tt --> 0) solve();
}
```

</details>

<details>
<summary>Решение на Python3</summary>

```python
def solve():

    print()

for _ in range(tt):
    solve()
```

</details>

## D. Кто-то пролез в середину очереди...

<details>
<summary>Текстовый разбор</summary>

Здесь удобно разделить текущую очередь на две половины. В левой половине будем хранить первые элементы очереди, в правой – оставшиеся. Поддержим инвариант: размер левой половины либо равен размеру правой, либо больше на один (если элементов нечётное количество). Тогда первый элемент всей очереди всегда лежит в начале левой половины.

Рассмотрим операции. Запрос $\text{+}\ x$ добавляет число в конец, значит его можно положить в конец правой половины. Запрос $\text{*}\ x$ должен вставить число после первых $\left\lceil \frac{n}{2} \right\rceil$ элементов, а это как раз конец левой половины. Запрос $-$ снимает первый элемент левой половины. 

После каждого запроса остаётся только восстановить баланс двух половин: если размер правой половины больше размера левой половины, то перекидываем элемент из начала правой половины в конец левой, а если размер левой половины больше размера правой, то наоборот, из конца левой половины в начало правой.

По времени решение работает за $O(q)$, по памяти – за $O(q)$.

Альтернативное решение 1. Структура данных связный список (в C++ это `std::list<int>`) поддерживает вставку в любое место за $O(1)$, поэтому можно поддерживать указатель на середину этого списка, аналогично сдвигая его влево или вправо после каждого запроса, и вставлять новый элемент перед ним.

Альтернативное решение 2. Оказалось, что $200000$ вставок в середину `std::vector<int>` работают за $0.7$ секунд. Да, каждая вставка сдвигает всю половину элементов вправо, что в худшем случае работает за $O(q)$ на запрос и достигает итоговые $O(q^2)$ по времени, но константа у $O(q)$ слишком низкая, потому что современные процессоры двигают данные не по одному элементу, а блоками от $256$ до $512$ бит – то есть от $8$ до $16$ целых чисел за раз. Грубо говоря, получится что-то вроде $O(\tfrac{q^2}{16})$.

</details>

<details>
<summary>Решение на C++</summary>

```cpp
#include <bits/stdc++.h>
#include <cassert>
using namespace std;
void solve() {
    deque<int> left, right;
    int q; cin >> q;
    while (q --> 0) {
        char c; cin >> c;
        if (c == '+') {
            // кладём в конец очереди:
            int x; cin >> x;
            right.push_back(x);
        } else if (c == '*') {
            // кладём в середину очереди:
            int x; cin >> x;
            left.push_back(x);
        } else {
            assert(c == '-');
            // вынимаем элемент из головы очереди:
            cout << left.front() << '\n';
            left.pop_front();
        }
        // после выполнения запроса поддерживаем примерно равный размер половин:
        while (left.size() < right.size()) {
            left.push_back(right.front());
            right.pop_front();
        }
        while (left.size() > right.size()+1) {
            right.push_front(left.back());
            left.pop_back();
        }
    }
}
main() {
    int tt; cin >> tt;
    while (tt --> 0) solve();
}
```

</details>

<details>
<summary>Решение на Python3</summary>

```python
from collections import deque

def solve():
    left, right = deque(), deque()
    q = int(input())
    for _ in range(q):
        query = input().split()
        c = query[0]
        if c == '+': # кладём в конец очереди:
            x = int(query[1])
            right.append(x)
        elif c == '*': # кладём в середину очереди:
            x = int(query[1])
            left.append(x)
        else: # вынимаем элемент из головы очереди:
            print(left.popleft())
        # после выполнения запроса поддерживаем примерно равный размер половин:
        while len(left) < len(right):
            left.append(right.popleft())
        while len(left) > len(right) + 1:
            right.appendleft(left.pop())

tt = int(input())
for _ in range(tt):
    solve()
```

<summary>Разбор решения на C++</summary>

Разберём данный фрагмент:

Эти строки поддерживают инвариант размеров половин. После любого запроса левая половина должна содержать либо столько же элементов, сколько правая, либо на один больше (если элементов нечётное количество). Первый цикл перекидывает элементы из начала правой половины в конец левой половины, а второй цикл – из конца левой половины в начало правой половины.

</details>

<details>
<summary>Разбор решения на Python3</summary>

Разберём данный фрагмент:

```python

    left.append(right.popleft())
while len(left) > len(right) + 1:
    right.appendleft(left.pop())
```

Здесь поддерживается тот же инвариант, что и в решении на C++: в левой половине должно быть либо столько же элементов, сколько в правой, либо на один больше. Тогда первый элемент очереди всегда лежит в начале `left`, а вставка в середину попадает в конец `left`.

Первый цикл перекидывает элементы из начала правой половины в конец левой половины. Второй цикл делает обратное: переносит элементы из конца левой половины в начало правой половины. За счёт этого после каждого запроса структура снова приходит к правильному соотношению размеров.

Методы `popleft()` и `appendleft()` у `deque` позволяют за $O(1)$ брать элемент из начала дека и класть его в начало. Поэтому перебалансировка не ломает линейную асимптотику решения.

</details>

## E. Агрессивные элементы массива

<details>
<summary>Текстовый разбор</summary>

Посмотрим, что именно делает элемент $a_i$, когда становится агрессивным. Он будет побеждать соседей слева, пока не встретит первый элемент, строго больший него. То же самое происходит справа. Значит, для каждого индекса достаточно найти ближайший элемент слева, который строго больше $a_i$, и ближайший элемент справа, который строго больше $a_i$. Все элементы между ними и позицией $i$ будут побеждены.

Эти ближайшие большие элементы удобно искать монотонным стеком. Один проход слева направо даёт вклад от левой стороны, второй проход справа налево – вклад от правой. После этого остаётся сложить оба расстояния. По времени решение работает за $O(n)$, по памяти – за $O(n)$.

</details>

<details>
<summary>Решение на C++</summary>

```cpp
#include <bits/stdc++.h>
using namespace std;
const int inf = 1000000007;
void solve() {
    // читаем данные:
    int n; cin >> n;
    vector<int> a(n);
    for (int i = 0; i < n; i++)
        cin >> a[i];
    // идём слева-направо и находим расстояние до ближайшего > элемента слева:
    vector<int> answ(n);
    vector<pair<int,int>> st;
    st.push_back({inf, -1});
    for (int i = 0; i < n; i++) {
        while (st.back().first <= a[i])
            st.pop_back();
        answ[i] += (i - st.back().second - 1);
        st.push_back({a[i], i});
    }
    // теперь то же самое, но справа-налево:
    st = {{inf, n}};
    for (int i = n-1; i >= 0; i--) {
        while (st.back().first <= a[i])
            st.pop_back();
        answ[i] += (st.back().second - i - 1);
        st.push_back({a[i], i});
    }
    // выводим ответ:
    for (int i = 0; i < n; i++)
        cout << answ[i] << ' ';
    cout << '\n';
}
main() {
    int tt; cin >> tt;
    while (tt --> 0) solve();
}
```

</details>

<details>
<summary>Решение на Python3</summary>

```python
def solve():
    n = int(input())
    a = list(map(int, input().split()))
    # идём слева-направо и находим расстояние до ближайшего > элемента слева:
    answ = [0] * n
    st = [(10**10, -1)] # в стеке храним пару (число, индекс)
    for i in range(n):
        while st[-1][0] <= a[i]:
            st.pop()
        answ[i] = i - st[-1][1] - 1
        st.append((a[i], i))
    # теперь то же самое, но справа-налево:
    st = [(10**10, n)]
    for i in range(n - 1, -1, -1):
        while st[-1][0] <= a[i]:
            st.pop()
        answ[i] += st[-1][1] - i - 1
        st.append((a[i], i))
    # выводим ответ:
    for i in range(n):
        print(answ[i], end=' ')
    print()

tt = int(input())
for _ in range(tt):
    solve()
```

</details>

<details>
<summary>Разбор решения на C++</summary>

Разберём фрагмент первого прохода:

```cpp
while (st.back().first <= a[i])
    st.pop_back();
answ[i] += (i - st.back().second - 1);
st.push_back({a[i], i});
```

Стек хранит кандидатов на ближайший больший элемент слева в строго убывающем порядке. Пока на вершине лежит число, не большее `a[i]`, оно уже не сможет быть ответом ни для текущего элемента, ни для кого-то правее (его заместит `a[i]`), поэтому его можно убрать.

После этих удалений на вершине остаётся ближайший индекс слева, где стоит строго большее значение. Тогда выражение `i - st.back().second - 1` даёт количество элементов между этим барьером и позицией `i`, то есть ровно число побеждённых соседей слева. Второй проход справа налево устроен полностью аналогично.

</details>

<details>
<summary>Разбор решения на Python3</summary>

Разберём фрагмент первого прохода:

```python

    st.pop()
answ[i] = i - st[-1][1] - 1
st.append((a[i], i))
```

Список `st` используется как монотонный стек, а каждая его вершина хранит пару `(значение, индекс)`. Пока на вершине лежит число, не большее `a[i]`, оно уже не может быть ближайшим большим элементом слева, поэтому его можно удалить.

После этого в `st[-1]` остаётся нужный барьер слева. Его индекс берётся как `st[-1][1]`, и выражение `i - st[-1][1] - 1` сразу даёт количество элементов между этим барьером и текущей позицией. Проход справа налево работает по той же схеме.

</details>
