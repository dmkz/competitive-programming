## 22. Монотонный стек как олимпиадный шаблон

Знакомство с монотонным стеком начнём с разбора конкретного примера: для каждого элемента массива нужно найти **ближайший больший элемент слева**.

Пример массива:

```text
2 1 4 3 5 2
```

Если идти слева направо, в стеке удобно хранить **кандидатов, которые ещё могут стать ответом для будущих позиций**.

В этой задаче значения в стеке будут идти **строго по убыванию**:

- после `2` стек выглядит как `2`;
- после `1` стек выглядит как `2, 1`;
- после `4` стек превращается просто в `4`. Элементы `2, 1` удаляются, так как они больше никогда не станут ответом;
- после `3` стек выглядит как `4, 3`.

То есть стек хранит элементы не просто в порядке прихода, а в специальном упорядоченном порядке по значениям. Такой стек и называют монотонным.

---

## 23. Что означает, что элемент больше не нужен?

Посмотрим на тот же массив `2 1 4 3 5 2`.

Когда мы дошли до элемента `4`, в стеке лежат `2, 1`.

Теперь элемент `1` можно удалить:

- он стоит левее `4`;
- он меньше `4`;
- значит, для любого будущего элемента справа число `4` будет не хуже кандидата `1` и при этом ближе.

По той же причине можно удалить и `2`.

Именно это и означает фраза «элемент больше не нужен»: появился новый элемент, который для всех будущих ответов полностью вытеснил старого кандидата.

Поэтому, когда приходит `4`, стек `2, 1` превращается в стек `4`.

## 24. Строгое и нестрогое сравнение

Самая тонкая часть темы — правильно выбрать знак в условии удаления из стека.

Пусть мы снова ищем ответ слева и пришли к двум подряд идущим равным элементам:

```text
5 5
```

Если задача требует **строго больший** элемент слева, то для второго `5` первый `5` не подходит как ответ. Значит, равные элементы надо удалять, и условие в цикле будет таким:

```cpp
while (!st.empty() && a[st.back()] <= a[i]) {
	st.pop_back();
}
```

Если же задача требует **больший или равный** элемент слева, то первый `5` уже подходит для второго `5`. Значит, равные элементы удалять нельзя, и условие меняется на `<`.

Таким образом достаточно сделать знак в цикле строгим.

## 25. Что хранить: значения, индексы или пары

В олимпиадных задачах чаще всего удобнее хранить **индексы**, так как обычно требуют находить расстояния до ближайшего элемента.

Зная индекс, можем в любой момент найти значение элемента, если мы храним оригинальный массив.

Если задача спрашивает только о значениях, иногда хватает и самих значений, но индексы обычно универсальнее.

---

## 26. Пример: ближайший больший элемент слева

<details>
<summary><strong>Код на C++</strong></summary>

```cpp
#include <iostream>
#include <vector>
using namespace std;

int main() {
	int n;
	cin >> n;

	vector<int> a(n);
	for (int i = 0; i < n; i++) {
		cin >> a[i];
	}

	vector<int> st;
	for (int i = 0; i < n; i++) {
		while (!st.empty() && a[st.back()] <= a[i]) {
			st.pop_back();
		}

		if (st.empty()) {
			cout << -1 << ' ';
		} else {
			cout << a[st.back()] << ' ';
		}

		st.push_back(i);
	}
	cout << '\n';
}
```

</details>

<details>
<summary><strong>Код на Python3</strong></summary>

```python
n = int(input())
a = list(map(int, input().split()))

st = []
for i, x in enumerate(a):
	while st and a[st[-1]] <= x:
		st.pop()

	if st:
		print(a[st[-1]], end=' ')
	else:
		print(-1, end=' ')

	st.append(i)
print()
```

</details>

<details>
<summary><strong>Пример ввода и вывода</strong></summary>

Для ввода

```text
6
2 1 4 3 5 2
```

получим вывод

```text
-1 2 -1 4 -1 5
```

</details>

Хотя в ответе печатаются значения, в стеке всё равно удобнее хранить именно индексы.

## 27. Почему здесь снова получается `O(n)`?

Хотя внутри прохода обычно стоит `while`, общая сложность остаётся линейной.

Причина та же, что и раньше:

- каждый индекс добавляется в стек не более одного раза;
- каждый индекс удаляется из стека не более одного раза.

Поэтому суммарное число операций на стеке пропорционально длине массива.
