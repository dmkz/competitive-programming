## 8. Стек как рабочее состояние уже обработанного префикса

В олимпиадных задачах стек важен не сам по себе, а как способ хранить **активное состояние** после обработки очередного префикса последовательности:
- идём по массиву или строке слева направо;
- часть элементов уже полностью обработана;
- часть ещё ждёт дальнейшей судьбы;
- именно эта живая часть и лежит в стеке.

---

## 9. Что хранят в стеке?

В олимпиадном коде в стеке часто лежат не просто числа, а также:
- значения;
- их индексы;
- пары из значений и индексов;
- символы или позиции в строке.

Индексы особенно полезны, когда:

- нужно искать позиции элементов;
- одно и то же значение встречается несколько раз;
- позже понадобятся расстояния или ссылки на элементы исходного массива.

## 10. Пример: удаление соседних одинаковых символов

<details>
<summary><strong>Постановка задачи</strong></summary>

Дана строка. Пока в ней есть две одинаковые соседние буквы, такую пару нужно удалить.

Важно, что после удаления строка **схлопывается**: символы слева и справа от удалённой пары становятся соседними, поэтому может появиться новая пара для удаления.

Например, из строки `ABBACA` сначала удаляется `BB`, получается `AACA`, потом удаляется `AA`, и остаётся `CA`.

Нужно вывести строку после того, как больше никаких удалений сделать нельзя.

</details>

<details>
<summary><strong>C++: стек хранит живой хвост строки</strong></summary>

```cpp
#include <iostream>
#include <string>
#include <vector>
using namespace std;

int main() {
    string s;
    cin >> s;

    vector<char> st;
    for (char c : s) {
        if (!st.empty() && st.back() == c) {
            st.pop_back();
        } else {
            st.push_back(c);
        }
    }

    for (char c : st) {
        cout << c;
    }
    cout << '\n';
}
```

</details>

<details>
<summary><strong>Python3: тот же приём на list</strong></summary>

```python
s = input().strip()
st = []

for c in s:
    if st and st[-1] == c:
        st.pop()
    else:
        st.append(c)

print(''.join(st))
```

</details>

<details>
<summary><strong>Пример ввода и вывода</strong></summary>

Для ввода

```text
ABBACA
```

получим вывод

```text
CA
```

</details>

## 11. Пример: баланс скобок

<details>
<summary><strong>Постановка задачи</strong></summary>

Дана строка, состоящая только из символов `(` и `)`.

Нужно определить, является ли она правильной скобочной последовательностью.

Это означает:

- в любом префиксе строки число символов `)` не больше числа символов `(`;
- во всей строке число открывающих и закрывающих скобок совпадает.

Например, строка `(()())()` корректна, а строки `())(` и `(()` некорректны.

Нужно вывести `correct`, если последовательность правильная, и `incorrect` иначе.

</details>

<details>
<summary><strong>C++: в стеке лежат ещё не закрытые скобки</strong></summary>

```cpp
#include <iostream>
#include <string>
#include <vector>
using namespace std;

int main() {
    string s;
    cin >> s;

    vector<char> st;
    bool ok = true;

    for (char ch : s) {
        if (ch == '(') {
            st.push_back(ch);
        } else {
            if (st.empty()) {
                ok = false;
                break;
            }
            st.pop_back();
        }
    }

    if (!st.empty()) {
        ok = false;
    }

    cout << (ok ? "correct" : "incorrect") << '\n';
}
```

</details>

<details>
<summary><strong>Python3: тот же приём на list</strong></summary>

```python
s = input().strip()
stack = []
ok = True

for ch in s:
    if ch == '(':
        stack.append(ch)
    else:
        if not stack:
            ok = False
            break
        stack.pop()

if stack:
    ok = False

print("correct" if ok else "incorrect")
```

</details>

<details>
<summary><strong>Пример ввода и вывода</strong></summary>

Для ввода

```text
(()())()
```

получим вывод

```text
correct
```

</details>

Здесь в стеке лежат открывающие круглые скобки, для которых мы ещё не встретили закрывающую.

> [!NOTE]
> Для одного вида скобок здесь хватило бы и обычного счётчика.
> Но стек полезно показать явно, потому что именно так потом решаются более общие задачи на скобки и вложенные структуры.

## 12. Олимпиадные паттерны

В задачах спортивного программирования могут присутствовать маркеры, намекающие на необходимость в использовании стека:

- удаление соседних элементов по локальному правилу;
- поддержание цепочки ещё не закрытых объектов;
- хранение кандидатов, которых может вытеснить следующий элемент;
- поддержание монотонной структуры.

Во всех этих случаях стек хранит не всю историю, а только ту часть, которая ещё может повлиять на ответ.

---

## 13. Линейная сложность при использовании стека

Во многих задачах внутри прохода стоит цикл `while`, и начинающим кажется, что алгоритм квадратичный.

Но если каждый элемент:

- добавляется в стек не более одного раза;
- удаляется из стека не более одного раза,

то суммарное число стековых операций остаётся линейным.

> [!IMPORTANT]
> Сам по себе цикл `while` внутри прохода ещё не означает `O(n^2)`. Важно считать, сколько раз каждый элемент реально попадает в стек и покидает его.
