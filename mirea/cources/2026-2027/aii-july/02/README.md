# Лекция 2. Stack, queue, deque и линейная обработка последовательностей

## 1. Stack, queue и deque в олимпиадном программировании

В задачах на обработку последовательностей чаще всего встречаются три базовые структуры данных:

- `stack`;
- `queue`;
- `deque`.

Их полезно воспринимать не как абстрактные объекты из учебника, а как инструменты, которые поддерживают нужный набор операций:

- сначала смотрим, какие операции нужны;
- потом выбираем самую простую структуру, которая их поддерживает;
- если стандартный контейнер неудобен, берём более практичную реализацию той же идеи.

---

## 2. Стек

Стек поддерживает операции только на конце структуры:

- добавить элемент в конец;
- получить последний элемент;
- удалить последний элемент.

То есть стек работает по принципу **LIFO**: последним пришёл — первым ушёл.

Стек полезен, когда нужно:

- откатить последнее действие;
- хранить незавершённые объекты;
- идти слева направо по последовательности и держать только её активный хвост;
- поддерживать монотонную структуру.

### Что обычно используют в C++?

Формально можно взять `stack<T>`:

```cpp
stack<int> st;
```

Но в олимпиадном программировании очень часто используют обычный `vector<T>`:

```cpp
vector<int> st;
st.push_back(x);
st.back();
st.pop_back();
```

Такой стек делает всё то же самое, что и `stack<T>`, но дополнительно позволяет:

- получать доступ к элементам по индексу;
- удобно хранить индексы, пары и другие структуры;
- печатать содержимое;
- при необходимости делать бинарный поиск по монотонному стеку.

### Что обычно используют в Python3?

В Python3 роль стека почти всегда играет обычный `list`:

```python
st = []
st.append(x)
st[-1]
st.pop()
```

Это естественный и быстрый выбор: операции на конце списка работают эффективно.

> [!NOTE]
> В Python3 `list.pop()` сразу удаляет элемент и возвращает его.
> В C++ у `vector` чтение вершины и удаление разделены: обычно сначала делают `st.back()`, а потом `st.pop_back()`.

### Пример: читаем числа и выводим их в обратном порядке

<details>
<summary><strong>C++: стек на vector</strong></summary>

```cpp
#include <iostream>
#include <vector>
using namespace std;

int main() {
    int n;
    cin >> n;

    vector<int> st;
    for (int i = 0; i < n; i++) {
        int x;
        cin >> x;
        st.push_back(x);
    }

    while (!st.empty()) {
        cout << st.back() << ' ';
        st.pop_back();
    }
    cout << '\n';
}
```

</details>

<details>
<summary><strong>Python3: стек на list</strong></summary>

```python
n = int(input())
st = list(map(int, input().split()))

while st:
    print(st.pop(), end=' ')
print()
```

</details>

<details>
<summary><strong>Пример ввода и вывода</strong></summary>

Для ввода

```text
5
3 1 4 1 5
```

получим вывод

```text
5 1 4 1 3
```

</details>

Файлы примера: [C++](Part-01-Linear-structures/examples/example-01.cpp), [Python3](Part-01-Linear-structures/examples/example-01.py).

## 3. Очередь

Очередь поддерживает другой набор операций:

- добавить элемент в конец;
- получить первый элемент;
- удалить первый элемент.

Это уже принцип **FIFO**: первым пришёл — первым ушёл.

Очередь используют, когда нужно:

- моделировать обработку запросов в порядке их поступления;
- хранить текущее начало обрабатываемой последовательности;
- работать с процессом, где элементы покидают структуру в порядке прихода.

### Что используют в C++?

Самый прямой вариант — `queue<T>`:

```cpp
queue<int> q;
```

У него есть `push`, `front`, `pop`, `empty`, `size`.

Иногда очередь удобно написать руками на `vector`, если хранить индекс головы:

```cpp
vector<int> q;
int head = 0;
```

Тогда добавление идёт через `push_back`, а удаление из начала превращается в `head++`.

> [!NOTE]
> Приём с `head` полезен, когда кроме обычных операций очереди нужен ещё доступ к элементам по индексу.

### Что используют в Python3?

В Python3 для очереди почти всегда используют `collections.deque`, а не `list`.

```python
from collections import deque

q = deque()
q.append(x)
q[0]
q.popleft()
```

> [!NOTE]
> В Python3 `deque.popleft()` сразу и удаляет элемент, и возвращает его.
> В C++ у `queue<T>` чтение и удаление разделены: сначала `q.front()`, потом `q.pop()`.

> [!WARNING]
> В Python3 не стоит моделировать очередь через `list.pop(0)`: этот вызов сдвигает элементы и в худшем случае работает за `O(n)`.

> [!NOTE]
> `queue<T>` в C++ специально имеет узкий интерфейс: у него нет индексации и по нему нельзя напрямую пройтись по элементам, как по обычному контейнеру.
> `collections.deque` в Python3 в этом смысле гибче: его можно итерировать, а к краям обращаться через индексы `q[0]` и `q[-1]`.

### Пример: читаем числа и выводим их в том же порядке

<details>
<summary><strong>C++: очередь на queue</strong></summary>

```cpp
#include <iostream>
#include <queue>
using namespace std;

int main() {
    int n;
    cin >> n;

    queue<int> q;
    for (int i = 0; i < n; i++) {
        int x;
        cin >> x;
        q.push(x);
    }

    while (!q.empty()) {
        cout << q.front() << ' ';
        q.pop();
    }
    cout << '\n';
}
```

</details>

<details>
<summary><strong>Python3: очередь на deque</strong></summary>

```python
from collections import deque

n = int(input())
q = deque(map(int, input().split()))

while q:
    print(q.popleft(), end=' ')
print()
```

</details>

<details>
<summary><strong>Пример ввода и вывода</strong></summary>

Для ввода

```text
5
3 1 4 1 5
```

получим вывод

```text

```

</details>

## 4. Двунаправленная очередь

`deque` — это структура, у которой активно используются оба конца.

Она умеет эффективно:

- добавлять элемент в начало;
- добавлять элемент в конец;
- получать элемент из начала и из конца;
- удалять элемент из начала и из конца.

Это особенно полезно в задачах, где структура нужна сразу с двух сторон:

- очередь на минимум;
- монотонная очередь;
- поддержание активного отрезка;
- некоторые симуляции и специальные варианты поиска кратчайший путей.

### `deque` в C++

В C++ `deque<T>` поддерживает оба конца и индексируется:

```cpp
deque<int> d;
d.push_front(x);
d.push_back(y);
d.front();
d.back();
d[i];
```

> [!IMPORTANT]
> В C++ `deque` можно индексировать эффективно за `O(1)`.

### `deque` в Python3

В Python3 снова используется `collections.deque`:

```python
from collections import deque

d = deque()
d.append(x)
d.appendleft(y)
d.pop()
d.popleft()
```

У него тоже есть индексация, то есть можно написать `d[i]`, но лучше этого никогда не делать, потому что:

> [!WARNING]
> В Python3 у `collections.deque` доступ по произвольному индексу в худшем случае линейный по длине структуры — необходимо физически пройти все элементы, пока не найдём нужный нам.

> [!NOTE]
> В Python3 методы `d.pop()` и `d.popleft()` возвращают удалённый элемент.
> В C++ у `deque` чтение и удаление тоже разделены: `d.back()` и `d.front()` читают значение, а `pop_back()` и `pop_front()` только удаляют его.

Если нужен частый доступ к середине по индексам, обычно лучше взять `list`.

### Пример: кладём числа то в начало, то в конец

<details>
<summary><strong>C++: базовые операции deque</strong></summary>

```cpp
#include <deque>
#include <iostream>
using namespace std;

int main() {
    int n;
    cin >> n;

    deque<int> d;
    for (int i = 0; i < n; i++) {
        char side;
        int x;
        cin >> side >> x;

        if (side == 'L') {
            d.push_front(x);
        } else {
            d.push_back(x);
        }
    }

    for (int x : d) {
        cout << x << ' ';
    }
    cout << '\n';
}
```

</details>

<details>
<summary><strong>Python3: базовые операции с deque</strong></summary>

```python
from collections import deque

n = int(input())
d = deque()

for _ in range(n):
    side, value = input().split()
    x = int(value)

    if side == 'L':
        d.appendleft(x)
    else:
        d.append(x)

print(*d)
```

</details>

<details>
<summary><strong>Пример ввода и вывода</strong></summary>

Для ввода

```text
5
L 3
R 8
L 1
R 7
L 2
```

получим вывод

```text
2 1 3 8 7
```

</details>

## 5. Как быстро выбрать структуру

Выбор структуры начинается не с названия контейнера, а с набора операций, которые требует задача:

- нужен только конец структуры — стек;
- нужны добавление в конец и доступ к началу — очередь;
- нужны оба конца — `deque`;
- нужен частый доступ по индексу — массив или список.

---

## 6. Сложность основных операций

### C++

- `vector.push_back`, `vector.back`, `vector.pop_back` — `O(1)` амортизированно;
- `queue.push`, `queue.front`, `queue.pop` — `O(1)`;
- `deque.push_front`, `deque.push_back`, `deque.pop_front`, `deque.pop_back` — `O(1)`;
- `deque[i]` — `O(1)`.

### Python3

- `list.append`, чтение `list[-1]`, `list.pop()` — `O(1)` амортизированно;
- `deque.append`, `deque.appendleft`, `deque.pop`, `deque.popleft` — `O(1)`;
- `deque[i]` — в худшем случае `O(n)`.

## 8. Стек как рабочее состояние уже обработанного префикса

В олимпиадных задачах стек важен не сам по себе, а как способ хранить **активное состояние** после обработки очередного префикса последовательности:
- идём по массиву или строке слева направо;
- часть элементов уже полностью обработана;
- часть ещё ждёт дальнейшей судьбы;
- именно эта живая часть и лежит в стеке.

---

## 9. Что хранят в стеке?

В олимпиадном коде в стеке часто лежат не просто числа, а также:
- значения;
- их индексы;
- пары из значений и индексов;
- символы или позиции в строке.

Индексы особенно полезны, когда:

- нужно искать позиции элементов;
- одно и то же значение встречается несколько раз;
- позже понадобятся расстояния или ссылки на элементы исходного массива.

## 10. Пример: удаление соседних одинаковых символов

<details>
<summary><strong>Постановка задачи</strong></summary>

Дана строка. Пока в ней есть две одинаковые соседние буквы, такую пару нужно удалить.

Важно, что после удаления строка **схлопывается**: символы слева и справа от удалённой пары становятся соседними, поэтому может появиться новая пара для удаления.

Например, из строки `ABBACA` сначала удаляется `BB`, получается `AACA`, потом удаляется `AA`, и остаётся `CA`.

Нужно вывести строку после того, как больше никаких удалений сделать нельзя.

</details>

<details>
<summary><strong>Код на C++</strong></summary>

```cpp
#include <iostream>
#include <string>
#include <vector>
using namespace std;

int main() {
    string s;
    cin >> s;

    vector<char> st;
    for (char c : s) {
        if (!st.empty() && st.back() == c) {
            st.pop_back();
        } else {
            st.push_back(c);
        }
    }

    for (char c : st) {
        cout << c;
    }
    cout << '\n';
}
```

</details>

<details>
<summary><strong>Код на Python3</strong></summary>

```python
s = input().strip()
st = []

for c in s:
    if st and st[-1] == c:
        st.pop()
    else:
        st.append(c)

print(''.join(st))
```

</details>

<details>
<summary><strong>Пример ввода и вывода</strong></summary>

Для ввода

```text
ABBACA
```

получим вывод

```text
CA
```

</details>

## 11. Пример: баланс скобок

<details>
<summary><strong>Постановка задачи</strong></summary>

Дана строка, состоящая только из символов `(` и `)`.

Нужно определить, является ли она правильной скобочной последовательностью.

Это означает:

- в любом префиксе строки число символов `)` не больше числа символов `(`;
- во всей строке число открывающих и закрывающих скобок совпадает.

Например, строка `(()())()` корректна, а строки `())(` и `(()` некорректны.

Нужно вывести `correct`, если последовательность правильная, и `incorrect` иначе.

</details>

<details>
<summary><strong>C++: в стеке лежат ещё не закрытые скобки</strong></summary>

```cpp
#include <iostream>
#include <string>
#include <vector>
using namespace std;

int main() {
    string s;
    cin >> s;

    vector<char> st;
    bool ok = true;

    for (char ch : s) {
        if (ch == '(') {
            st.push_back(ch);
        } else {
            if (st.empty()) {
                ok = false;
                break;
            }
            st.pop_back();
        }
    }

    if (!st.empty()) {
        ok = false;
    }

    cout << (ok ? "correct" : "incorrect") << '\n';
}
```

</details>

<details>
<summary><strong>Python3: тот же приём на list</strong></summary>

```python
s = input().strip()
stack = []
ok = True

for ch in s:
    if ch == '(':
        stack.append(ch)
    else:
        if not stack:
            ok = False
            break
        stack.pop()

if stack:
    ok = False

print("correct" if ok else "incorrect")
```

</details>

<details>
<summary><strong>Пример ввода и вывода</strong></summary>

Для ввода

```text
(()())()
```

получим вывод

```text
correct
```

</details>

Здесь в стеке лежат открывающие круглые скобки, для которых мы ещё не встретили закрывающую.

> [!NOTE]
> Для одного вида скобок здесь хватило бы и обычного счётчика.
> Но стек полезно показать явно, потому что именно так потом решаются более общие задачи на скобки и вложенные структуры.

## 12. Олимпиадные паттерны

В задачах спортивного программирования могут присутствовать маркеры, намекающие на необходимость в использовании стека:

- удаление соседних элементов по локальному правилу;
- поддержание цепочки ещё не закрытых объектов;
- хранение кандидатов, которых может вытеснить следующий элемент;
- поддержание монотонной структуры.

Во всех этих случаях стек хранит не всю историю, а только ту часть, которая ещё может повлиять на ответ.

---

## 13. Линейная сложность при использовании стека

Во многих задачах внутри прохода стоит цикл `while`, и начинающим кажется, что алгоритм квадратичный.

Но если каждый элемент:

- добавляется в стек не более одного раза;
- удаляется из стека не более одного раза,

то суммарное число стековых операций остаётся линейным.

> [!IMPORTANT]
> Сам по себе цикл `while` внутри прохода ещё не означает `O(n^2)`. Важно считать, сколько раз каждый элемент реально попадает в стек и покидает его.

## 15. Очередь на минимум

Одна из самых интересных олимпиадных структур на `deque` — это **очередь на минимум**.

Поддерживаемые операции:

- добавить элемент в конец;
- удалить элемент из начала;
- быстро получить минимум среди всех текущих элементов.

Обычная очередь такие запросы на минимум эффективно не поддерживает, поэтому нужна дополнительная идея.

---

## 16. Хранение кандидатов на минимум

Вместо всех элементов подряд удобно хранить только **кандидатов на минимум**.

Если удобно обращаться к значениям через исходный массив, в деке хранят индексы.
Если хочется держать всё в одной структуре, можно хранить пары `(значение, позиция)`.

Кандидаты в деке поддерживаются в неубывающем порядке значений.

Это означает:

- минимум всегда лежит в начале дека;
- элементы, которые заведомо хуже нового, можно удалить с конца.

---

## 17. Добавление и удаление

Пусть мы добавляем новый элемент `x` с позицией `pos`.

Пока в конце дека лежат элементы со значением не меньше `x`, их можно удалить:

- они стоят левее `x`;
- они не меньше `x`;
- после появления `x` они уже никогда не станут минимумом раньше него.

После этого в конец дека добавляется новый кандидат.

Когда из очереди уходит элемент с позицией `left_pos`, нужно проверить начало дека. Если там стоит кандидат, который уже покинул очередь, его тоже надо удалить.

> [!IMPORTANT]
> Если в деке хранятся пары `(значение, позиция)`, то при добавлении нового `x` обычно выталкивают с конца все элементы `>= x`, а не только `> x`.

## 18. Пример: обработка запросов к очереди на минимум

<details>
<summary><strong>Постановка задачи</strong></summary>

Нужно обработать запросы к очереди:

- `+ x` — добавить число `x` в конец;
- `-` — удалить элемент из начала;
- `?` — вывести минимум среди всех элементов очереди.

Гарантируется, что запросы `-` и `?` применяются только к непустой очереди.

</details>

В этом учебном варианте будем хранить только дек кандидатов на минимум.

У каждого кандидата будем помнить пару:

- значение;
- позицию этого значения в порядке добавления в очередь.

Отдельно будем хранить:

- `left_pos` — позицию текущего левого элемента очереди;
- `right_pos` — позицию, которую получит следующий добавленный элемент.

> [!NOTE]
> Позиция нужна, потому что самый левый элемент очереди не обязан стоять в начале дека минимумов.
> Поэтому при запросе `-` мы удаляем голову дека только если её позиция совпадает с `left_pos`.

<details>
<summary><strong>Код на C++</strong></summary>

```cpp
#include <deque>
#include <iostream>
#include <utility>
using namespace std;

int main() {
    int q;
    cin >> q;

    deque<pair<int, int>> mins;
    int left_pos = 0;
    int right_pos = 0;

    for (int i = 0; i < q; i++) {
        char op;
        cin >> op;

        if (op == '+') {
            int x;
            cin >> x;

            while (!mins.empty() && mins.back().first >= x) {
                mins.pop_back();
            }
            mins.push_back({x, right_pos});
            right_pos++;
        } else if (op == '-') {
            if (!mins.empty() && mins.front().second == left_pos) {
                mins.pop_front();
            }
            left_pos++;
        } else {
            cout << mins.front().first << '\n';
        }
    }
}
```

</details>

<details>
<summary><strong>Код на Python3</strong></summary>

```python
from collections import deque

q = int(input())
mins = deque()
left_pos = 0
right_pos = 0

for _ in range(q):
    parts = input().split()
    op = parts[0]

    if op == '+':
        x = int(parts[1])

        while mins and mins[-1][0] >= x:
            mins.pop()
        mins.append((x, right_pos))
        right_pos += 1
    elif op == '-':
        if mins and mins[0][1] == left_pos:
            mins.popleft()
        left_pos += 1
    else:
        print(mins[0][0])
```

</details>

Здесь в деке лежат только кандидаты на минимум, а не все элементы очереди. Каждый кандидат хранится как пара `(значение, позиция)`.

## 19. Почему это работает за линейное время?

Хотя при добавлении стоит цикл `while`, общая сложность остаётся линейной:

- каждый элемент добавляется в дек не более одного раза;
- каждый элемент удаляется из дека не более одного раза.

Значит, суммарное число операций с деком равно `O(n)`.

---

## 20. Использование очереди на минимум

Очередь на минимум полезна, когда:

- левая граница структуры сдвигается только вправо;
- новые элементы добавляются только справа;
- в каждый момент нужен минимум по текущему активному набору.

Это отдельный полезный шаблон, который часто возникает в задачах на скользящие окна, два указателя и линейную обработку массива.

## 22. Монотонный стек как олимпиадный шаблон

Знакомство с монотонным стеком начнём с разбора конкретного примера: для каждого элемента массива нужно найти **ближайший больший элемент слева**.

Пример массива:

```text
2 1 4 3 5 2
```

Если идти слева направо, в стеке удобно хранить **кандидатов, которые ещё могут стать ответом для будущих позиций**.

В этой задаче значения в стеке будут идти **строго по убыванию**:

- после `2` стек выглядит как `2`;
- после `1` стек выглядит как `2, 1`;
- после `4` стек превращается просто в `4`. Элементы `2, 1` удаляются, так как они больше никогда не станут ответом;
- после `3` стек выглядит как `4, 3`.

То есть стек хранит элементы не просто в порядке прихода, а в специальном упорядоченном порядке по значениям. Такой стек и называют монотонным.

---

## 23. Что означает, что элемент больше не нужен?

Посмотрим на тот же массив `2 1 4 3 5 2`.

Когда мы дошли до элемента `4`, в стеке лежат `2, 1`.

Теперь элемент `1` можно удалить:

- он стоит левее `4`;
- он меньше `4`;
- значит, для любого будущего элемента справа число `4` будет не хуже кандидата `1` и при этом ближе.

По той же причине можно удалить и `2`.

Именно это и означает фраза «элемент больше не нужен»: появился новый элемент, который для всех будущих ответов полностью вытеснил старого кандидата.

Поэтому, когда приходит `4`, стек `2, 1` превращается в стек `4`.

## 24. Строгое и нестрогое сравнение

Самая тонкая часть темы — правильно выбрать знак в условии удаления из стека.

Пусть мы снова ищем ответ слева и пришли к двум подряд идущим равным элементам:

```text
5 5
```

Если задача требует **строго больший** элемент слева, то для второго `5` первый `5` не подходит как ответ. Значит, равные элементы надо удалять, и условие в цикле будет таким:

```cpp
while (!st.empty() && a[st.back()] <= a[i]) {
    st.pop_back();
}
```

Если же задача требует **больший или равный** элемент слева, то первый `5` уже подходит для второго `5`. Значит, равные элементы удалять нельзя, и условие меняется на `<`.

Таким образом достаточно сделать знак в цикле строгим.

## 25. Что хранить: значения, индексы или пары

В олимпиадных задачах чаще всего удобнее хранить **индексы**, так как обычно требуют находить расстояния до ближайшего элемента.

Зная индекс, можем в любой момент найти значение элемента, если мы храним оригинальный массив.

Если задача спрашивает только о значениях, иногда хватает и самих значений, но индексы обычно универсальнее.

---

## 26. Пример: ближайший больший элемент слева

<details>
<summary><strong>Код на C++</strong></summary>

```cpp
#include <iostream>
#include <vector>
using namespace std;

int main() {
    int n;
    cin >> n;

    vector<int> a(n);
    for (int i = 0; i < n; i++) {
        cin >> a[i];
    }

    vector<int> st;
    for (int i = 0; i < n; i++) {
        while (!st.empty() && a[st.back()] <= a[i]) {
            st.pop_back();
        }

        if (st.empty()) {
            cout << -1 << ' ';
        } else {
            cout << a[st.back()] << ' ';
        }

        st.push_back(i);
    }
    cout << '\n';
}
```

</details>

<details>
<summary><strong>Код на Python3</strong></summary>

```python
n = int(input())
a = list(map(int, input().split()))

st = []
for i, x in enumerate(a):
    while st and a[st[-1]] <= x:
        st.pop()

    if st:
        print(a[st[-1]], end=' ')
    else:
        print(-1, end=' ')

    st.append(i)
print()
```

## 27. Почему здесь снова получается `O(n)`?

Хотя внутри прохода обычно стоит `while`, общая сложность остаётся линейной.

Причина та же, что и раньше:

- каждый индекс добавляется в стек не более одного раза;
- каждый индекс удаляется из стека не более одного раза.

Поэтому суммарное число операций на стеке пропорционально длине массива.
