# Разбор задач

## A. Второй максимум

<details>
<summary>Текстовый разбор</summary>

Решим задачу за два прохода по массиву. На первом проходе найдём наибольшее значение массива, обозначим его как $\text{max1}$. После этого на втором проходе рассматриваем только те элементы, которые строго меньше $\text{max1}$. Среди них выберем наибольший. Это и будет второй максимум, обозначим его как $\text{max2}$.

По времени решение работает за $O(n)$, потому что делает два линейных прохода по массиву. По памяти используется $O(n)$, так как массив хранится целиком.

Можно решить и за $O(1)$ по памяти, поддерживая два максимума и обновляя их одновременно с чтением элементов, но оставим это для собеседований на работу, а не для олимпиад.

</details>

<details>
<summary>Решение на C++</summary>

```cpp
#include <bits/stdc++.h>
using namespace std;
void solve() {
    // читаем массив:
    int n;
    cin >> n;
    vector<int> a(n);
    for (int i = 0; i < n; i++)
        cin >> a[i];
    // первый максимум:
    const int max1 = *max_element(a.begin(), a.end());
    // второй максимум:
    int max2 = -1000000000;
    for (int i = 0; i < n; i++)
        if (max2 < a[i] && a[i] < max1)
            max2 = a[i];
    cout << max2 << endl;
}
int main() {
    int tt; cin >> tt;
    while (tt --> 0) solve();
}
```

</details>

<details>
<summary>Решение на Python3</summary>


</details>

## B. Чётная сортировка

<details>
<summary>Текстовый разбор</summary>

Разделим элементы массива на две группы: элементы на нечётных позициях и на чётных. Первую группу отсортируем по убыванию, вторую — по возрастанию. После этого соберём ответ и выведем его. При выводе по очереди берём элементы из этих двух групп.

Для удобства решения для каждой группы создадим свой массив, добавляя элементы в конец и сортируя их. Не стоит изощряться и делать всё в одном массиве.

По времени решение работает за $O\left(n \cdot \log \left(n\right)\right)$ из-за двух сортировок. По памяти используется $O(n)$.

</details>

<details>
<summary>Решение на C++</summary>

```cpp
#include <bits/stdc++.h>
using namespace std;
void solve() {
    // читаем данные и распределяем их в два вектора: нечётные и чётные
    int n; cin >> n;
    vector<int> odd, even;
    for (int i = 1; i <= n; i++) {
        int x; cin >> x;
        if (i % 2 == 1) odd.push_back(x);
        else even.push_back(x);
    }
    // сортируем оба вектора:
    sort(odd.begin(), odd.end(), greater<int>());
    sort(even.begin(), even.end());
    // чередуя, выводим ответ:
    int i = 0, j = 0;
    while (i < odd.size()) {
        // нечётный элемент:
        cout << odd[i] << ' ';
        i++;
        // чётный элемент:
        if (j < even.size()) {
            cout << even[j] << ' ';
            j++;
        }
    }
    cout << '\n';
}
int main() {
    int tt; cin >> tt;
    while (tt --> 0) solve();
}
```

</details>

<details>
<summary>Решение на Python3</summary>

```python
def solve():
    n = int(input())
    a = list(map(int, input().split()))
    odd = [] # нечётные элементы
    even = [] # чётные элементы
    for i in range(1, n + 1):
        x = a[i - 1]
        if i % 2 == 1:
            odd.append(x)
        else:
            even.append(x)
    # сортируем оба вектора:
    odd.sort(reverse=True)
    even.sort()
    # чередуя, выводим ответ:
    i, j = 0, 0
    while i < len(odd):
        # нечётный элемент:
        print(odd[i], end=' ')
        i += 1
        # чётный элемент:
        if j < len(even):
            print(even[j], end=' ')
            j += 1
    print()

tt = int(input())
for _ in range(tt):
    solve()
```

</details>

<details>
<summary>Разбор решения на C++</summary>

Здесь стоит отдельно посмотреть на строки:

```cpp
sort(odd.begin(), odd.end(), greater<int>());
sort(even.begin(), even.end());
```

Функция `sort(begin, end)` сортирует весь диапазон от начала контейнера до его конца. В первой строке передан третий аргумент `greater<int>()`. Он задаёт правило сравнения: большее число должно идти раньше меньшего. Поэтому вектор `odd` будет отсортирован по убыванию.

Во второй строке третьего аргумента нет, поэтому используется обычный порядок сравнения. Значит, вектор `even` будет отсортирован по возрастанию.

</details>

## C. Запросы к массиву

<details>
<summary>Текстовый разбор</summary>

Будем хранить текущий массив и обрабатывать запросы по одному. Для запроса $\text{push}\ x$ добавим число в конец и выведем новую длину массива. Для запроса $\text{pop}$ возьмём последний элемент, удалим его и выведем это значение.

А теперь разберём запрос $\text{get}\ i$. Если текущий массив равен $a_1, a_2, \ldots, a_n$, то дальше он будет повторяться по кругу. Значит, нас будет интересовать не сама позиция $i$, а её остаток по модулю длины массива. Из-за нумерации с единицы нужный индекс будет равен $(i - 1) \bmod n$. Берём элемент по найденной позиции и выводим его.

Оценим асимптотику решения. Все запросы обрабатываются по одному, и после каждого запроса ответ печатается сразу. По времени решение работает за $O(q)$, потому что каждый запрос обрабатывается за $O(1)$. По памяти используется $O(q)$, так как в худшем случае в массиве может накопиться до $q$ элементов.

</details>

<details>
<summary>Решение на C++</summary>

```cpp
#include <bits/stdc++.h>
#include <cassert>
using namespace std;
void solve() {
    int q; cin >> q;
    vector<int> a;
    while (q --> 0) {
        string s; cin >> s;
        if (s == "push") {
            // добавление элемента в конец:
            int x;
            cin >> x;
            a.push_back(x);
            cout << a.size() << '\n';
        } else if (s == "pop") {
            // удаление элемента из конца:
            int x = a.back();
            a.pop_back();
            cout << x << '\n';
        } else {
            // значение i-го элемента
            assert(s == "get");
            long long i;
            cin >> i;
            cout << a[(i-1) % a.size()] << '\n';
        }
    }
}
int main() {
    int tt; cin >> tt;
    while (tt --> 0) solve();
}
```

</details>

<details>
<summary>Решение на Python3</summary>

```python
def solve():
    q = int(input())
    a = []
    while q > 0:
        q -= 1
        s = input().split()
        if s[0] == "push":
            x = int(s[1])
            a.append(x)
            print(len(a))
        elif s[0] == "pop":
            x = a[-1]
            a.pop()
            print(x)
        else:
            assert s[0] == "get"
            i = int(s[1])
            print(a[(i - 1) % len(a)])

tt = int(input())
for _ in range(tt):
    solve()
```

</details>

<details>
<summary>Разбор решения на C++</summary>

Здесь есть два места, на которые стоит посмотреть отдельно. Иногда спортивным программистам становится скучно, и они начинают писать что-то вроде такого:

```cpp
while (q --> 0) {
```

На вид это похоже на новую операцию «стремления», но её здесь нет. Выражение читается как `(q-- > 0)`. Сначала текущее значение $q$ сравнивается с нулём, а потом $q$ уменьшается на единицу. Поэтому цикл выполнится ровно $q$ раз.

```cpp
cout << a[(i-1) % a.size()] << '\n';
```

Здесь программа отвечает на запрос $\text{get}\ i$. Выражение `(i - 1)` переводит нумерацию из условия к нумерации с нуля. Остаток по модулю `a.size()` показывает, на какой позиции внутри текущего массива окажется $i$-й элемент бесконечного повторения. После этого остаётся взять элемент массива по найденному индексу.

</details>

<details>
<summary>Разбор решения на Python3</summary>

Ключевая строка:

```python
print(a[(i - 1) % len(a)])
```

Список `a` хранит только текущий массив, без бесконечного повторения. Поэтому программа не будет строить длинную последовательность явно. Вместо этого она посчитает, куда попадёт позиция $i$ внутри одного блока длины `len(a)`.

Выражение `(i - 1)` снова переводит нумерацию к индексам Python, которые начинаются с нуля. Остаток по модулю `len(a)` даёт индекс нужного элемента в текущем списке.

</details>

## D. Сколько меньших?

<details>
<summary>Текстовый разбор</summary>

Скопируем массив и отсортируем его копию. После этого для каждого элемента $a_i$ будем искать в отсортированной копии позицию первого элемента, который не меньше $a_i$. Именно столько элементов будут стоять левее него в отсортированном порядке, а значит, именно столько элементов будут строго меньше $a_i$.

По времени решение работает за $O\left(n \cdot \log \left(n\right)\right)$: $O\left(n \cdot \log \left(n\right)\right)$ занимает сортировка, и ещё $n$ бинарных поисков по $O\left(\log \left(n\right)\right)$. По памяти используется $O(n)$, потому что создаётся отсортированная копия массива.

Альтернативное решение: создать массив пар `(a[i], i)` и отсортировать их лексикографически. Тогда, сохранив оригинальную нумерацию и зная индекс элемента в отсортированном массиве, мы можем найти ответ без бинарного поиска.

Но у альтернативного решения есть недостаток: требуется аккуратная обработка блоков из повторяющихся элементов, поэтому рекомендуется первое решение.

</details>

<details>
<summary>Решение на C++</summary>

```cpp
#include <bits/stdc++.h>
using namespace std;
void solve() {
    // читаем вектор
    int n; cin >> n;
    vector<int> a(n);
    for (int i = 0; i < n; i++)
        cin >> a[i];
    // копируем и сортируем копию:
    vector<int> b = a;
    sort(b.begin(), b.end());
    // выводим ответ, считая количество меньших элементов в отсортированной копии:
    for (int i = 0; i < n; i++)
        cout << int(lower_bound(b.begin(), b.end(), a[i]) - b.begin()) << ' ';
    cout << '\n';
}
int main() {
    int tt; cin >> tt;
    while (tt --> 0) solve();
    return 0;
}
```

</details>

<details>
<summary>Решение на Python3</summary>

```python
from bisect import bisect_left

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    b = sorted(a)
    for i in range(n):
        print(bisect_left(b, a[i]), end=' ')
    print()

tt = int(input())
for _ in range(tt):
    solve()
```

</details>

<details>
<summary>Разбор решения на C++</summary>

Разберём выражение:

```cpp
cout << int(lower_bound(b.begin(), b.end(), a[i]) - b.begin()) << ' ';
```

Функция `lower_bound(b.begin(), b.end(), a[i])` ищет в отсортированном массиве `b` первое место, где стоит элемент, не меньший `a[i]`. Результат возвращается не в виде числа, а в виде итератора.

Разность `lower_bound(...) - b.begin()` означает расстояние от найденного элемента до начала массива. Она превращает этот итератор в индекс. Этот индекс и будет количеством элементов, которые стоят раньше, то есть количеством элементов, строго меньших `a[i]`.

Преобразование `int(...)` здесь нужно в учебных целях только затем, чтобы явно вывести обычное целое число.

</details>

<details>
<summary>Разбор решения на Python3</summary>

Разберём:

```python
print(bisect_left(b, a[i]), end=' ')
```

Функция `bisect_left(b, a[i])` ищет в отсортированном списке `b` позицию первого элемента, который не меньше `a[i]`. В $0$-индексации это и будет количеством элементов, стоящих левее, а значит, и количеством элементов, строго меньших `a[i]`.

Поэтому результат `bisect_left` можно сразу печатать как ответ для текущего элемента.

</details>

## E. Пары с ограниченной суммой

<details>
<summary>Текстовый разбор</summary>

Отсортируем массив и посмотрим на пары, которые нам нужно посчитать. Основное условие: $a_i + a_j \le s$. Упростим его.

Зафиксируем значение $a_j$ и переберём в цикле по $j$. Теперь мы знаем $a_j$ и можем перенести его в правую часть неравенства: $a_i \le s - a_j$. Получили, что для $a_j$ нам нужно считать, сколько значений на префиксе $[0, j)$ массива удовлетворяют неравенству $a_i \le s - a_j$.

Так как префикс уже отсортирован, это количество найдём бинарным поиском и сразу прибавим к ответу. Каждая подходящая пара будет посчитана ровно один раз (т.к. мы обеспечили $i < j$).

По времени решение работает за $O\left(n \cdot \log \left(n\right)\right)$: одна сортировка и затем $n$ бинарных поисков. По памяти используется $O(n)$, так как массив хранится целиком.

</details>

<details>
<summary>Решение на C++</summary>

```cpp
#include <bits/stdc++.h>
using namespace std;
void solve() {
    // читаем данные (сразу в 64-битный тип, чтобы не переполниться)
    int64_t n, s;
    cin >> n >> s;
    vector<int64_t> a(n);
    for (int i = 0; i < n; i++)
        cin >> a[i];
    sort(a.begin(), a.end());
    int64_t answ = 0;
    for (int j = 0; j < n; j++)
        answ += int(upper_bound(a.begin(), a.begin() + j, s - a[j]) - a.begin());
    cout << answ << '\n';
}
int main() {
    int tt; cin >> tt;
    while (tt --> 0) solve();
    return 0;
}
```

</details>

<details>
<summary>Решение на Python3</summary>

```python
from bisect import bisect_right

def solve():
    n, s = map(int, input().split())
    a = list(map(int, input().split()))
    a.sort()
    answ = 0
    for j in range(n):
        answ += bisect_right(a, s - a[j], 0, j)
    print(answ)

tt = int(input())
for _ in range(tt):
    solve()
```

</details>

<details>
<summary>Разбор решения на C++</summary>

Самая важная строка в решении:

```cpp
answ += int(upper_bound(a.begin(), a.begin() + j, s - a[j]) - a.begin());
```

Выражение `s - a[j]` задаёт ограничение на `a[i]` — левый элемент пары, чтобы сумма не превысила $s$. Поиск выполняется не по всему массиву, а только на префиксе от `a.begin()` до `a.begin() + j`. Это и есть все кандидаты с индексами меньше $j$.

Функция `upper_bound` возвращает первую позицию элемента, который не подходит под ограничение — его значение уже больше `s - a[j]`. Значит, все элементы раньше этой позиции подходят. Разность с `a.begin()` превращает результат в количество подходящих элементов, и это количество сразу добавляется к `answ`.

</details>

<details>
<summary>Разбор решения на Python3</summary>

Разберём строку:

```python
answ += bisect_right(a, s - a[j], 0, j)
```

Первый аргумент — это сам отсортированный список `a`. Второй аргумент `s - a[j]` задаёт ограничение для `a[i]` — левого элемента пары. Третий и четвёртый аргументы, `0` и `j`, говорят функции, что искать нужно только на префиксе списка, то есть среди элементов с индексами меньше $j$.

Функция `bisect_right` вернёт первую позицию элемента, который уже больше этого ограничения. Поэтому возвращённое число сразу совпадает с количеством подходящих элементов слева, и его можно прибавить к `answ`.

</details>
