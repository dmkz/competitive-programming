Даже корректный перебор часто упирается не в логику, а во время работы. Тогда главным инструментом становятся отсечения: нужно как можно раньше понять, что в текущую ветку углубляться бессмысленно.

## 11. Почему без отсечений перебор быстро умирает

Если из каждой вершины выходит по три ветви и глубина равна $20$, получаем уже порядка $3^{20}$ состояний. Для перестановок рост ещё жёстче: число вариантов примерно равно $n!$.

Поэтому в реальной задаче мало написать корректный генератор. Нужно ещё сделать так, чтобы он не ходил по заведомо плохим веткам.

Хорошее отсечение — это правило вида: «из этого состояния ответа уже не получится» или «даже лучший возможный добор нас не спасёт». Как только такое правило найдено, ветку можно закрывать сразу, не дожидаясь листа.

Возьмём уже знакомый объект из первых двух частей: двоичные строки длины $12$. Но теперь хотим оставить только те строки, в которых нет двух единиц подряд.

Теперь у нас есть условие на префикс: если последний символ уже равен `1`, то ещё одну `1` дописывать нельзя.

Без отсечения мы всё равно честно идём в обе ветви на каждом шаге и строим полное двоичное дерево глубины $12$. Тогда будет посещено $2^{13} - 1 = 8191$ состояний.

<details>
<summary>Без отсечения</summary>

```cpp
void dfs(int pos) {
	visited++;
	if (pos == n) return;

	current += '0';
	dfs(pos + 1);
	current.pop_back();

	current += '1';
	dfs(pos + 1);
	current.pop_back();
}
```

```python
def dfs(pos: int) -> None:
	global visited
	visited += 1
	if pos == n:
		return

	current.append('0')
	dfs(pos + 1)
	current.pop()

	current.append('1')
	dfs(pos + 1)
	current.pop()
```

</details>

С отсечением проверка становится очень простой: если текущий префикс уже заканчивается на `1`, то ветку с ещё одной `1` мы даже не создаём.

<details>
<summary>С отсечением</summary>

```cpp
void dfs(int pos) {
	visited++;
	if (pos == n) return;

	current += '0';
	dfs(pos + 1);
	current.pop_back();

	if (current.empty() || current.back() != '1') {
		current += '1';
		dfs(pos + 1);
		current.pop_back();
	}
}
```

```python
def dfs(pos: int) -> None:
	global visited
	visited += 1
	if pos == n:
		return

	current.append('0')
	dfs(pos + 1)
	current.pop()

	if not current or current[-1] != '1':
		current.append('1')
		dfs(pos + 1)
		current.pop()
```

</details>

На этой же задаче получится всего `985` посещённых состояний вместо `8191`, то есть примерно в $8191 / 985 \approx 8.3$ раза меньше.

Здесь хорошо видно, за счёт чего появляется выигрыш:

- после префикса `01` ветка `011...` даже не создаётся;
- после префикса `1010` ветка `10101...` ещё разрешена, а ветка `101011...` уже будет запрещена на следующем шаге.

Полные программы для генерации двоичных строк без двух единиц подряд: [C++](examples/example-01.cpp), [Python3](examples/example-01.py).

## 12. Типовые отсечения

Отсечения чаще всего появляются из нескольких стандартных источников:

- текущий префикс уже нарушил ограничение;

> [!TIP]
> Пример. Если мы строим строку длины `n` и запрещаем две единицы подряд, то префикс `...11` уже нарушил ограничение. У такой ветки не бывает хорошего продолжения, поэтому её надо обрывать сразу.

- даже в самом оптимистичном продолжении мы не улучшим текущий ответ;

> [!TIP]
> Пример. Пусть мы строим строку длины `6` из цифр `0`, `1`, `2` и хотим максимизировать сумму цифр, но не превышать `8`. Лучший найденный ответ уже равен `8`. В текущем префиксе сумма пока `3`, а позиций осталось `2`. Даже если в обе позиции поставить `2`, итоговая сумма будет только `7`, так что эту ветку уже нет смысла продолжать.

- такое состояние уже встречалось раньше;

> [!TIP]
> Пример. Пусть мы считаем число строк длины `n` без двух единиц подряд. Префиксы `010` и `100` разные, но оба приводят к одному и тому же состоянию: уже обработано `3` позиции, а последний символ равен `0`. Значит, число продолжений дальше у них одинаковое, и второй раз этот хвост можно не пересчитывать.

- порядок перебора можно выбрать так, чтобы плохие ветви погибали раньше.

> [!TIP]
> Пример. В той же задаче мы хотим получить какую-нибудь строку, у которой сумма цифр не больше `8`. Поэтому часто выгодно пробовать цифры в порядке `2`, `1`, `0`, а не `0`, `1`, `2`. Тогда ветви со слишком большой суммой цифр погибают раньше, и сильный текущий ответ находится раньше.

## 13. Культура отката и классические сюжеты перебора с возвратом

Термин «перебор с возвратом» обычно используют там, где мы меняем состояние, углубляемся, а потом обязаны аккуратно вернуть всё назад.

Этот шаблон встречается во многих классических задачах:

- построение перестановки с локальными запретами;
- обход доски без повторов;
- разбиение объектов по группам;
- построение раскладки предметов по ячейкам с локальными запретами.

Возьмём новый простой пример, который хорошо показывает саму механику отката: нужно посчитать количество перестановок чисел от `1` до `n`, в которых на позиции `i` нельзя ставить число `i`.

Например, при `n = 4` перестановка `2 1 4 3` подходит, потому что ни одно число не стоит на своём месте. А перестановка `2 1 3 4` уже не подходит: число `3` осталось на позиции `3`, а число `4` — на позиции `4`.

Состояние здесь почти такое же, как в обычном переборе перестановок:

- `current` хранит уже построенный префикс;
- `used[]` помнит, какие числа уже заняты.

Когда мы заполняем позицию `pos`, нельзя брать число `pos + 1`. Все остальные ещё не использованные числа можно пробовать. После рекурсивного вызова обязательно делаем откат: убираем последнее число из `current` и снимаем пометку из `used[]`.

<details>
<summary>Пример на C++</summary>

```cpp
void dfs(int pos) {
	// Если все позиции заполнены, нашли одну подходящую перестановку.
	if (pos == n) {
		ways++;
		return;
	}

	// Пробуем поставить на позицию pos любое ещё не использованное число.
	for (int value = 1; value <= n; value++) {
		if (used[value - 1]) continue;

		// Число value нельзя ставить на его собственную позицию.
		if (value == pos + 1) continue;

		used[value - 1] = 1;
		current.push_back(value);
		dfs(pos + 1);
		// После возврата обязательно откатываем оба изменения.
		current.pop_back();
		used[value - 1] = 0;
	}
}
```

</details>

<details>
<summary>Пример на Python3</summary>

```python
def dfs(pos: int) -> None:
	global ways
	# Если все позиции заполнены, нашли одну подходящую перестановку.
	if pos == n:
		ways += 1
		return

	# Пробуем поставить на позицию pos любое ещё не использованное число.
	for value in range(1, n + 1):
		if used[value - 1]:
			continue

		# Число value нельзя ставить на его собственную позицию.
		if value == pos + 1:
			continue

		used[value - 1] = True
		current.append(value)
		dfs(pos + 1)
		# После возврата обязательно откатываем оба изменения.
		current.pop()
		used[value - 1] = False
```

</details>
