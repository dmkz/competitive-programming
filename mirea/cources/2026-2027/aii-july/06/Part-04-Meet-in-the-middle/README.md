## 14. Meet-in-the-middle для задач около $n \approx 40$

Иногда полный перебор по всем подмножествам уже слишком велик. Если у массива около $40$ элементов, то вариантов ровно $2^{40}$, и это уже слишком много.

В такой ситуации помогает meet-in-the-middle: мы делим массив пополам, отдельно перебираем все подмножества каждой половины, а потом аккуратно совмещаем результаты.

На числе `40` разница особенно хорошо видна численно:

- прямой перебор по всему массиву потребовал бы просмотреть $2^{40} = 1\ 099\ 511\ 627\ 776$ подмножеств;
- после деления пополам получаем две половины по `20` элементов;
- у каждой половины уже только $2^{20} = 1\ 048\ 576$ подмножеств;
- значит, нужно построить два списка примерно по миллиону сумм, а не один перебор больше чем на триллион вариантов.

Даже если грубо сравнить только число генерируемых сумм, получаем около `2 097 152` значений вместо `1 099 511 627 776`. Это меньше примерно в `524 288` раз.

> [!TIP]
> Если в задаче нужно что-то делать со всеми подмножествами, а размер около `40`, то meet-in-the-middle стоит проверить почти сразу. Для `n = 20` обычно хватает обычных масок, а вот для `n = 40` прямой перебор уже слишком дорог.

Разберём это на конкретной задаче. Пусть нужно найти максимальную сумму подмножества, не превосходящую `21`, для массива

```text
8 5 6 3 12 10
```

### Шаг 1. Делим массив на две половины

Левую половину берём как `{8, 5, 6}`, а правую как `{3, 12, 10}`.

<details>
<summary>Фрагмент на C++</summary>

```cpp
int mid = n / 2;
vector<long long> left(a.begin(), a.begin() + mid);
vector<long long> right(a.begin() + mid, a.end());
```

</details>

<details>
<summary>Фрагмент на Python3</summary>

```python
mid = n // 2
left = a[:mid]
right = a[mid:]
```

</details>

### Шаг 2. Получаем все подмножества левой половины

Для половины `{8, 5, 6}` перебираем все маски от `000` до `111`. Каждая маска показывает, какие элементы мы берём.

<details>
<summary>Фрагмент на C++</summary>

```cpp
vector<long long> build_sums(const vector<long long>& part) {
	vector<long long> sums;
	int m = (int)part.size();
	sums.reserve(1 << m);
	for (int mask = 0; mask < (1 << m); mask++) {
		long long current_sum = 0;
		for (int i = 0; i < m; i++) {
			if ((mask >> i) & 1) current_sum += part[i];
		}
		sums.push_back(current_sum);
	}
	return sums;
}
```

</details>

<details>
<summary>Фрагмент на Python3</summary>

```python
def build_sums(part: list[int]) -> list[int]:
	sums = []
	for mask in range(1 << len(part)):
		current_sum = 0
		for i in range(len(part)):
			if (mask >> i) & 1:
				current_sum += part[i]
		sums.append(current_sum)
	return sums
```

</details>

Для левой половины получаем все подмножества и их суммы:

| Маска | Подмножество | Сумма |
| --- | --- | --- |
| `000` | `{}` | `0` |
| `001` | `{8}` | `8` |
| `010` | `{5}` | `5` |
| `011` | `{8, 5}` | `13` |
| `100` | `{6}` | `6` |
| `101` | `{8, 6}` | `14` |
| `110` | `{5, 6}` | `11` |
| `111` | `{8, 5, 6}` | `19` |

### Шаг 3. Получаем все подмножества правой половины

Теперь делаем ровно то же самое для правой половины `{3, 12, 10}`.

<details>
<summary>Фрагмент на C++</summary>

```cpp
vector<long long> left_sums = build_sums(left);
vector<long long> right_sums = build_sums(right);
sort(right_sums.begin(), right_sums.end());
```

</details>

<details>
<summary>Фрагмент на Python3</summary>

```python
left_sums = build_sums(left)
right_sums = sorted(build_sums(right))
```

</details>

Все подмножества правой половины и их суммы:

| Маска | Подмножество | Сумма |
| --- | --- | --- |
| `000` | `{}` | `0` |
| `001` | `{3}` | `3` |
| `010` | `{12}` | `12` |
| `011` | `{3, 12}` | `15` |
| `100` | `{10}` | `10` |
| `101` | `{3, 10}` | `13` |
| `110` | `{12, 10}` | `22` |
| `111` | `{3, 12, 10}` | `25` |

После сортировки правые суммы идут так:

```text
0, 3, 10, 12, 13, 15, 22, 25
```

> [!NOTE]
> В учебном разборе мы явно перечисляем сами подмножества, чтобы было видно, откуда берутся суммы. В реальном коде обычно достаточно хранить только суммы, потому что для поиска лучшего ответа сами списки элементов часто уже не нужны.

### Шаг 4. Для каждой суммы слева ищем лучшую парную сумму справа

Если слева уже набрана сумма `left_sum`, то справа можно добрать не больше `21 - left_sum`. Значит, среди правых сумм нужно найти максимальную, не превосходящую этого значения.

Именно здесь и используется бинарный поиск.

> [!WARNING]
> Если забыть отсортировать суммы правой половины, бинарный поиск здесь перестанет быть корректным. Ещё одна типичная ошибка: искать справа сумму, равную `21 - left_sum`, вместо максимальной суммы, не превосходящей этого значения.

<details>
<summary>Фрагмент на C++</summary>

```cpp
long long best = 0;
for (long long left_sum : left_sums) {
	if (left_sum > limit_sum) continue;

	long long need = limit_sum - left_sum;
	auto it = upper_bound(right_sums.begin(), right_sums.end(), need);
	if (it == right_sums.begin()) {
		best = max(best, left_sum);
		continue;
	}
	--it;
	best = max(best, left_sum + *it);
}
```

</details>

<details>
<summary>Фрагмент на Python3</summary>

```python
best = 0
for left_sum in left_sums:
	if left_sum > limit_sum:
		continue

	need = limit_sum - left_sum
	pos = bisect_right(right_sums, need) - 1
	if pos >= 0:
		best = max(best, left_sum + right_sums[pos])
	else:
		best = max(best, left_sum)
```

</details>

Для нашего примера получаем такие совмещения:

| Левое подмножество | Сумма слева | Можно добрать справа не больше | Лучшая правая сумма | Пример правого подмножества | Итог |
| --- | --- | --- | --- | --- | --- |
| `{}` | `0` | `21` | `15` | `{3, 12}` | `15` |
| `{8}` | `8` | `13` | `13` | `{3, 10}` | `21` |
| `{5}` | `5` | `16` | `15` | `{3, 12}` | `20` |
| `{8, 5}` | `13` | `8` | `3` | `{3}` | `16` |
| `{6}` | `6` | `15` | `15` | `{3, 12}` | `21` |
| `{8, 6}` | `14` | `7` | `3` | `{3}` | `17` |
| `{5, 6}` | `11` | `10` | `10` | `{10}` | `21` |
| `{8, 5, 6}` | `19` | `2` | `0` | `{}` | `19` |

Максимальный допустимый итог равен `21`.

Успешные совмещения здесь даже не одно, а сразу несколько:

- `{8}` слева и `{3, 10}` справа;
- `{6}` слева и `{3, 12}` справа;
- `{5, 6}` слева и `{10}` справа.

Во всех трёх случаях получаем сумму `21`.

Полные программы для той же задачи через meet-in-the-middle: [C++](examples/example-02.cpp), [Python3](examples/example-02.py).
