## 5. Зачем после циклов вообще нужна рекурсия

В первой части мы уже научились генерировать знакомые объекты: перестановки, двоичные строки, троичные строки. Там ответы удобно было получать циклами, масками и `next_permutation`.

Но у всех этих примеров есть и другая, очень важная форма записи: можно строить ответ не целиком сразу, а по одному шагу. Сначала выбираем первый символ или первый элемент, потом второй, потом третий и так далее. Именно такой взгляд и приводит к рекурсии.

Будем думать так: каждый вызов `dfs(...)` отвечает за некоторый уже построенный префикс ответа. Этот префикс зафиксирован, а функция выбирает, как его продолжить.

Например, если мы строим двоичную строку длины $3$, то в корне дерева префикс пустой. Из него выходят две ветви: дописать `0` или дописать `1`. Из префикса `0` снова выходят две ветви, и так далее. Рекурсия просто проходит по такому дереву сверху вниз.

Поэтому запись `dfs(pos, ...)` удобно читать так: первые `pos` шагов уже сделаны, теперь нужно выбрать шаг номер `pos`. Остальные параметры описывают состояние текущего префикса: текущую сумму, баланс, массив `used[]`, последний выбранный элемент и всё, что ещё важно для продолжения.

> [!IMPORTANT]
> Сначала формулируем состояние, а уже потом пишем рекурсию.
>
> Иначе код быстро превращается в набор вызовов без понятного смысла.

### Что такое рекурсия и что делает стек вызовов

Рекурсия — это обычная функция, которая в какой-то момент вызывает саму себя, но уже для другого состояния. В наших задачах это значит: функция получила некоторый префикс ответа и передаёт более глубокому вызову задачу достроить оставшуюся часть.

Важно, что рекурсивный вызов ничем не отличается от любого другого вызова функции. Когда программа выполняет `dfs(pos + 1, ...)`, она создаёт новый кадр в стеке вызовов. В этом кадре лежат:

- параметры нового вызова;
- его локальные переменные;
- место в коде, куда нужно вернуться после `return`.

Предыдущий вызов при этом не исчезает. Он просто ждёт, пока более глубокий вызов закончит работу. Когда тот завершается, его кадр снимается со стека, и выполнение продолжается ровно со следующей строки после вызова.

Именно поэтому в полном переборе так естественно работает схема: сначала изменили состояние, потом вызвали рекурсию, а после возврата откатили изменение. Например, в строках `current += '0'; dfs(pos + 1); current.pop_back();` первый вызов добавляет символ, второй уходит глубже, а после возврата мы снова оказываемся в том же старом вызове и можем аккуратно убрать добавленный символ.

> [!NOTE]
> Стек вызовов сам хранит всю цепочку незавершённых состояний.
>
> Поэтому после возврата из `dfs(pos + 1)` нам не нужно заново восстанавливать параметры вызова `dfs(pos)`: программа уже вернулась именно в него.

## 6. Те же двоичные строки, но рекурсией

Начнём с самого знакомого примера из первой части: все двоичные строки длины $n$.

Основная идея: если первые `pos` символов уже выбраны, то на позицию `pos` можно поставить либо `0`, либо `1`. После этого вызываем рекурсию для следующей позиции. Когда `pos == n`, строка уже построена полностью, и её можно печатать.

<details>
<summary>Пример на C++</summary>

```cpp
void dfs(int pos) {
	// Если дошли до длины n, строка уже полностью построена.
	if (pos == n) {
		cout << current << '\n';
		return;
	}

	// Ветка 1: ставим на позицию pos символ '0'.
	current += '0';
	dfs(pos + 1);
	current.pop_back();

	// Ветка 2: ставим на позицию pos символ '1'.
	current += '1';
	dfs(pos + 1);
	current.pop_back();
}
```

</details>

<details>
<summary>Пример на Python3</summary>

```python
def dfs(pos: int) -> None:
	# Если дошли до длины n, строка уже полностью построена.
	if pos == n:
		print(''.join(current))
		return

	# Ветка 1: ставим на позицию pos символ '0'.
	current.append('0')
	dfs(pos + 1)
	current.pop()

	# Ветка 2: ставим на позицию pos символ '1'.
	current.append('1')
	dfs(pos + 1)
	current.pop()
```

</details>

Полные программы: [C++](examples/example-01.cpp), [Python3](examples/example-01.py).

## 7. Те же троичные строки, но рекурсией

Теперь возьмём второй знакомый пример: строки из цифр `0`, `1`, `2`.

Здесь ничего принципиально не меняется. В каждой вершине дерева уже не две ветви, а три: можно дописать `0`, `1` или `2`. Поэтому рекурсивный шаблон получается даже естественнее, чем в варианте с переводом числа в троичную запись.

Более того, этот же код сразу обобщается на любой маленький алфавит: достаточно вместо трёх фиксированных цифр запускать цикл по всем допустимым символам.

<details>
<summary>Пример на C++</summary>

```cpp
void dfs(int pos) {
	// Если дошли до длины n, строка уже полностью построена.
	if (pos == n) {
		cout << current << '\n';
		return;
	}

	// На позицию pos по очереди ставим все допустимые цифры.
	for (char digit = '0'; digit <= '2'; digit++) {
		current += digit;
		dfs(pos + 1);
		current.pop_back();
	}
}
```

</details>

<details>
<summary>Пример на Python3</summary>

```python

```

## 8. Те же перестановки, но рекурсией

С перестановками ситуация чуть интереснее. В двоичных и троичных строках на каждой позиции набор вариантов был одинаковым. У перестановки так уже нельзя: число, которое мы поставили раньше, второй раз брать нельзя.

Поэтому состояние здесь состоит не только из текущего префикса, но и из массива `used[]`, который помнит, какие элементы уже заняты.

Если первые `pos` мест уже заполнены, то `dfs(pos)` перебирает все числа, которые ещё не использованы, ставит одно из них следующим, уходит глубже, а потом откатывает изменение.

<details>
<summary>Пример на C++</summary>

```cpp
void dfs(int pos) {
	// Если длина префикса уже равна n, перестановка готова.
	if (pos == n) {
		print_permutation();
		return;
	}

	// Пробуем поставить на позицию pos любое ещё не использованное число.
	for (int value = 1; value <= n; value++) {
		if (used[value - 1]) continue;

		// Занимаем число value и углубляемся в следующую позицию.
		used[value - 1] = 1;
		current.push_back(value);
		dfs(pos + 1);
		// После возврата обязательно откатываем все изменения.
		current.pop_back();
		used[value - 1] = 0;
	}
}
```

</details>

<details>
<summary>Пример на Python3</summary>

```python
def dfs(pos: int) -> None:
	# Если длина префикса уже равна n, перестановка готова.
	if pos == n:
		print(*current)
		return

	# Пробуем поставить на позицию pos любое ещё не использованное число.
	for value in range(1, n + 1):
		if used[value - 1]:
			continue

		# Занимаем число value и углубляемся в следующую позицию.
		used[value - 1] = True
		current.append(value)
		dfs(pos + 1)
		# После возврата обязательно откатываем все изменения.
		current.pop()
		used[value - 1] = False
```

</details>

Полные программы: [C++](examples/example-03.cpp), [Python3](examples/example-03.py).

> [!WARNING]
> Если забыть откатить `used[]` или убрать последний элемент из текущего префикса, всё дерево перестановок сразу сломается.

## 9. Шаблон «взять или не взять»

Теперь можно выделить ещё один очень важный частный случай. Иногда на каждом шаге мы не выбираем один из многих символов, а отвечаем на бинарный вопрос: включать объект в ответ или нет.

Такой шаблон лежит в основе перебора всех подмножеств. Для элемента с номером `pos` есть ровно два перехода:

- не брать его и перейти дальше;
- взять его и перейти дальше.

Это ровно то же дерево решений, только теперь ветви означают не символы строки, а решения про элементы множества.

<details>
<summary>Пример на C++</summary>

```cpp
void dfs(int value, int n) {
	// Если дошли до числа n + 1, одно подмножество уже готово.
	if (value > n) {
		print_subset();
		return;
	}

	// Ветка 1: не берём value.
	dfs(value + 1, n);

	// Ветка 2: берём value.
	current.push_back(value);
	dfs(value + 1, n);
	current.pop_back();
}
```

</details>

<details>
<summary>Пример на Python3</summary>

```python
def dfs(value: int) -> None:
	# Если дошли до числа n + 1, одно подмножество уже готово.
	if value > n:
		print_subset()
		return

	# Ветка 1: не берём value.
	dfs(value + 1)

	# Ветка 2: берём value.
	current.append(value)
	dfs(value + 1)
	current.pop()
```

</details>

## 10. Допустимые префиксы: когда не все ветви имеют смысл

Во всех примерах выше любой построенный префикс можно было продолжать дальше. Но так бывает не всегда.

Например, в правильной скобочной последовательности нельзя допускать отрицательный баланс. Если префикс уже плохой, дальше его продолжать бессмысленно: никакой суффикс это не исправит.

В этом и состоит главная сила рекурсии: она позволяет не только строить ответ по шагам, но и рано отбрасывать ветви, которые уже сейчас заведомо не приведут к решению.

<details>
<summary>Пример на C++</summary>

```cpp
void dfs(int open_used, int close_used, int n) {
	// Если использовали все n открывающих и все n закрывающих скобок, ответ готов.
	if (open_used == n && close_used == n) {
		cout << current << '\n';
		return;
	}

	// Открывающую скобку можно добавить, пока не исчерпали лимит.
	if (open_used < n) {
		current += '(';
		dfs(open_used + 1, close_used, n);
		current.pop_back();
	}

	// Закрывающую скобку добавляем только к допустимому префиксу.
	if (close_used < open_used) {
		current += ')';
		dfs(open_used, close_used + 1, n);
		current.pop_back();
	}
}
```

</details>

<details>
<summary>Пример на Python3</summary>

```python
def dfs(open_used: int, close_used: int) -> None:
	# Если использовали все n открывающих и все n закрывающих скобок, ответ готов.
	if open_used == n and close_used == n:
		print(''.join(current))
		return

	# Открывающую скобку можно добавить, пока не исчерпали лимит.
	if open_used < n:
		current.append('(')
		dfs(open_used + 1, close_used)
		current.pop()

	# Закрывающую скобку добавляем только к допустимому префиксу.
	if close_used < open_used:
		current.append(')')
		dfs(open_used, close_used + 1)
		current.pop()
```

</details>
