# Сжатие координат, отрезков и прямоугольников

## О чём лекция

Сжатие координат — это техника, которая позволяет заменить большие значения на компактные индексы, сохранив порядок между ними.
Она особенно полезна в задачах, где координаты или значения могут быть очень большими, но реально используются только некоторые из них. 
В лекции эта идея разбирается сначала на одномерном примере, затем переносится на отрезки и прямоугольники.

## Зачем это нужно

Если значения маленькие, можно завести массив по всем возможным координатам и работать напрямую. 
Но когда числа доходят до $10^{18}$ или похожих величин, хранить массив такого размера уже нельзя, и приходится оставлять только реально встречающиеся координаты.
Именно это и делает сжатие: оно выбрасывает пустые промежутки и оставляет только важные точки. 

## Базовая идея

Пусть дан массив чисел `a[i]`, и для каждого запроса `[L, R]` нужно найти сумму тех элементов массива, чьи значения попадают в диапазон от `L` до `R`. 
Если бы значения были маленькими, можно было бы хранить в `b[x]` вклад всех элементов со значением `x`, а затем посчитать префиксные суммы. 
Сжатие координат делает почти то же самое, только вместо оси `0..10^18` мы работаем по оси из уникальных встречающихся значений. 

## Классическая схема решения

1. Собираем все значения `a[i]` в вектор `x`.  
2. Сортируем `x`. 
3. Удаляем дубликаты через `unique` и `erase`, потому что `unique` сам по себе только сдвигает уникальные элементы в начало диапазона. 
4. Для каждого `a[i]` находим его индекс в `x` через `lower_bound`.
5. На позиции этого индекса накапливаем вклад элемента. 
6. Строим префиксные суммы по массиву вкладов. 

Минимальный шаблон на C++ выглядит так:

```cpp
sort(x.begin(), x.end());
x.erase(unique(x.begin(), x.end()), x.end());

int id = lower_bound(x.begin(), x.end(), value) - x.begin();
```

Функция `lower_bound` возвращает первый элемент в отсортированном диапазоне, который не меньше искомого значения. 
Для обычного суммирования префиксы можно строить через `partial_sum` или `inclusive_scan`; в лекции обсуждаются оба подхода. 

## Как отвечать на запросы правильно

Самый безопасный вариант — хранить префиксный массив размера `m + 1`, где `pref[0] = 0`, а `pref[i + 1] = pref[i] + b[i]`.  
Тогда для запроса `[L, R]` удобно делать так:

```cpp
int l = lower_bound(x.begin(), x.end(), L) - x.begin();
int r = upper_bound(x.begin(), x.end(), R) - x.begin();
long long ans = pref[r] - pref[l];
```

Здесь `lower_bound` находит первую координату `>= L`, а `upper_bound` — первую координату `> R`, поэтому формула сразу работает без лишних поправок на границы. 
По смыслу это соответствует тому, что мы берём все сжатые координаты, попавшие в диапазон значений от `L` до `R`. 

## Асимптотика

Чтение массива занимает $O(N)$, сортировка — $O(N \log N)$, построение префиксов — $O(N)$. 
Если каждый запрос обрабатывается двумя бинарными поисками, то все запросы дают ещё $O(Q \log N)$. 
Итоговая оценка получается порядка $O((N + Q)\log N)$. 

## Продвинутое сжатие без повторного бинарного поиска

В лекции разбирается более продвинутый подход: значения `a[i]`, левые границы `L[i]` и правые границы `R[i]` складываются в один общий массив записей. 
Каждая запись хранит значение, тип источника и индекс, например что это элемент массива, левая граница запроса или правая граница запроса. 
После сортировки можно пройти по этому массиву и сразу сопоставить всем нужным объектам их позиции, не делая отдельный `lower_bound` для каждого элемента. 

Ключевая деталь — порядок равных значений. 
Для инклюзивных запросов `[L, R]` в лекции используется порядок типов `L < a < R`, чтобы на одинаковых координатах левая граница считалась раньше элемента, а элемент — раньше правой границы. 
Это позволяет аккуратно обработать совпадения на границах диапазона. 

## Детали C++

В лекции отдельно проговаривается стандартная связка `sort` + `unique` + `erase`.
Это базовый паттерн для получения отсортированного массива уникальных координат. 
Также обсуждается `lower_bound` как стандартный бинарный поиск по отсортированному диапазону.

Для префиксных сумм упоминаются `partial_sum`, `inclusive_scan` и `exclusive_scan`. 
При обычном сложении `partial_sum` и `inclusive_scan` дают привычный массив префиксов, а сами scan-алгоритмы формально относятся к отдельному семейству стандартных алгоритмов. 
На практике в задачах по алгоритмам важно не название функции, а то, что после неё сумма на диапазоне считается за \(O(1)\). 

## Сжатие отрезков

Когда мы переходим от отдельных точек к отрезкам, объектами становятся уже не только координаты концов, но и промежутки между соседними важными координатами. 
Именно поэтому сжатие отрезков требует аккуратнее следить за длинами: после сжатия важна не просто позиция, а реальная длина элементарного интервала в исходных координатах. 
В лекции этот переход обсуждается как следующий естественный шаг после обычного сжатия точек. 

Для задач с добавлением и удалением отрезков полезно мыслить не исходной числовой прямой, а набором сжатых элементарных промежутков. 
Тогда обновления делаются по индексам, а длина ответа восстанавливается через реальные разности соседних координат. 
Это и есть причина, почему сжатие так часто встречается в задачах с очень большими границами. 

## Сжатие прямоугольников

На плоскости идея расширяется на две оси: отдельно собираются важные координаты по `x` и по `y`. 
После этого плоскость разбивается на сетку из элементарных ячеек, и внутри каждой такой ячейки состояние уже однородно: она либо целиком покрыта, либо целиком не покрыта. 
Это позволяет перейти от огромной плоскости к компактной сетке по важным границам прямоугольников. 

Именно на этой идее основана задача ACMP 380: дано \(N\) прямоугольников со сторонами, параллельными осям координат, и требуется найти площадь фигуры, образованной их объединением. [acmp](https://acmp.ru/index.asp?main=task&id_task=380)
Для такой задачи после сжатия удобно суммировать площади покрытых ячеек, где площадь ячейки равна \(\Delta x \cdot \Delta y\). [acmp](https://acmp.ru/index.asp?main=task&id_task=380)
В лекции эта задача используется как важная мотивация для изучения сжатия прямоугольников. [acmp](https://acmp.ru/index.asp?main=task&id_task=380)

## Где появляется сканирующая прямая

В лекции отдельно поднимается вопрос о сканирующей прямой. 
Сжатие координат и sweep line часто идут вместе: сжатие уменьшает размер оси, а сканирующая прямая позволяет обрабатывать события в правильном порядке. 
Даже если в конкретном решении хватает одной только компрессии, понимание этой связи очень полезно для более сложных задач на площадь, покрытие и объединение фигур. 

## Частые ошибки

- Забыть, что `unique` не удаляет элементы из вектора сам, а только возвращает новый логический конец диапазона. 
- Перепутать смысл `lower_bound`: это первый элемент `>= value`, а не первый строго больший. 
- Ошибиться на границах диапазона при работе с префиксными суммами.  
- Сжимать только точки, когда задача на самом деле требует сжимать промежутки между соседними координатами. 
- Не продумать порядок равных событий в продвинутом варианте сжатия. 

## Что стоит запомнить

Сжатие координат — это не просто “заменить числа на маленькие”, а выделить все реально важные координаты и работать только с ними. 
Для точек обычно хватает массива уникальных значений и бинарного поиска, а для отрезков и прямоугольников нужно ещё учитывать длины элементарных промежутков между соседними координатами. 
Если вы уверенно понимаете эту разницу, то очень многие задачи на диапазоны, покрытия и геометрию становятся заметно проще.
